Reflect

前言

很多人在学习 ES6 的时候，都会产生一个疑问，就是这个 Reflect 到底是什么，它到底起了一个什么样的作用。其实要回答它的作用只需要一句话：就是 调用对象的基本方法（或者叫 基本操作 或者叫 内部方法，都是一个意思）。

对象的内部方法

剩下的事情就是要解释一下什么是基本操作，什么叫内部方法。内部方法也好，基本操作也好，他们都是指向同一个意思，这一点在 ES6 的 官方文档 里描述的非常清楚。

在官方文档的 pdf 文档中，找到 Table 4: Essential Internal Methods，它里面就罗列了 对象 里边所有的 基本操作，或者叫 内部方法。它指的是什么意思呢？就是 JS 开发者在代码层面无论怎么使用对象，它最终都要对应到这里边的内部方法。

比方说下面的代码实例：

const obj = {}

// 给对象的某一个属性赋值，这是一个语法层面的东西，
// 它在真正运行的时候，它运行的是对象对边的基本方法，
// 这个基本方法的名称是：[[Set]]。
// [[Set]] 是一个函数，它的作用就是给对象的某个属性赋值。
obj.a = 1

// 如果在读取对象某个属性的时候，它内部有一个基本方法，
// 叫做 [[Get]]
obj.a

方法名称如下图所示：

也就是说 我们书写的语法层面的代码，它最终运行的时候，实际上运行的是它的内部方法。对一个对象的所有操作都逃不出这一点。

Reflect 的作用

接下来再看一下 Reflect，Reflect 是反射的意思。它的 作用是可以不用去管这个对象的书写语法了，你可以直接去调用对象的内部方法。（调用的是内部方法对应的反射方法）

比方说下面的代码实例：

const obj = {}

// 在 ES6 之前没有任何办法去直接调用内部方法
obj.a = 1
obj.a

// Reflect 的写法
// 直接调用基本方法，这里的 set 是内部方法对应反射方法，
// 具体的内部方法对应的反射方法可以查看 mdn 文档
Reflect.set(obj, 'a', 2)

那么这个 Reflect 有什么意义呢？我给这个对象的某一个属性赋值，直接赋值不也可以实现结果吗？为什么还要使用 Reflect 呢？

这是因为，通过 obj.a = 1 的语法或者是某一些 API，间接去调用这个内部方法的话，它和直接调用内部方法 Reflect.set(obj, 'a', 2) 是有区别的。

区别在哪呢？当你使用某一种语法去做一件事的时候，实际上它会经过一系列的 规则和步骤，在这些规则和步骤当中，其中某一步是在调用它内部的 基本方法。也就是说如果是没有通过 Reflect 来操作对象的话，它除了在内部调用 基本方法 之外，还有一些 额外的步骤，这些额外的步骤是什么就要看你写的语法是什么。

如果你不希望有这些额外步骤的存在，那么你就必须要直接调用这个 基本方法 才行，也就说只能通过使用 Reflect 才行。

比方说举个例子，看一下代码：

const obj = {
    a: 1,
    b: 2,
    get c() {
        return this.a + this.b
    }
}

// 小 tips：
// 上面的 obj.c 的写法，相当于下面的 defineProperty 写法
// 上面的写法相当于是一个语法糖
// Obejct.defineProperty(obj, 'c', {
//	get() {},
//	set() {}
//})

console.log(obj.c) // 打印结果是 3

当我们去读取这个属性 c 的时候，它内部是不是在调用基本方法。调用的是谁？调用的是这个 [[Get]] 方法。

根据之前的截图，我们可以看到 [[Get]] 方法的使用：(propertyKey, Receiver) → any。它接收两个参数，最后一个参数是 Receiver，这个 Receiver 指定的是什么？它指定的是对象里边的 this 指向。什么意思呢？

就当你读取某一个属性的时候，它是有可能会经过一个函数的，像这里的情况，读取 c 属性 的时候，经过了 get() 的方法。那么这个 get 函数 里边的 this 指向谁是由这个 内部方法 里边的第二个参数 Receiver 来决定的。这个 Receiver 是什么，this 就指向什么。

我们再看一下这个语法层面 console.log(obj.c) ，在这个语法层面上，我们并没有给它指定 this，但它的 this 指向谁呢？它指向的是 obj。我们通过打印结果是 3，就说明这里的 this 指向的是 obj。

也就是意味着通过这种语法的操作，它做了那些事情呢？它并不是在直接调用 基本方法。它首先会把这个 obj 定义为 this，然后再去调用这个 基本方法 [[Get]]，调用的时候传入 obj 对象、obj 的 c 属性、读的时候 this 的指向：[[Get]](obj, 'c', obj)。它就会有这么一个间接的处理，这个间接的处理我们是 无法 通过 语法层面 来干涉的。

也就意味着，如果说你希望改动这里的 this 指向，改不了。但是，如果你通过 Reflect 的话，就可以做到了。

比方说下面的代码示例：

// Reflect 读取了 对象obj，并且手动改变了 this 的指向，
// this 指向了 { a:3, b: 4 }
console.log(Reflect.get(obj, 'a', { a: 3, b: 4 })) // 打印结果 7

Reflect 和 proxy 的结合使用

可能又有人会说了，Reflect 这东西平时谁会这样去写代码呢。那就要看你做的是什么事了，比方说你要封装一些 代理对象 的时候，Reflect 就很重要了。

举个例子：

const obj = {
    a: 1,
    b: 2,
    get c() {
        return this.a + this.b
    }
}

// 创建一个代理对象 proxy
const proxy = new Proxy(obj, {
    get(target, key) {
        console.log(`读取了 ${key} 属性`)
        return target[key]
    },
    
    set(target, key, val) {
        // ...
    }
})

// 读取 属性a，打印结果是正常的
proxy.a // 打印：读取了 a 属性

// 接下来读取 属性c，按理说打印结果是什么呢？
// 它不仅读取了 属性c，属性c 里又通过 get 方法
// 读取了 属性a 和 属性b。所以，应该要打印 三个 属性。
// 但是这里只打印了 属性c。
proxy.c // 打印：读取了 c 属性

上述的代码示例中，为什么读取 属性c 的时候，只打印了一次呢？因为 Proxy 的 get 方法在返回的时候，是直接把那个 原始对象 的 c 属性返回，也就是 return target[key]，那么这样子一来，这个 c 里边的 this 会指向谁？会指向那个原始对象 target，target 就是 obj 对象。那么读取 target 里面的 a 和 b 是不是就收不到任何通知了。因为你必须要读取代理对象里边的 a 和 b 才能收到通知。

也就是说我在读取这个 属性c 的时候，是不是希望那些 this 指向要指向 代理 才行。因此，上面的代码应该怎么写？应该要写成 Reflect：

const proxy = new Proxy(obj, {
    get(target, key) {
        console.log(`读取了 ${key} 属性`)
        // return target[key]
        
        // 改成 Reflect 的写法
        // 获取 target 的 key 属性，并且改变 this 指向为 proxy 代理对象
        return Reflect.get(target, key, proxy)
    }
    
    set(target, key, val) {
        // ...
    }
})

其他实例

const obj = {
    a: 1,
    b: 2
}

// 我们要得到一个对象的所有属性，我们应该用 Object.keys
const keys = Object.keys(obj)
console.log(keys) // ['a', 'b']

实际上 Object.keys 这个 API 的内部也会调用 基本方法，基本方法 叫做 [[GetOwnProperty]]，得到一个对象的所有属性。但是，它还是会和之前说的一样，它也是会经过一系列的 规则和步骤。

如果说我不希望它做哪些额外的步骤和处理，就必须要使用 Reflect：

// 使用 Reflect，ownKeys 是内部方法 [[OwnPropertyKeys]] 的反射方法
const keys = Reflect.ownKeys(obj)
console.log(keys) // ['a', 'b']

其他 Reflet 的映射方法可以查看 MDN 文档。

总结

理解了这一切过后，再回来看一下这句话： 调用对象的基本方法（或者叫 基本操作 或者叫 内部方法），就非常清晰了。就是在 ES6 之前是没有任何办法直接去操作这个 基本方法 的，只能通过语法层面去间接调用。但是间接地调用会做一些额外的操作，不够直接。

但是有了 Reflect 之后，我们可以直接使用对象的 基本方法。

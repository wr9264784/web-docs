apply、bind、call 方法 改变 this 指向

一、改变 this

Javascript 中还允许指定函数中 this 的指向，有 3 个方法可以动态指定普通函数中 this 的指向：

  • call()
  • apply()
  • bind()

二、call（了解）

使用 call 方法调用函数，同时指定被调函数中 this 的值。

语法：

fun.call(thisArg, arg1, arg2, ....)

说明：

  • thisArg：在 fun 函数运行时指定的 this 值（写谁 this 就指向谁）。
  • arg1，arg2：传递的其他参数。
  • 返回值就是函数的返回值，因为它就是调用函数。

如下所示：

function fn(params1, params2) {
    console.log(this) // window
    console.log(params1, params2)
}

const obj = {
    name: '张三'
}

// call 的作用：
// 1. 调用函数
// 2. 改变指向 this
fn.call(obj, '李四', '王五') // { name: '张三' }，'李四', '王五'

三、apply（理解）

使用 apply 方法调用函数，同时指定被调用函数中 this 的值。

语法：

fun.apply(thisArg, [argsArray])

说明：

  • thisArg：在 fun 函数运行时指定的 this 值。
  • argsArray：传递的参数值，必须包含在数组里面。
  • 返回值就是函数的返回值，因为它就是调用函数。
  • 因此 apply 主要跟数组有关系，比如使用 Math.max() 求数组的最大值。

如下所示：

function fn(params1, params2) {
    console.log(this) // window
    console.log(params1, params2)
}

const obj = {
    name: '张三'
}

// apply 的作用：
// 1. 调用函数
// 2. 改变指向 this
// 注意：第二个参数必须是一个数组
fn.apply(obj, ['李四', '王五']) // { name: '张三' }，'李四', '王五'

使用场景：

求数组最大值：

const arr = [1, 3, 4, 2, 5, 7, 9, 8, 0]

// 不用 apply 时
// 需要传递的是解构后的各个数组项
const max1 = Math.max(...arr)

// 使用 apply 时
// 可以直接传递整个数组，不需要解构
// this 的指向不指定时，可以传 null
const max2 = Math.max.apply(Math, arr)

四、bind（重点）

bind 方法不会调用函数，但是能改变函数内部 this 指向。

语法：

fun.bind(thisArg, arg1, arg2, ...)

说明：

  • thisArg：在 fun 函数运行时指定的 this 值。
  • arg1，arg2：传递的其他参数。
  • 返回值由指定的 this 值和初始化参数改造的 原函数拷贝（新函数）。
  • 因此当我们只是想改变 this 指向，并且不想调用这个函数的时候，可以使用 bind，比如改变定时器内部的 this 指向。

如下实例：

function fn(params1, params2) {
    console.log(this) // window
    console.log(params1, params2)
}

const obj = {
    name: '张三'
}

// bind 的作用：
// 1. 不调用函数
// 2. 改变指向 this
// 返回值是一个原函数的拷贝函数，返回的新函数的 this 是改变的
const newFn = fn.bind(obj, '李四', '王五') // 不调用函数，没有打印结果

newFn() // { name: '张三' }，'李四', '王五'

使用场景：

当我们想改变 this 指向，但又不想立即调用该函数时。

比如：定时器的使用。

// 需求：有一个按钮，点击立马就禁用，2s 后开启

const btn = document.querySelector('button')
btn.onclick = function() {
    this.disabled = true
    
    setTimeout(function() {
        // // 这里用的是普通函数，不是箭头函数
        // // this 指向的是 window
        // this.disabled = false
        
        // btn.disabled = false
        
        // 如果我们不想使用 btn，就是想使用 this 来控制
        // 就需要用到 bind
        this.disabled = false
        
        // 注意，这里 .bind(btn) 中的 btn 和 onclick 里的 this 是一样的，所以 .bind(btn) 和改成 .bind(this)
    }.bind(this), 2000)
}

五、call、apply、bind 总结

相同点：

  • 都可以改变函数内部的 this 指向。

区别点：

  • call 和 apply 会调用函数，并且改变函数内部的 this 指向。
  • call 和 apply 传递的参数不一样，call 传递参数以 arg1, arg2.. 的形式传递，apply 必须以数组的形式 [args]。
  • **bind 不会调用函数 **，可以改变函数内部的 this 指向。
  • bind 传递的参数形式和 call 一样。

主要应用场景：

  • call 调用函数并且可以传递参数。
  • apply 经常跟数组有关系，比如借助数学对象实现数组最大值、最小值。
  • bind 不调用函数，但是还想改变 this 指向，比如改变定时器内部的 this 指向。

this 指向

环境对象

环境对象：指的是 函数内部特殊的变量 this ，它代表着当前函数运行时所处的环境。

作用：弄清楚 this 的指向，可以让我们代码更简洁。

  1. 函数的调用方式不同，this 指代的对象也不同。
  2. 【谁调用， this 就是谁】 是判断 this 指向的 粗略规则。
  3. 直接调用函数，其实相当于通过 window.函数() 的形式调用，所以 this 指代 window。
function fn () {
  console.log(this); // 打印结果是 window
}

fn() // 直接调用函数，其实相当于 window.fn() 进行调用

在非严格模式下，默认不改变 this 指向的前提下，普通函数 的 this 指向的是 window 对象。

// 页面上有一个 button 按钮
const btn = document.querySelector('button')

btn.addEventListener('click', function(){
  // 指向函数的调用者，这里是 button 调用了该回调函数，
  // 所以，this 指向 button 按钮 DOM 对象。
  console.log(this);
})

普通函数中的 this 指向

注意：在函数未被调用时，函数内部的 this 指向是无法确定的。

在 绝大多数情况下，函数的 调用方式 决定了 this 的值（运行时绑定）。this 不能在 执行期间 被赋值，并且在每次函数被调用时 this 的值也可能会不同。

常规情况下，函数中的 this 取决于 执行函数的方式（四种绑定规则）。

默认绑定

在 默认情况 下（即函数独立调用，并没有绑定在某个对象上调用时），this 的指向都为 Window对象。（严格模式 下为 undefined）。

function myFun() {
    console.log(this);
}

function myFun2(fun) {
    fun();
}

myFun(); // Window
"use strict"; // 严格模式

function myFun() {
    console.log(this);
}

function myFun2(fun) {
    fun();
}

myFun(); // undefined

隐式绑定

当函数被 某个对象调用时，函数的 this 就会 自动指向调用它的对象，这种行为可以称之为 隐式绑定。通过 隐式绑定 我们可以大概搞明白，为什么在全局中的函数调用会绑定 Window 了，因为 fun() 其实就相当于 Window.fun()，是 Window对象 调用了 fun 函数。

function foo() {
  console.log(this);
}

const obj = {
  name: "obj",
  foo: foo
};

const fun = obj.foo;

foo() // window
fun() // window
obj.foo(); // {name: 'obj', foo: f}
"use strict"; // 严格模式

function foo() {
  console.log(this);
}

const obj = {
  name: "obj",
  foo: foo
};

const fun = obj.foo;

foo() // undefined
fun() // undefined
obj.foo(); // {name: 'obj', foo: f}

显示绑定

js 为我们提供了 三种 显示绑定 this 的方法，分别为 call、apply、bind。

通过 call、apply、bind 可以手动设置 函数的 this 指向，并且可以根据 不同的规则 给函数传参，这种方式称之为 显式绑定。

const obj = {
    name: "obj",
    a: 20,
    b: 30,
};

function add(c) {
    console.log(this.a + this.b + c);
}

add.call(obj, 10); // 60
add.apply(obj, [30]); // 80
const newFun = add.bind(obj, 20); 
newFun(); // 70

在 js 中 所有函数 都可以使用 call、apply、bind 方法；

call、apply 是在 原有函数上 绑定 this，bind 则是 返回一个新函数。

三者的 第一个参数 都是要绑定 this 的值。

call 的传参方式是一个 列表，有多少参数就传多少，apply 的传参方式是把 所有参数 放进一个 数组，然后把这个数组传过来，bind 的传参和 call 一样。

构造函数绑定

构造函数 中的 this 指向的是通过该 构造函数 实例化 出来的 对象。

// 声明一个构造函数
function Person(name, age, gender) {
    // this 表示指向实例化出来的那个对象
    // 这里 this.name = name 表示：
    // 给实例化创建出来的对象添加一个 name 属性，属性的值是接收的形参的值
    // this.age 和 this.gender 同理
    this.name = name
    this.age = age
    this.gender = gender
}

// 创建张三对象
const zhangsan = new Person('张三', 18, '男')

console.log(zhangsan) // { name: '张三', age: 18, gender: '男' }

箭头函数中的 this 指向

箭头函数 中的 this 与 普通函数 完全不同，也不受调用方式的影响，事实上 箭头函数中并不存在 this。

  1. 箭头函数 会默认帮我们绑定 外层 this 的值，所以在 箭头函数 中 this 的值和外层的 this 是一样的。
  2. 箭头函数 中 this 引用的就是 最近作用域中的 this。
  3. 向外层作用域中，一层一层地查找 this，直到有 this 的定义。
const user = {
  name: '张三',

  // 箭头函数
  fn: () => {
    // 箭头函数内部没有 this，它绑定的是 最近作用域中的 this
    // 该箭头函数的上一级是 user 对象，但是对象中并没有作用域，
    // 所以，继续往上查找，到了最外层的作用域，也就是 window 对象。
    console.log(this);
  }
}

user.fn() // window

再看一个例子：

const btn = document.querySelector('button')

// 箭头函数，this 指向了 window
btn.addEventListener('click', () => {
  console.log(this);
})

// 普通函数，this 指向了 button
btn.addEventListener('click', function() {
  console.log(this);
})

总结：

  1. 箭头函数中不存在 this，沿用上一级的作用域中的 this。
  2. 箭头函数 **不适用 **于 构造函数、原型函数、dom元素的事件函数 等等。
    1. 因为 构造函数 中的 this 需要指向的是 实例化对象，所以不能用箭头函数作为构造函数，this 指向不是 实例化出来的对象，并且还有其他的一些问题（这里不细说）。
    2. 构造函数 中的 原型对象 身上的 原型函数 中的 this 也是指向的 实例化的对象。
    3. dom元素的事件 中，一般也不适用 箭头函数，因为我们一般都是想要通过 事件的回调函数中，通过 this 拿到当前的 dom元素对象，如果使用的 箭头函数，拿到的就 不是当前的 dom 元素对象 了。
  3. 箭头函数 适用 于需要使用上一层 this 的地方。

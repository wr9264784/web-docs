数字格式化

如果我们想要给数字在对应的位数添加逗号，比如：10000000000 -> 10,000,000,000，我们可以进行以下的操作：

/**
 * 给你一个数字字符串：'10000000000'
 * 把它转换成如下格式：10,000,000,000
 */
const str = '10000000000'

// 方式一：纯正则表达式，一行代码实现
/**
 * 那么使用纯正则表达式应该怎么做呢？
 * 
 * 首先我们来分析一下这道题：
 * '10000000000' ---> '10,000,000,000'
 * 
 * 无非就是每 3 位的位置添加一个逗号的进行分割，那么
 * 这些位置有什么样的特点呢？
 * 
 * 首先看最右边最后一个逗号，它后面有 1组 3位 数字，
 * 再看右边倒数第二个逗号，它后面有 2组 3位 数字，
 * 最后左侧第一个逗号，它后面有 3组 3位 数字。
 * 
 * 这就让我们想到了正则表达式。那么怎么写这个正则表达式呢？
 */

// \d{3} 表示数字出现 3 次
// (\d{3})+ 表示数字出现 3 次的情况有一次或多次
// 但是这样匹配的是 10 后面所有的数字了。
// 我们要匹配的不是数字的本身，而是每三位数字前面的位置，这就要用到前瞻。
// const reg = /(\d{3})+$/g

// 把上面的正则中的 (\d{3})+$ 放到 (?= ) 里面，
// 那什么是前瞻呢？
// 就是符合正则表达的所处的前面的位置。
const reg = /(?=(\d{3})+$)/g

// 然后把这些位置全部替换成逗号
const newStr = str.replace(reg, ',')
console.log(newStr) // 10,000,000,000

// 但是光上面的正则还是有点 bug
// 如果我们的数字字符串在加一个零：'100000000000'，
// 这时，就会有问题，会发现 100 前面会多了一个逗号。
const str1 = '100000000000'
const newStr1 = str1.replace(reg, ',')
console.log(newStr1) // ,100,000,000,000

// 这时候就需要修改正则表达式，匹配非单词边界，
// 因为 '100000000000' 里面的 1 前面的位置就是单词的边界，最后一个 0 的后面也是单词的边界。
// 这里的 \B 就表示匹配的是非单词边界
const reg1 = /(?=\B(\d{3})+$)/g

// 然后把这些位置全部替换成逗号
const newStr2 = str1.replace(reg1, ',')
console.log(newStr2) // 100,000,000,000

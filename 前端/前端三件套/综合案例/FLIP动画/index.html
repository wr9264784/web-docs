FLIP 动画

前言

如果用 height，width，top，left等 CSS 属性来做动画，性能要远远差于用 transform 或 opacity 属性的动画。动画元素多的话，前者会带来页面卡顿。

这是因为不同的动画，涉及浏览器的不同阶段：Layout ➠ Pain ➠ Composite。如果单独触发 Composite，那么浏览器的负担就很小，动画会很丝滑。

触发布局的属性一旦发生变化,浏览器就会非常忙碌.它需要递归的计算其它元素的布局是否会发生变动.

一旦这个计算时长超过1个动画帧（一般是 60 帧每秒,也就是说超过16.7ms）， 那么这帧动画将不会绘制，产生页面卡顿。

FLIP 动画技术，就是只利用 transform 或 opacity 模拟布局变动的技巧，因为它不触发Layout，所以动画比较丝滑。

一、FLIP 基础概念

FLIP 代表 First 和 Last 以及 Invert 还有 Play 四个单词的组合。

  • First 初始状态。
  • Last 最终状态。
  • Invert 回到初始状态。
  • Play 执行动画。

二、First

First 就是在最开始的时候，针对要监控的元素（也就是要实现动画的元素），记录这个元素的起始位置。一般是相对于视口的位置，或者相对于页面的位置。

因为当前案例只涉及纵坐标的位置，所以不关注横坐标和其他信息：

const firstItem = document.querySelector('.list-item:first-child')

// First：获取最开始的位置，相对于视口而言
function getLocation() {
  const rect = firstItem.getBoundingClientRect()
  return rect.top
}

三、Last

Last 就是改变元素的位置，并且记录元素结构变化后的位置。也是动画让元素停止的状态。

const btn = document.querySelector('button')

// Last；记录元素变化后的位置
btn.onclick = () => {
  list.insertBefore(firstItem, null) // 把第一个元素放到最后
  const end = getLocation() // 记录变化后的位置
  console.log('Last：', end)
}

四、Invert

Invert 表示倒置（计算元素第一个位置（First）和 最后一个位置（Last）之间的位置变化 ），就是我们已经拿到了 起始位置 和 结束位置，就可以算出它的偏移量，然后利用 transform 就可以把它偏移到最开始的位置。

这里有一个重要的知识点（也是 FLIP 的原理）：

就是当我们在记录结束位置的时候，实际上页面我们是看不到元素在最后一个位置的。当按钮点击后，代码执行到 const end = getLocation() 时，我们拿到结束位置的时候， 实际上页面还没有画出来，它的排版和布局都完成了，但就是还没有渲染出来。

所以，我们可以利用这一点，把元素偏移回最开始的 First 位置。

btn.onclick = () => {
  list.insertBefore(firstItem, null) // 把第一个元素放到最后
  const end = getLocation() // 记录变化后的位置
  console.log('Last：', end)

  // 延迟指定时间，模拟 js 代码正在执行，当 js 代码在执行的过程中，
  // 浏览器不会留有任何的空间去绘制页面。
  // delay()

  const dis = start - end // 计算偏移量
  // 把元素偏移回原来 First 的位置
  // 注意：这个时候我们是看不到偏移的过程的，因为 JS 代码还在执行中，
  // 浏览器不会留有任何的空间去绘制页面。
  firstItem.style.transform = `translateY(${ dis }px)`
  console.log('Invert：', dis);
}

五、Play

最后就是添加过渡效果，让动画动起来，让它回到 Last 的位置。

btn.onclick = () => {
  list.insertBefore(firstItem, null) // 把第一个元素放到最后
  const end = getLocation() // 记录变化后的位置
  console.log('Last：', end)

  // 延迟指定时间，模拟 js 代码正在执行，当 js 代码在执行的过程中，
  // 浏览器不会留有任何的空间去绘制页面。
  // delay()

  const dis = start - end // 计算偏移量
  // 把元素偏移回原来 First 的位置
  // 注意：这个时候我们是看不到偏移的过程的，因为 JS 代码还在执行中，
  // 浏览器不会留有任何的空间去绘制页面。
  firstItem.style.transform = `translateY(${ dis }px)`
  console.log('Invert：', dis);

  raf(() => {
    // 添加过渡动画
    firstItem.style.transition = 'transform 1s'

    // 移除 transform 是为了让元素回到 Last 的位置
    firstItem.style.removeProperty('transform')
    console.log('Play');
  })
}

// 执行动画的函数
function raf(callback) {
  requestAnimationFrame(() => {
    requestAnimationFrame(callback)
  })
}

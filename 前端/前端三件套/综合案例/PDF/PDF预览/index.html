PDF 分片加载预览 / 懒加载预览

前言

在开发过程中，我们应该都会遇到过预览 PDF 文件的需求。

预览 PDF 文件的常规方式一般会有以下：

  • 如果是在 PC 端预览，会有如下方式：
    • 通过 window.open(url) 打开一个浏览器新窗口，url 就是 PDF 文件的地址。
    • 通过 iframe 标签预览，iframe 的 src 属性就是 PDF 文件的下载地址。
    • 使用 pdf.js 插件预览。
  • 如果是在 H5 端预览
    • 一般会通过 iframe 标签进行预览。
    • 或者使用 pdfh5 插件。

我们应该都有遇到过这种情况：当 PDF 比较大的时候，加载速度缓慢以及占用内存过大会是一个比较严重的问题，这将会导致用户体验很差。

我们一般会采取 分片（切片） 的形式来加载 PDF 文件。有两种 分片 方式，分别是：

  • 后端把 PDF 文件的每一页转换成一张图片，前端请求 PDF 文件时，拿到的是多张图片，然后前端展示的时候通过图片懒加载的形式展示。
  • 后端把一个完整大的 PDF 文件按页码切割成多个小的 PDF 文件，前端请求 PDF 文件的时候请求的是每一小块 PDF 文件，再使用常规的预览方式去预览。点击加载更多或者下划再去加载下一个片段。

相关库介绍

这里先介绍一些相关库，案例中不一定有用到。

  • pdfjs-dist：操作 pdf 文件工具包。避免新版本的用法不同，本文安装固定包为@2.7.570。

  • pdf-lid：这个库是基于 pdf.js 和 pdfkit 的、文档健全、操作方便、api 调用简介。可以对 PDF 进行提取、编辑、增删改查。

  • pdf2pic：这个库用于将 PDF 每页转为图片但是不能进行 PDF 的一系列加工但是这并不影响他的强大。

  • pdf.js：这个库是一个处理 PDF 非常强大的库、可以对 PDF 进行一系列操作、也可用于 web 端渲染 PDF 每页。

  • pdfh5：这是是移动端预览 PDF 的插件库。根据官网文档进行使用（在原生 JS、Vue 中的使用）。

  • pdfmake：主要生成 PDF、可以对生成的内容进行精细化的加工 (建议看 2.x 的 文档)。

  • pdfkit：这也是用与生成 PDF 的一个库、个人觉得生成 PDF 还是 pdfmake 方便一点

  • canvas：基于 node 服务中使用 canvas 动态合成图片的使用。

PDF 图片分片

实现流程

图片分片的大体流程是：

  1. 首先从前端上传 PDF 文件到服务器。
  2. 服务器在接收文件后，对 PDF 文件按照每页进行切片为图片。
  3. 前端请求 PDF 文件进行预览时，后端返回原 PDF 文件对应的切片图片列表以及 PDF 原文件。
  4. 前端接收到返回结果后，优先加载切片图直至原 PDF 文件资源加载完成后，显示 PDF 原文件。

本文用到的 素材 PDF 文件，点击可以跳转下载。

在 pdfToImage.js 文件中声明一系列把 PDF 文件分片成图片的相关方法：

const fs = require('fs');
const pdfjsLib = require("pdfjs-dist/build/pdf.js");
const { createCanvas } = require('canvas');
const path = require('path');
const { host, port } = require('../app.js');

class NodeCanvasFactory {
  create(width, height) {
    const canvas = createCanvas(width, height);
    const context = canvas.getContext('2d');
    return {
      canvas,
      context,
    };
  }

  reset(canvasAndContext, width, height) {
    canvasAndContext.canvas.width = width;
    canvasAndContext.canvas.height = height;
  }

  destroy(canvasAndContext) {
    canvasAndContext.canvas.width = 0;
    canvasAndContext.canvas.height = 0;
    canvasAndContext.canvas = null;
    canvasAndContext.context = null;
  }
}

/**
 * 将 PDF 文件转换为图片并保存到指定目录
 * @param {*} pdfPath pdf 文件路径
 * @param {*} outputDir pdf 文件转成图片后的输出目录
 * @returns 图片 url 数组
 */
const convertPdfToImages = async (pdfPath, outputDir) => {
  // 读取 PDF 文件为 Buffer
  const pdfBuffer = fs.readFileSync(pdfPath);

  // 使用 pdfjsLib 库加载 PDF 文档
  const pdfDocument = await pdfjsLib.getDocument({ data: pdfBuffer }).promise;

  // 获取 PDF 文档的页数
  const numPages = pdfDocument.numPages;

  // 用于存储图片 URL 的数组
  const imageUrls = [];

  // 遍历每一页
  for (let pageNum = 1; pageNum <= numPages; pageNum++) {
    console.log('执行中~');

    // 调用 processPage 函数处理当前页，并获取图片 URL
    const imageUrl = await processPage(pdfDocument, pageNum, outputDir);

    // 将图片 URL 添加到 imageUrls 数组中
    imageUrls.push(imageUrl);
  }

  // 返回所有图片 URL 的数组
  return imageUrls;
}

/**
 * 将 PDF 文档的指定页面处理为 PNG 图片并保存到指定目录
 *
 * @param pdfDocument PDF文档对象
 * @param pageNumber 页码，从 1 开始
 * @param outputDir 输出目录路径
 * @returns 返回处理后图片的 URL
 */
const processPage = async (pdfDocument, pageNumber, outputDir) => {
  // 获取指定页码的页面
  const page = await pdfDocument.getPage(pageNumber);

  // 获取页面的视口（渲染区域），缩放比例为 1.8
  const viewport = page.getViewport({ scale: 1.8 });

  // 创建一个 NodeCanvasFactory 实例
  const canvasFactory = new NodeCanvasFactory();

  // 使用 canvasFactory 创建一个指定宽高的 canvas 和 context
  const canvasAndContext = canvasFactory.create(viewport.width, viewport.height);

  // 渲染上下文对象
  const renderContext = {
    // canvas 的 context
    canvasContext: canvasAndContext.context,
    
    // 页面的视口（渲染区域）
    viewport: viewport,

     // canvasFactory 实例
    canvasFactory: canvasFactory,
  };

   // 执行页面渲染任务
  const renderTask = page.render(renderContext);

  // 等待渲染任务完成
  await renderTask.promise;

  // 将 canvas 转换为 Buffer
  const imageBuffer = canvasAndContext.canvas.toBuffer();

  // 构建输出文件的文件名
  const outputFileName = path.join(outputDir, `output_page_${pageNumber}.png`);

  console.log('outputFileName', outputFileName);

  // 将 Buffer 写入文件
  await fs.writeFileSync(outputFileName, imageBuffer);

  // 返回图片的网络地址
  return `http://localhost:7001/pdf_page_${pageNumber}.png`;
}

module.exports = {
  convertPdfToImages,
  processPage,
}

/**
 * 加载图片
 * @param {*} images 图片地址数组
 */
function loadImages(images) {
  let index = 0;

  function loadNextImage() {
    if (index < images.length) {
      const img = new Image();
      img.src = images[index];

      // 当前图片加载完成后，继续加载下一张图片
      img.onload = function () {
        index++;
        loadNextImage(); // 加载下一张图片
      };

      // 当前图片加载失败，跳过该图片，继续加载下一个图片
      img.onerror = function (error) {
        console.error(`第 ${index} 张图片加载失败：`, img.src, error);
        index++;
        loadNextImage(); // 加载下一张图片
      };

      // 图片加载完成，添加到页面中
      imageList.appendChild(img);
    }
  }
  
  // 开始加载第一张图片
  loadNextImage();
}

PDF 文件分片

上面介绍的是把 PDF 转成图片之后，利用加载多图的方式来做 PDF 的展示。下面这种方式是，在前端上传文件时，把一个大的 PDF 切分成若干个小的 PDF，然后在前端请求预览时，再使用常规的预览方式去预览。

后端实现

新建 utils/splitPdf.js 文件，编写对应 PDF 文件分片逻辑：

// utils/splitPdf.js

const fs = require("fs");
const { PDFDocument } = require("pdf-lib");
const path = require("path");

/**
 * 切割 pdf 文件
 * @param inputPath 输入文件路径
 * @param outPath 切片输出路径
 */
const splitPdf = async (inputPath, outPath) => {
   // 读取 PDF 文件内容
  const pdfBytes = await fs.promises.readFile(inputPath);

  // 加载 PDF 文档
  const pdfDoc = await PDFDocument.load(pdfBytes);

   // 获取 PDF 文档的总页数
  const totalPages = pdfDoc.getPageCount();

  // 每个分块包含的页数
  const pagesPerChunk = 10;

  // 遍历所有页面，按每分块的页数进行分割
  for (let startPage = 1; startPage <= totalPages; startPage += pagesPerChunk) {
    // 计算当前块的结束页数
    // 如果当前块的结束页数大于总页数，则设置为总页数
    const endPage = Math.min(startPage + pagesPerChunk - 1, totalPages);

    // 创建一个新的 PDF 文档
    const newPdfDoc = await PDFDocument.create();

    // 复制当前分块的页面到新文档中
    for (let pageNum = startPage; pageNum <= endPage; pageNum++) {
      // 复制对应分页的页面
      const [copiedPage] = await newPdfDoc.copyPages(pdfDoc, [pageNum - 1]);

      // 将复制的页面添加到新文档中
      newPdfDoc.addPage(copiedPage);
    }

    // 设置输出文件路径
    const outputPath = `${outPath}/${Math.floor(endPage / 10)}.pdf`;

    // 保存新文档为字节流
    const newPdfBytes = await newPdfDoc.save();

    // 写入文件
    await fs.promises.writeFile(outputPath, newPdfBytes);

    console.log(`PDF 从 page ${startPage} to ${endPage} 切片成功`);
  }
}

module.exports = {
  splitPdf
};

前端通过 pdf.js 优化预览

之前我们通过按钮点击来加载下一个分片的内容，其实这是很不合理的，我们应该在获取 pdf 文件地址时，初始化时默认展示第一分片的内容，当滚动条滚动到某个位置时加载下一分片的内容并追加到当前分片的后面。这里可以通过 pdf.js 来实现，这里后端的逻辑不变。

const btn1 = document.querySelector('.fetch-btn')
const btn2 = document.querySelector('.load-more')
const input = document.querySelector('input')
const pdfContainer = document.querySelector('#pdfContainer')
let i = 0 // pdf 当前展示的切片列表的索引
let total = 0 // 总切片数量
let pdfUrl = [] // pdf 切片文件地址
let loading = false // 是否正在加载更多

// 文件上传事件
input.onchange = async (e) => {
  const file = e.target.files[0]
  const formData = new FormData()
  formData.append('file', file)
  formData.append('name', file.name)
  formData.append('type', file.type)

  const res = await axios({
    url: 'http://192.168.100.12:7001/api/upload',
    method: 'post',
    data: formData,
  })
}

// 请求 pdf 文件并默认展示第一分块的内容
btn1.onclick = async () => {
  pdfUrl = await fetchPdf()
  total = pdfUrl.length
  const url = pdfUrl[i]
  loadPdf(url)
}

const fetchPdf = async () => {
  const res = await axios({
    method: 'get',
    url: 'http://192.168.100.12:7001/api/getPdf'
  })

  const { pdfs } = res.data.data

  return pdfs
}

const loadPdf = async (url) => {
  // pdfjsLib 是引入了 pdf.js 库后的全局变量
  const loadingTask = pdfjsLib.getDocument(url);  // 创建一个 PDF 加载任务

  try {
    const pdf = await loadingTask.promise // 加载 PDF 文档
    const numPages = pdf.numPages // 获取 pdf 分片文件的总页数

    for (let i = 1; i <= numPages; i++) {
      await loadPdfPageToCanvas(pdf, i) // 获取页面
      loading = false // 将 loading 设置为 false，表示加载完成
    }
  } catch (error) {
    // PDF 加载错误
    console.error(error);
  }
}

/**
 * 加载 pdf 指定页码的内容到 canvas 中
 * @param { Obejct } pdf pdf 分片文件对象
 * @param { Number } pageIndex 页码
 */
const loadPdfPageToCanvas = (pdf, pageIndex) => {
  return new Promise(async (resolve, reject) => {
    try {
      // 获取某个页码页面的内容
      const page = await pdf.getPage(pageIndex)

      // 准备 canvas 元素
      var canvas = document.createElement('canvas');
      var context = canvas.getContext('2d');

      // 获取页面大小（这里假设我们想要以原始大小渲染页面）
      var viewport = page.getViewport({ scale: 1.0 });

      // 设置 canvas 的尺寸
      canvas.height = viewport.height;
      canvas.width = viewport.width;

      // 将 canvas 添加到容器中
      const container = document.getElementById('pdfContainer');
      container.appendChild(canvas);

      // 渲染 PDF 页面到 canvas 上下文中
      var renderContext = {
        canvasContext: context,
        viewport: viewport
      };
      page.render(renderContext);
      resolve()
    } catch (error) {
      // 页面加载错误
      console.error('Error loading page: ' + error);
    }
  })
}

/**
 * 监听滚动事件
 */
pdfContainer.addEventListener('scroll', async function(e) {
  const scrollTop = this.scrollTop // 滚动条距离顶部的高度
  const scrollHeight = this.scrollHeight // 元素的总高度
  const clientHeight = this.clientHeight // 可视区域的高度
  const scrollBottom = scrollHeight - scrollTop - clientHeight // 滚动条距离底部的距离

  // 如果滚动条距离底部的距离小于或等于阈值，并且没有在加载中，则加载下一页
  if (scrollBottom < 200 && !loading) {
    console.log('加载更多~');

    // 设置为 true，表示正在加载 pdf，防止多个 pdf 同时加载，造成 pdf 文件页面加载错乱
    loading = true
    i++
    const url = pdfUrl[i]
    await loadPdf(url)
  }
})

PDF 文件流预览（最优方案）

一般在实际开发中，很少会通过后端来进行分片，更多的是通过后端返回一个 pdf 的文件流，然后前端通过这个文件流来进行分片地读取预览。

这种方式的好处是，前端上传文件时，后端不再需要对文件进行切片，后端返回前端需要的 pdf 文件时，也不需要找对应的切片文件返回了，而是直接返回一个文件流。

后端实现

在 app.js 中：

// 引入 Express框架
const express = require("express");
const cors = require("cors");
const path = require("path");
const fs = require("fs");

const app = express();
const host = '192.168.100.12'
const port = 7001;

app.use(cors()) // 允许跨域

// 设置静态文件目录为 upload 目录
const staticPath = path.resolve(__dirname, './upload')
app.use(express.static(staticPath));

// 获取 pdf 文件
app.get('/api/getPdf', (req, res) => {
  // 找到对应的 pdf 文件的路径，这里写死了，具体实际项目中应该根据用户请求动态获取对应的文件
  const filePath = path.join(__dirname, 'upload/NVIDIA-2024-Annual-Report.pdf')

  // 设置响应头，告知浏览器这是一个文件下载
  const head = {
    'Content-Disposition': 'attachment; filename=NVIDIA-2024-Annual-Report.pdf',
    "Content-Type": "application/pdf",
  };

  // 设置 HTTP 状态码为 200（成功）
  res.writeHead(200, head);

  // 创建一个可读流来读取文件
  const fileStream = fs.createReadStream(filePath);

  // 将文件流发送给客户端
  fileStream.pipe(res);

  // 监听结束事件（可选）
  fileStream.on('end', () => {
    console.log('File has been downloaded successfully.');
  });

  // 监听错误事件
  fileStream.on('error', err => {
    console.error('Error while streaming the file:', err);

    // 如果出现错误，向客户端返回 500 状态码和错误信息
    res.status(500).send('Internal Server Error');
  });
})

// 启动服务器，监听指定的端口
app.listen(port, () => {
  console.log(`Server is running on port ${port}`);
});

module.exports = {
  host,
  port,
}

注意：参数必须要 encodeURIComponent 编码。

Web Worker 优化导出大量数据

前言

在前端列表数据导出为 excel 进行下载，列表数据可到百万以上，因此在主线程内对大量数据进行 excel 导出时不可避免的会对主线程进行阻塞，而主线程阻塞对于用户感知来说就是 页面卡顿。

显然，这种业务是一种典型的 CPU 密集型任务，如果可以放在主线程以外的线程进行执行那么就可以有效的避免对主线程的阻塞了；Worker 正是为此而生，通过创建 worker 来新建线程，就可以有效的分担主线程的压力。

Worker

至于 worker 的使用，可以参考 使用 Web Workers - Web API 接口参考 | MDN，但是，worker 虽好，但是其运行上下文跟主线程还是不一样的，实际上是一个子集语法，因此 DOM 对象和很大一部分的 BOM 对象都不能访问和使用；具体限制可以参考 Web Workers 可以使用的函数和类 - Web API 接口参考 | MDN。

Worker 的通信

主线程和 Worker 实例之间都是通过 postMessage 方法来向对方发送消息，而又通过 onmessage 方法来注册各自的消息接收处理事件。

Worker 中实现 excel 的导出

前端有个著名的 excel 导出库——sheetjs，幸运的是这个库的 大部分 方法都能在 worker 中直接进行使用，因此基本上可以参照正常的导出逻辑搬到 worker 文件中即可。

假设此时可以获取到对表格本身的单元格二维数组（即以行为主导，每行里面的元素就是列），那么根据二维数组就能生成一个 WorkBook 对象。

造成 主线程堵塞 的 原因 就是 WorkerBook 本质上是一个包含大量数据的对象结构。

而有了 WorkBook 对象，正常来说基于 XLSX.writeFile() 等方法就能直接进行 excel 下载操作了；一般现在的文件下载操作都是通过 a 标签来实现的，而 worker 本身是限制 DOM 访问的，因此不能通过上述方法在 worker 内实现下载操作，所以应该 将下载操作放在主线程，并且将必要的数据传送到主线程。

那么问题来了，应该选择 传送什么样的数据到主线程？

思路一：传送 WorkBook 对象

从直觉上来说直接把 WorkBook 对象传送到主线程，然后让主线程继续接着 XLSX.writeFile() 方法的路子不就好了。但事实证明，这种思路有 两个弊端：

  1. 线程间的通信成本：WorkBook 对象 本质上是一个包含大量数据的对象结构（包含单元格数据和单元格设置等），而线程间的通信成本会随着数据量加大而陡增（结构化克隆，序列化，反序列化等等）；当数据量较大时，主线程在接收来自 Worker 的消息依然会造成明显的阻塞。
  2. 从 WorkBook 对象到下载的成本：由于 WorkBook 对象本身不能直接进行下载，因此XLSX.writeFile() 这类方法内部都需要先将 WorkBook 对象转为 Blob/File 对象，然后通过URL.createObjectURL() 来创建一个可访问的 Blob URL；但实际上将 WorkBook 对象转为Blob/File 对象也是一个挺耗时的过程。

综上，如果采用该方案，那么 worker 的使用效果就很尴尬了。

思路二：传送 Blob URL

幸运的是，worker 内可以使用 URL.createObjectURL() 方法，且创建的 Blob URL 可以被主线程访问：

到此，剩下的问题就只有一个了，那就是如何将 WorkBook 对象转为 Blob 对象；事实上，sheetjs 本身就提供了将 WorkBook 对象转为二进制数据（Uint8Array）的 API XLSX.write()，所以这事就很简单了。

至此，只需要将 Blob URL 传送到主线程即可完成最后的下载步骤；显然，传送一个 URL 字符串的通信成本几乎为 0，这样就可以把 excel 导出下载任务的绝大多数运算控制在 worker 内部了，最小可能的对主线程进行占用。

export.js：

/**
 * 导出 excel
 * @param { object[] } data 需要导出的数据
 * @param { String } sheetName 工作表的名称
 * @param { String } exportName 导出的 excel 文件名称
 */
function exportToExcel(sheetName, exportName, data, headers) {
  const workbook = XLSX.utils.book_new() // 创建一个工作簿
  const worksheet = XLSX.utils.json_to_sheet(data) // 创建一个工作表
  console.dir(worksheet);
  // 将工作表添加到工作簿中， 新工作表将被称为传入的 "sheetName"
  // 默认情况下，json_to_sheet 创建一个带有标题行的工作表，标头来自 JS 对象键。
  XLSX.utils.book_append_sheet(workbook, worksheet, sheetName)

  // 如果有传入的自定义的表头名称
  if (headers && headers.length > 0) {
    // XLSX.utils.sheet_add_aoa 可以将文本值写入从单元格 A1 开始的现有工作表
    XLSX.utils.sheet_add_aoa(worksheet, [headers], { origin: "A1" });
  }

  // writeFile 尝试将 wb 写入指定 filename 的本地文件。
  // 在浏览器中，它会尝试提示用户下载文件。 
  // 在 NodeJS 中，它将写入本地目录。
  // 第三个参数是 写作选项，具体参数需参考官网：https://xlsx.nodejs.cn/docs/api/write-options/
  XLSX.writeFile(workbook, `${exportName}.xlsx`, { compression: true })
}

import.js：

/**
 * 导入 excel 数据
 * @param { ArrayBuffer } file 要读取的文件 buffer 缓冲区
 * @returns 生成的 HTML 字符串
 */
function importExcel(file) {
  // 解析文件
  // read 尝试解析 file 并返回 工作簿对象
  // 第二个参数是 options 对象，options 的 type 决定了 data 的解释方式。 对于字符串数据，默认解释是 Base64。
  const workbook = XLSX.read(file)

  // 检查工作表数据
  // Sheets 是一个对象，其键是工作表名称，其值是工作表对象。
  // SheetNames 属性是工作簿中工作表名称的有序列表。 
  // 例如，通过索引 SheetNames 并使用名称索引 Sheets 来拉取第一个工作表：
  const firstSheet = workbook.Sheets[workbook.SheetNames[0]]

  // sheet_to_html 从工作表对象生成 HTML 表
  const table = XLSX.utils.sheet_to_html(firstSheet)
  
  return table
}

index.js：

const exportUrl = 'https://sheetjs.com/data/executive.json' // 导出数据的 url
const importUrl = 'https://sheetjs.com/data/PortfolioSummary.xls' // 导入数据的 url
let exportData = []

// 使用 web worker 创建子线程优化导出大量的列表数据的情况
// 主线程下创建 worker 线程
const myWorker = new Worker("./worker.js");

// 获取要导出的数据
async function getData() {
  let res = await (await fetch(exportUrl)).json()
  // 向 worker 线程发送消息
  myWorker.postMessage({ action: 'request', data: res })
}
getData()

// 监听接收 worker 线程发的消息
myWorker.onmessage = function (e) {
  console.log('主线程收到 worker 线程消息：', e.data)
  const { action, data } = e.data
  if (action === 'requestEnd') {
    exportData = data
  }
  if (action === 'exportEnd') {
    download(data, 'abc.xlsx')
  }
}

function download(url, name) {
    const tmpLink = document.createElement("a");

    tmpLink.href = url;
    tmpLink.download = name;
    document.body.appendChild(tmpLink);
    tmpLink.click();

    document.body.removeChild(tmpLink);
    URL.revokeObjectURL(url);
}

const exportBtn = document.querySelector('.export-btn')
const ipmortBtn = document.querySelector('.import-btn')

// 导出按钮点击
exportBtn.onclick = function() {
  const start = Date.now()
  
  // 像子线程发起导出的消息
  myWorker.postMessage({ action: 'export', data: exportData })
  
  const end = Date.now()
  console.log(`导出结束，耗时 ${end - start} ms`);
}

// 导入按钮点击
ipmortBtn.onclick = async function() {
  const file = await (await fetch(importUrl)).arrayBuffer()
  const tableContent = importExcel(file)
  const table = document.querySelector('.table')
  table.innerHTML = tableContent
}

worker.js：

importScripts('../lib/xlsx-0.20.1/package/dist/xlsx.full.min.js')
importScripts('../scripts/write.js')

addEventListener('message', function (e) {
  console.log('子线程收到消息：', e.data);
  const { action, data } = e.data

  // 其实这里的模拟大量数据的情况可以不放在 web worker 中，
  // 在主线程中使用时间分片的形式也可以进行优化。
  if (action === 'request') {
    let exportData = []
    // 模拟大量数据
    for (let i = 0; i < 2000; i++) {
      exportData = [...exportData, ...data]
    }
    exportData = exportData.map(item => {
      return {
        id: item.id.bioguide + item.id.govtrack + item.id.icpsr_prez,
        name: item.name.first + item.name.last,
        gender: item.bio.gender,
        birthday: item.bio.birthday
      }
    })

    postMessage({ action: 'requestEnd', data: exportData })
  }

  // 处理导出，生成暂存在内存中的文件
  if (action === 'export') {
    // 调用写入 excel 到内存中的方法，并拿到返回的 blob url
    const blobUrl = writeToExcel('sheet1', 'excel', data, ['ID', '姓名', '性别', '出生年月'])
    postMessage({ action: 'exportEnd', data: blobUrl })
  }
}, false);

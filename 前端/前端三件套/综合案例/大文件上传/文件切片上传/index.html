文件分片

当我们读取完文件后，就可以执行文件分片的操作了，文件分片的核心是用 Blob 对象的 slice 方法。我们在上一步中获取的文件是一个 File 对象，File 对象是继承于 Blob 的，所以我们就可以使用 slice 方法来进行分片。

用法如下：

// 假设 file 是一个文件对象，则可以使用 slice 分片
const chunk = file.slice([start [, end [, contentType]]])

start 和 end 代表的是 Blob 的下标，表示被拷贝进新的 Blob 的 字节 的 起始位置 和 结束位置。因为文件的大小一般都是用 字节 来区分的。

contentType 会给新的 Blob 赋予一个新的文档类型，在这里我们用不到。更细节的详情使用可以查看 文档。

注意：要想进行分片，还有一个前提条件，就是要知道每个分片的大小。

我们先来了解一下平时用来区分文件大小的单位：

1MB = 1024kB = 1024 * 1024B

我们分片的时候一般对每个分片的大小都是按照 MB 的单位来划分的，我这里定义每个分片的大小为 10MB，也就是 1024 * 1024 * 10B。

因为我们获取 File 对象时，里面的 size 属性的单位是 B，所以我们定义的大小也要用 B，这样比较好对比。

接下来就让我们在代码中进行文件分片的实现，修改 index.js：

const input = document.querySelector('input');

// 定义每个分片大小，这里定义 10MB
const CHUNK_SIZE = 1024 * 1024 * 10;

/**
 * 文件分片方法
 * @param { File } file 文件对象
 * @returns { Array } 分片数组
 */
const createchunks = file => {
  let curIndex = 0 // 当前分片的字节的开始位置
  const chunks = [] // 存储所有分片

  // 当分片还没有分完的时候，继续分片
  while(curIndex < file.size) {
    // curIndex 当前分片的字节的开始位置，curIndex + CHUNK_SIZE 字节的结束位置
    const chunk = file.slice(curIndex, curIndex + CHUNK_SIZE)

    // 添加到分片数组中
    chunks.push(chunk)

    // 每次分片后，curIndex 索引位置增加 CHUNK_SIZE
    curIndex += CHUNK_SIZE
  }

  return chunks
}


input.onchange = e => {
  const files = e.target.files

  if (!files) return
  const file = files[0]

  // 对进行进行分片
  const chunks = createchunks(file)
  console.log(chunks);
}

hash 计算

在拿到了所有切片之后，是不是就要马上开始上传呢？并不是的。

先来思考一个问题，在向服务器上传文件时，怎么去区分不同的文件呢？如果根据文件名去区分可以吗？

答案是：不可以。因为文件名我们可以随便修改的，所以不能根据文件名去区分。但是 每一份文件的内容都不一样，我们可以根据文件内容去区分，具体怎么做呢？

可以根据文件内容产生一个唯一的 hash 值，大家应该都见过用 webpack 打包出来的文件名都有一串不一样的字符串，这个字符串就是根据文件内容生成的 hash 值，文件内容变化，hash 值就会跟着变化。

我们在这里也可以用这个办法来区分不同的文件，而且通过这个办法，我们还可以实现 秒传 的功能，怎么做呢？

就是服务器在处理上传文件的请求时，要先判断一下对应的文件的 hash 值有没有记录，如果 A 和 B 两个用户先后上传了一份内容相同的文件，所以这两份文件的 hash 值是一样的。

当 A 上传的时候会根据文件内容生成一个对应的 hash 值，然后在服务器上就会有一个对应的文件 。B 再上传的时候，服务器就会发现这个文件的 hash 值之前已经有记录了，说明之前已经有人上传过相同内容的文件了，所以就不需要处理 B 的这个上传请求了，直接返回一个成功的结果，给用户的感觉就像是实现了 秒传。

那么怎么计算文件的 hash 值呢？可以通过一个工具 spark-md5。

在上一步获取到了文件的所有切片，我们就可以用这些切片来计算该文件的 hash 值，但是如果一个文件特别大，每个切片的所有内容都参与计算的话会很耗时，所以我们可以采取以下策略：

  1. 第一个和最后一个切片的内容全部参与计算。
  2. 中间剩余的所有切片，我们分别在 每个切片 的前面、后面和中间各取前 2 个字节参与计算。

这样就即能保证所有的切片参与了计算，也能保证不耗费很长的时间。

其实除了以上的策略还可以使用 hash 的增量算法，具体可以查看该 教程。

然后在 index.js 中添加 hash 计算的处理：

const input = document.querySelector('input');

// 定义每个分片大小，这里定义 10MB
const CHUNK_SIZE = 1024 * 1024 * 10;

/**
 * 文件分片方法
 * @param { File } file 文件对象
 * @returns { Array } 分片数组
 */
const createchunks = file => {
  let curIndex = 0 // 当前分片的字节的开始位置
  const chunks = [] // 存储所有分片

  // 当分片还没有分完的时候，继续分片
  while (curIndex < file.size) {
    // curIndex 当前分片的字节的开始位置，curIndex + CHUNK_SIZE 字节的结束位置
    const chunk = file.slice(curIndex, curIndex + CHUNK_SIZE)

    // 添加到分片数组中
    chunks.push(chunk)

    // 每次分片后，curIndex 索引位置增加 CHUNK_SIZE
    curIndex += CHUNK_SIZE
  }

  return chunks
}

/**
 * 通过分片数组，计算完整的文件的 hash 值
 * 计算的时候并不是根据所有的分片计算，那样对于大文件来说很耗时，我们采用以下策略去计算：
 * 1. 第一个和最后一个切片的内容全部参与计算。
 * 2. 中间剩余的所有切片，我们分别每个切片在前面、后面和中间各取前 2 个字节参与计算。
 * 这样会节省计算 hash 值的时间。
 * 
 * @param { Array } chunks 分片数组
 */
const calcHash = (chunks) => {
  return new Promise((resolve, reject) => {
    const targets = [] // 存储所有参与计算的字节
	
    // 创建一个 SparkMD5 的实例对象
    const spark = new SparkMD5.ArrayBuffer()
    
    // 实例化一个文件读取对象，用于读取文件内容
    const fileReader = new FileReader()

    // 开始计算 hash 值
    chunks.forEach((chunk, i) => {
      // 1. 第一个和最后一个切片的内容全部参与计算
      if (i === 0 || i === chunks.length - 1) {
        targets.push(chunk)
      }
      // 2. 中间剩余的每个切片我们分别在其前面、后面和中间各取前 2 个字节参与计算
      else {
        // 前面取两个字节
        targets.push(chunk.slice(0, 2))
        // 后面取两个字节
        targets.push(chunk.slice(-2))
        // 中间取两个字节
        targets.push(chunk.slice(CHUNK_SIZE / 2, CHUNK_SIZE / 2 + 2))
      }
    })

    // new Blob(targets) 把所有参与计算的分片再转换成 Blob 对象
    fileReader.readAsArrayBuffer(new Blob(targets));

    // 等待 fileReader 读取完成，这是一个异步操作
    fileReader.onload = (e) => {
      // 把所有参与计算的分片都放到 spark 对象中
      spark.append(e.target.result);

      // 计算 hash 值
      const hash = spark.end()

      // 返回 hash 值
      resolve(hash)
    }
  })
}

input.onchange = async e => {
  const files = e.target.files

  if (!files) return
  const file = files[0]

  // 对进行进行分片
  const chunks = createchunks(file)

  // 计算文件的 hash 值
  const hash = await calcHash(chunks)

  console.log(hash);
}

文件上传

前面已经完成了上传的前置操作，接下来就看如何去上传这些切片。

我们以 1G 的文件来分析，假如每个分片的大小为 1M，那么总的分片将会是 1024 个，如果我们同时发送这 1024 个分片，浏览器肯定处理不了，原因是切片数量过多，浏览器一次性创建了太多的请求。

这是没有必要的，拿 chrome 浏览器来说，默认的并发数量只有 6，过多的请求并不会提升上传速度，反而是给浏览器带来了巨大的负担。因此，我们有必要限制前端请求个数。

怎么做呢？我们要创建 最大并发数量的请求，比如 6 个，那么同一时间我们就只允许浏览器发送 6 个请求，其中一个请求有了返回的结果后我们再发起一个新的请求，以此类推，直到所有的请求发送完毕。

上传文件时，一般还要用到 FormData 对象，需要将我们传递的文件，还有额外的信息放到这个 FormData 对象里面。

我们先在 index.html 中引入 axios 的在线包：

const input = document.querySelector('input');

// 定义每个分片大小，这里定义 10MB
const CHUNK_SIZE = 1024 * 1024 * 10;

/**
 * 文件分片方法
 * @param { File } file 文件对象
 * @returns { Array } 分片数组
 */
const createchunks = file => {

  const chunks = [] // 存储所有分片

    chunks.push(chunk)

/**

 * 这样会节省计算 hash 值的时间。
 * 
 * @param { Array } chunks 分片数组
 */
const calcHash = (chunks) => {

    // 创建一个 SparkMD5 的实例对象
    const spark = new SparkMD5.ArrayBuffer()

/**
 * 文件分片上传
 * @param { Array } chunks 分片数组
 * @param { String } hash 文件的 hash 值
 * @param { String } fileName 文件名
 */
const uploadchunks = async (chunks, hash, fileName) => {
  // 构建我们需要上传的数据
  const data = chunks.map((chunk, i) => {
    return {
      hash, // 完整文件的 hash 值
      fileName, // 完整文件的文件名
      chunkHash: `${ hash }-${ i }`, // 分片的 hash 值
      chunk // 分片的内容
    }
  })

  // 构建 FormData 列表
  const formDatas = data.map(item => {
    const formData = new FormData() // 创建一个 FormData 对象
    formData.append('hash', item.hash)
    formData.append('fileName', item.fileName)
    formData.append('chunkHash', item.chunkHash)
    formData.append('chunk', item.chunk)
    return formData
  })

  const max = 6 // 最大同时上传的请求并发数量

  // 构建每个分片的请求配置
  const requestOptions = formDatas.map((formData) => {
    return {
      url: 'http://localhost:3000/api/upload',
      method: 'POST',
      data: formData
    }
  })

  // 并发请求
  concurRequest(requestOptions, max)
}

/**
 * 并发请求方法
 * @param { Array } requestOptions 每个分片对应的请求配置
 * @param { Number } maxNum 最大并发数量
 */
function concurRequest(requestOptions, maxNum) {
  return new Promise((resolve) => {
    if (requestOptions.length === 0) resolve([])

    let nextIndex = 0 // 下一次请求对应的 url 地址下标
    let count = 0 // 请求完成的数量
    const result = [] // 结果数组

    /**
     * 请求方法
     * 根据 requestOptions 下标指向的配置，取出对应请求配置并发出请求
     */
    async function request() {
      const index = nextIndex // 当前请求的索引
      const { url, method, data } = requestOptions[index]
      nextIndex++

      try {
        await axios(url, { method, data })

        // 保存请求结果到对应的索引位置
        result[index] = res
      }
      // 如果请求失败了，把失败的信息放到对应的索引位置
      catch (error) {
        result[index] = error
      }
      // 某个请求完成后，还得继续调用 request，继续请求下一个 url
      finally { 
        
        count++ // 请求完成数量++

        // 请求完成数量和 requestOptions 相等，说明已经全部请求完成
        if (count === requestOptions.length) {
          resolve(result)
        }

        // 防止索引超出范围
        if (nextIndex < requestOptions.length) request()
      }
    }

    // 根据并发数量发起对应数量的请求
    // Math.min(requestOptions.length, maxNum) 取二者的最小值，
    // 因为有可能传递进来的 最大并发数量会 大于 requestOptions 的长度。
    for (let i = 0; i < Math.min(requestOptions.length, maxNum); i++) {
      request()
    }
  })
}

input.onchange = async e => {
  const files = e.target.files

  if (!files) return
  const file = files[0]

  // 对进行进行分片
  const chunks = createchunks(file)

  // 计算文件的 hash 值
  const hash = await calcHash(chunks)

  uploadchunks(chunks, hash, file.name)
}

上传接口

在前面的内容中，已经实现了前端的读取文件、文件分片、hash 计算以及文件上传的功能，接下来就需要实现后端的上传接口逻辑。

我们之前在创建后端项目目录结构的时候，已经创建一个了 upload 目录来存放上传的文件。

我们在处理每个上传的分片的时候，应该先将它们临时存放到服务器的 upload 目录中，方便我们合并的时候再去读取。

为了区分不同文件的分片，我们应该在 upload 目录下再创建一个临时目录，用来存放文件的所有切片。我们就用完整文件对应的 hash 值来作为这个临时目录的目录名称。

当我们的切片文件全部上传完毕后，合成完整的文件并存放到 upload 目录中，并删除存放切片的临时目录。

在 app.js 中，添加上传切片接口路由以及对应的处理逻辑：

const express = require('express')
const path = require('path')
const multipary = require('multiparty')
const fse = require('fs-extra')
const cors = require('cors')
const bodyParser = require('body-parser')

const app = express()
app.use(bodyParser.json())
app.use(cors())

// 声明存放上传文件的目录
const UPLOAD_DIR = path.join(__dirname, 'upload')

// 切片上传接口
app.post('/api/upload', (req, res) => {
  const form = new multipary.Form()
  try {
    form.parse(req, async (err, fields, files) => {

      // 如果有错误
      if (err) return res.send({ code: -9999, msg: '上传失败' })

      const hash = fields.hash[0] // 文件 hash 值
      const chunkHash = fields.chunkHash[0] // 切片 hash 值

      // 存放切片列表的临时目录
      const chunkDir = path.resolve(UPLOAD_DIR, hash)

      // 如果存放切片的临时目录不存在，则创建
      if (!fse.existsSync(chunkDir)) {
        await fse.mkdir(chunkDir)
      }

      const oldPath = files.chunk[0]['path']

      // 将切片放到这个文件夹里面
      // 目标路径为 upload/chucnkDir/chunkHash
      fse.move(oldPath, path.resolve(chunkDir, chunkHash))

      res.send({ code: 200, msg: 'ok' })
    })
  } catch (error) {
    console.log('上传错误：', error);
  }
})

app.listen(3000, () => {
  console.log('server is running on port 3000');
})

合并文件

上一步中我们已经实现了将所有切片上传到服务器了，上传完成之后，我们就可以将所有的切片合并成一个完整的文件了。

合并完成后，我们将生成的文件以 hash 值命名存放到对应的位置就可以了：

const UPLOAD_DIR = path.join(__dirname, 'upload')

  return fileName.slice(fileName.lastIndexOf('.'), fileName.length)

// 合并切片接口
app.post('/api/merge', async (req, res) => {
  const { hash, fileName, chunkSize } = req.body

  // 文件保存的路径
  const filePath = path.resolve(UPLOAD_DIR, hash + extractExt(fileName))
  
  // 如果已经存在该文件，就没必要合并
  if (fse.existsSync(filePath)) {
    res.send({ code: 200, msg: '合并成功' })
    return
  }

  // 如果不存在，执行以下逻辑
  // 获取存放切片的临时目录
  const chunkDir = path.resolve(UPLOAD_DIR, hash)

  // 如果保存切片的临时目录不存在
  if (!fse.existsSync(chunkDir)) {
    res.send({ code: -9999, msg: '切片目录不存在，合并错误！请重新上传' })
    return
  }

  // 开始合并切片
  // 读取目录下的所有文件
  const chunkNames = await fse.readdir(chunkDir)

  // 切片的合并是需要按顺序的，所以需要排序
  chunkNames.sort((a, b) => {
    return a.split('-')[1] - b.split('-')[1]
  })

  try {
    // 合并切片
    const res = await merge(filePath, chunkDir, chunkNames, chunkSize)
  } catch (error) {
    console.log('合并错误：', error);
  }
})

app.listen(3000, () => {

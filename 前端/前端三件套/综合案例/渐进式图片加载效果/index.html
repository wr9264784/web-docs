渐进式图片

渐进式图片就是一开始图片是模糊的，然后逐渐变得清晰。这不仅仅是一个动画效果，它是有实际作用的。有的时候，页面上会有一些大图，如果直接把大图渲染出来的话，它的传输时间比较长，用户会长时间陷入白屏。

那么，这个时候我们可以：

  1. 先给它展示一个小图。因为小图的尺寸小，所以说它能快速的显示出来。
  2. 然后再逐步地给它传输大图。大图传输完成之后再显示大图。

效果如下图所示：

可以看到，图片一开始是模糊的，然后逐渐变清晰，这就是渐进式图片。

那么这个渐进式图片具体怎么实现呢？有两种方案：

  1. 直接让设计师提供的图片就是 渐进式图片（jpg 格式的图片）就行了。这一块的原理简单了解一下就行。

    • 原理：jpg 图片实际上里边是可以支持多帧的，那么它就可以实现渐进式了，只要设计师在第一帧的时候放一个模糊的小图，然后再第二帧的时候放一个高清的大图。
    • 这样之后，它在传输的时候首先收到的是一个小帧，这个很快就读取完成了，读完之后浏览器会自动处理，把它先渲染出来，那么等了一会之后再传输大帧的图的时候，再一步一步地替换小的帧，丰富它里边模糊的细节。
    • 并且，神奇的是 jpg 图片里添加了两个帧，但是它的大小却没有明显的变大。
    • 这种方案会有兼容性问题，一些老版本的浏览器并不兼容。还有可能就是设计师可能也不懂怎么设置 jpg 图片的每帧的内容。

  2. 通过 js 代码和 css 样式实现，首先提供两个 img 标签，分别传入模糊小图的 url，高清大图的 url，只不过在最开始的时候这个大图是隐藏的，然后当大图加载完成后再把它显示出来，显示的时候可以给个动画（模糊效果 blur）。

    • html 结构：
    <!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>渐进式图片</title>
  <link rel="stylesheet" href="./index.css">
</head>

<body>
  <div>
    <!-- 缩略图 -->
    <img class="compress" src="./assets/compress.jpg" alt="">
  
    <!-- 原图 -->
    <img class="origin" src="./assets/origin.jpg" alt="">
  </div>
  <!-- 压缩的图片 -->

  <script src="./index.js"></script>
</body>

</html>
    • css 样式：
    * {
  margin: 0;
  padding: 0;
}

img {
  display: block;
  width: 100vw;
  height: 100vh;
}

.origin {
  opacity: 0;
  position: absolute;
  left: 0;
  top: 0;
  filter: blur(10px);
}

.loaded .origin {
  opacity: 1;
  filter: blur(0);
}
    • js 代码：
    const originImg = document.querySelector('.origin')

originImg.onload = function(e) {
  e.target.parentElement.classList.add('loaded')
}

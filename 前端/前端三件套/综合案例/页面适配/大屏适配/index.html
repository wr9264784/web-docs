大屏适配

大屏适配的核心：

  1. 设计尺寸建议按照输出分辨率设计。
  2. 拼接后像素在 4K 左右直接按照总和设计就行。
  3. 特殊尺寸不要考虑适配电脑屏幕又适配拼接屏，因为完全没有必要，也不可能一稿即适配电脑也是配各种尺寸的大屏。
  4. 优先考虑目标屏幕的适配，要针对性设计，而在小屏幕根据等比缩放显示。

常见的大屏适配方案：

  1. 百分比设置。
  2. rem 单位 + 动态设置 html 的 font-size。
  3. vw 单位。
  4. flex 弹性布局。
  5. scale 等比缩放（推荐）。

方案一、rem + js 做适配

和移动端中的 rem + js 做适配是一样的。只不过在 PC 端中，划分屏幕等份时，一般都是 划分为 24 等份。因为 PC 端的设计稿一般都是 1920 * 1080，所以 1920 / 24 = 80，也就是 1rem = 80px。

这样，和 移动端 中的 750px 划分为 10 等份 即 1rem = 75px 相差不大，也方便计算。

其实，这个划分多少等份完全可以看个人，怎么方便计算怎么来。

方案二、vw 适配

和移动端中的 vw 做适配是一样的。

方案三、scale 等比缩放适配

我们先来写一个页面结构，代码如下所示：

html 结构：

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>scale适配大屏</title>
  <link rel="stylesheet" href="./index.css">
</head>

<body>
  <ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
    <li>4</li>
    <li>5</li>
    <li>6</li>
  </ul>
</body>

</html>

css 样式：

* {
  margin: 0;
  padding: 0;
}

body {
  width: 1920px;
  height: 1080px;
  box-sizing: border-box;
  border: 3px solid red;
}

ul {
  display: flex;
  flex-wrap: wrap;
  width: 100%;
  height: 100%;
  list-style: none;
}

li {
  display: flex;
  align-items: center;
  justify-content: center;
  width: 33.333%;
  height: 50%;
  box-sizing: border-box;
  border: 2px solid green;
  font-size: 30px;
}

页面效果如下：

我们可以看到在样式中，给 body 设置了 width: 1920px; height: 1080px 后，在 1920 * 1080px 的屏幕中，页面是刚好铺满的。

但是，如果我们切换成 4K 的屏幕，会发现之前屏幕的页面，在 4K 的屏幕中没有铺满，如下图所示：

此时，就需要通过 js 代码来就行 scal 等比缩放适配。假设设计稿是 1920 * 1080，我们目标要适配 3 块屏幕，分别是 ：

  • 1920 * 1080
  • 3840 * 2160（相当于是 1920 * 1080 的 2 倍屏）
  • 7680 * 2160（相当于是 1920 * 1080 的 4 * 2 的屏幕）

1. 根据 "宽度比率" 进行缩放

JS 代码如下：

// 设计稿的宽度
const targetX = 1920

// 当前设备的宽度
const currentX = document.documentElement.clientWidth || document.body.clientWidth

// 获取缩放比例
const scaleRatio = currentX / targetX

// 设置缩放比例
document.body.style.transform = `scale(${scaleRatio})`

// 设置缩放的原点在左上角
document.body.style.transformOrigin = `left top`

在引入了以上 JS 代码后，页面的适配效果如下图所示：

从以上适配情况可以发现，通过 宽度比率 进行缩放适配时，1920 * 1080 的屏和 4K 屏都能完美适配，但是当适配 超宽屏 时，页面出现了滚动条。

为了解决这种情况，就需要使用第二种 scale 的适配方案：动态计算网页的宽高比，决定根据 宽度比率 还是 高度比率 缩放。

2. 动态计算网页的宽高比，决定根据 "宽度比率" 还是 "高度比率" 缩放

JS 代码如下所示：

// 设计稿的尺寸
const targetX = 1920
const targetY = 1080

// 设计稿的宽高比例
const targetRatio = 16 / 9

// 当前设备的尺寸
const currentX = document.documentElement.clientWidth || document.body.clientWidth
const currentY = document.documentElement.clientHeight || document.body.clientHeight

// 计算缩放比例
// 参照宽度进行缩放（默认情况）
let scaleRatio = currentX / targetX

// 当前设备宽高比例
const currentRatio = currentX / currentY

// 如果 当前设备的 宽高比 > 设计稿的 宽高比，说明是超宽屏
if (currentRatio > targetRatio) {
  // 参照高度进行缩放
  scaleRatio = currentY / targetY
  // 设置缩放比例
  document.body.style.transform = `scale(${scaleRatio}) translateX(-50%)`
  document.body.style.left = '50%'
}
// 如果 当前设备的 宽高比 <= 16 / 9
else {
  // 设置缩放比例
  document.body.style.transform = `scale(${scaleRatio}) translateY(-50%)`
  document.body.style.top = '50%'
}

// 设置缩放的原点在左上角
document.body.style.transformOrigin = `left top`

// 设置为绝对定位，否则设置 top 和 left 会无效
document.body.style.position = 'absolute'

在引入了以上 JS 代码后，页面的适配效果如下图所示：

从以上适配情况可以发现，在 1920 * 1080、4K 屏的情况下，页面可以完美适配，而后面两种屏幕中，会自动根据 当前设备的宽高比 和 设计稿的宽高比 进行比较，如果 当前设备的宽高比 大于 设计稿的宽高比，也就是说明当前设备屏幕的宽度较宽，那么此时就会使用 高度比例 进行缩放，其他情况就会默认使用 宽度比例 进行缩放，并且可以使得内容进行居中布局，但是周边会有留白的情况。

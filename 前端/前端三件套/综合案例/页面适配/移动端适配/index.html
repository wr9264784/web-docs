REM 适配

一、rem 适配的原理

  • rem（font size of the root element） 是一个相对单位，是相对于 根标签（html） 中的 font-size 为 基准 的。

  • 当页面大小发生变化时，同步修改 根 html 元素 的 font-size 大小，这样凡是属于 rem 单位的实际 px 也会跟着适配改变。

  • 1 rem = html 的 font-size 大小（px）。

例如：

* {
  margin: 0;
  padding: 0;
}

html {
  font-size: 16px;
}

.box {
  /* width: 100px;
  height: 100px; */
  
  /* 6.25rem = 100px */
  width: 6.25rem;
  height: 6.25rem;
  background-color: gold;
}

上述样式表中，设置了 html 的 font-size 为 16px，那么也就意味着：

1rem = 16px
2rem = 32px
3rem = 48px
...

二、rem 适配方式

1. rem + 媒体查询适配

（1）假定 html 根节点 font-size 的算法

这里有一个需求：无论在哪个设备中，页面中的文字始终占一行，盒子始终占一半的宽度，这里我们先用媒体查询的方式进行 rem 的适配，并且假定设计稿中的该宽度的设备的 html 的 font-size 为某个值进行计算：

页面结构如下：

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>rem适配</title>
  <link rel="stylesheet" href="./index.css">
</head>

<body>
  <div class="app">
    <div class="text">文字文字文字文字文字</div>
    <div class="box"></div>
  </div>
</body>

</html>

css 代码如下：

* {
  margin: 0;
  padding: 0;
}

html {
  font-size: 16px;
}

.box {
  width: 3.75rem;
  height: 3.75rem;
  background-color: gold;
}

.text {
  font-size: 0.75rem;
}

@media only screen and (device-width: 750px) {
  html {
    /* 假设 750px 是设计稿的宽度 */
    /* 那么通常会把 750px 的 html font-size 设置为 100px，方便计算 */
    font-size: 100px;
  }
}

@media only screen and (device-width: 375px) {
  html {
    /* 375px 是 750px 的一半，假设 x 为 375px 的 font-size 大小，那么根据： 375px / 750px = x / 100px */
    /* 可以算出在 375px 屏幕下，得到：font-size 是 x = 50px */
    font-size: 50px;
  }
}

@media only screen and (device-width: 320px) {
  html {
    font-size: 42.666px;
  }
}

效果如下图所示：

从上图中可以看到，不管是在设备宽度为 750px、375px 还是 320px，文字始终占一行、盒子使用占一半的宽度。

但是，通过 媒体查询 的方式进行 rem 的适配是有缺点的：

  • 需要适配市面上很多的屏幕，非常麻烦。
  • 需要计算不同屏幕的 html 的 font-size。

总结 媒体查询 适配 rem 的流程：

  1. 得到设计图，一般情况下大部分的设计图宽度都是 750px。
  2. 通常我们会把浏览器的模拟器也调成 750px。
  3. 严格按照设计图去还原页面，需要适配的盒子都用 rem 做单位。
  4. 开始写样式，对于 750px 的模拟器，通常会调整它的 html 的 font-size 为 100px，这叫做 基准值。例如：一个盒子的 宽度是 30px、高度是 55px，换成 rem 为：宽度是 0.3rem、高度是 0.55rem。
  5. 说白了，根据设计稿量出尺寸，需要适配的盒子都转换成 rem 单位， 量出来的 px 数值 / 100 就是对应的 rem 的数值。
  6. 到此，先不管其他屏，先在 750px 的屏上把设计稿都还原完成。
  7. 等到页面绘制完成后，开始适配，需要把你写的 750px 的页面迁移到其他屏，这个过程就是 适配。
  8. 说白了，就是设置其他屏的 html 的 font-size 的大小。

（2）把屏幕分成对应等分算出 font-size 的算法

这种算法可以更好地进行动态计算，也是为了方便配置 js 进行使用的，后续会讲解对应 js 适配的逻辑。

* {
  margin: 0;
  padding: 0;
}

html {
  font-size: 16px;
}

.box {
  width: 5rem;
  height: 5rem;
  background-color: gold;
}

.text {
  font-size: 1rem;
}

@media only screen and (device-width: 750px) {
  html {
    /* 把屏幕分成对应的等分，算出 font-size 大小 */
    /* 相当于把屏幕分成了 10 份，一般都是推荐用 10，为了方便计算 */
    /* 不管是任何屏幕，都会分成 10 份，这里每份就是 75px */
    font-size: 75px;
  }
}

@media only screen and (device-width: 375px) {
  html {
    /* 分成 10 份，每份就是 37.5px */
    font-size: 37.5px;
  }
}

@media only screen and (device-width: 320px) {
  html {
    font-size: 32px;
  }
}

通过把屏幕划分等份的形式算出 font-size 大小，仍然可以实现文字占一行，盒子占一半的效果，只不过需要修改对应的 rem 的值。

通过划分屏幕等份的算法，这里划分为 10 等份：

750px：
	基准值：1rem = 75px
	盒子大小：5rem = 375px，刚好占一半宽度
	文字大小：1rem = 75px，10 个文字刚好一行

375px：
	基准值：1rem = 37.5px
	盒子大小：5rem = 187.5px，刚好占一半宽度
	文字大小：1rem = 37.5px，10 个文字刚好一行

320px：
	基准值：1rem = 32px
	...同上

2. rem + js 适配

回顾一下上面讲的两套算法：

算法有两种（假定 750px 是设计图宽度）：

  • 算法1：把 750px 屏设备的 html 的 font-size 假定 设置为 100px。

    那么 1rem = 100px

一个盒子的宽高为：width: 111px; height: 123px

那么这个盒子对应的 rem 为：
width: 111px / 100px = 1.11rem
height: 123px / 100px = 1.23rem

这种方式，对于不同尺寸的 html 的 font-size 的换算会比较麻烦。

  • 算法2：把 750px 屏设备划分为 10 等份，html 的 font-size 为 75px。

    那么 1rem = 75px

一个盒子的宽高为：width: 111px; height: 123px

那么这个盒子对应的 rem 为：
width: 111px / 75px = 1.48rem
height: 123px / 75px = 1.64rem

这种方式，对于盒子的换算会比较麻烦。但是，可以更好地搭配 js 进行 html 的 font-size 的设置。

仍然实现之前的需求：文字占一行，盒子占一半宽度，下面通过 js 的方式适配 rem：

页面结构：

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>rem适配</title>
  <link rel="stylesheet" href="./index.css">
</head>

<body>
  <div class="app">
    <div class="text">文字文字文字文字文字</div>
    <div class="box"></div>
  </div>

  <script src="./rem.js"></script>
</body>

</html>

样式：

/* index.css */
* {
  margin: 0;
  padding: 0;
}

html {
  font-size: 16px;
}

.box {
  width: 5rem;
  height: 5rem;
  background-color: gold;
}

.text {
  font-size: 1rem;
}

JS 核心换算代码：

// rem.js
window.onload = function () {
  const htmlEle = document.documentElement

  function setRem() {
    const htmlWidth = htmlEle.clientWidth
    const htmlFontSize = htmlWidth / 10
    htmlEle.style.fontSize = htmlFontSize + 'px'
  }

  setRem()

  window.addEventListener('resize', setRem)
}

效果如下图所示：

tips：在实际开发中，可以通过在 vscode 安装对应的 px2rem 的插件，或者 npm 安装对应的插件进行换算，而不是我们手动地对每一个盒子进行换算。

阿里也有对应的开源 JS 库：https://github.com/amfe/lib-flexible。

具体代码如下，实现的功能和我们写的 rem.js 一致，只不过完善更多的细节：

(function flexible (window, document) {
  var docEl = document.documentElement
  var dpr = window.devicePixelRatio || 1

  // adjust body font size
  function setBodyFontSize () {
    if (document.body) {
      document.body.style.fontSize = (12 * dpr) + 'px'
    }
    else {
      document.addEventListener('DOMContentLoaded', setBodyFontSize)
    }
  }
  setBodyFontSize();

  // set 1rem = viewWidth / 10
  function setRemUnit () {
    var rem = docEl.clientWidth / 10
    docEl.style.fontSize = rem + 'px'
  }

  setRemUnit()

  // reset rem unit on page resize
  window.addEventListener('resize', setRemUnit)
  window.addEventListener('pageshow', function (e) {
    if (e.persisted) {
      setRemUnit()
    }
  })

  // detect 0.5px supports
  if (dpr >= 2) {
    var fakeBody = document.createElement('body')
    var testElement = document.createElement('div')
    testElement.style.border = '.5px solid transparent'
    fakeBody.appendChild(testElement)
    docEl.appendChild(fakeBody)
    if (testElement.offsetHeight === 1) {
      docEl.classList.add('hairlines')
    }
    docEl.removeChild(fakeBody)
  }
}(window, document))

vw vh 适配

在 flexible github 上有这样一段话：

由于viewport 单位得到众多浏览器的兼容，lib-flexible 这个过渡方案已经可以放弃使用，不管是现在的版本还是以前的版本，都存有一定的问题。建议大家开始使用 viewport 来替代此方。

前面不管是我们自己编写的 js，还是使用 flexible 来适配 rem，特点都是：

  1. 将 1rem 等同于屏幕宽度的 1 / 10，即 750px 屏的 1rem = 75px，320px 屏的 1rem = 32px。
  2. 然后，再去计算你量出来的盒子大小对应的 rem 值。
  3. 总之：1rem 等同于屏幕宽度的 1 / 10。

现在正好有一个单位，叫做 vw，而 1vw 正好是屏幕宽度的 1 / 100。不管是在什么宽度的屏幕上，1vw 始终是屏幕宽度的 百分之一。

还有一个单位叫做 vh，而 1vh 正好是屏幕高度的 1 / 100。

vw 和 vh 都属于 视口（viewport）单位。

viewport 单位的优势

vw 相比于 rem 的优势：

  1. 不需要去计算 html 的 font-size 大小，也不需要给 html 设置这样一个 font-size。
  2. 不会因为设置 html 的 font-size 大小，而必须给 body 再设置一个 font-size，防止继承。
  3. 因为不依赖 font-size 的尺寸，所以不用担心某些原因 html 的 font-size 尺寸被篡改导致页面尺寸混乱。
  4. vw 相比于 rem 更加语义化，1vw 刚好是 viewport 宽度的 1 / 100。
  5. 可以具备 rem 之前的所有优点。

vw 我们只需要面临一个问题，将 px 尺寸换算成 vw 单位即可，所以，目前比较与 rem，更加推荐使用 vw。（但是理解 rem 依然很重要）。

vw 的使用如下所示：

* {
  margin: 0;
  padding: 0;
}

.box {
  /* 1vw 就等于 屏幕宽度的 1/100 */
  /* 1vh 就等于 屏幕高度的 1/100 */
  /* 如果屏幕宽度为 750px，50vw 相当于 375px */
  width: 50vw;
  height: 50vh;
  background-color: gold;
}

.text {
  /* 如果屏幕宽度为 750px，10vw 相当于 75px */
  font-size: 10vw;
}

tips：vw 的换算仍然是可以通过 vscode 中的插件或项目中通过 npm 安装对应的 px2vw 插件帮助我们自动换算。

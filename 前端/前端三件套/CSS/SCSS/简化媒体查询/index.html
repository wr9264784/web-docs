用 SCSS 简化媒体查询

前言

我们看来一下以下的 媒体查询 代码：

.header {
  width: 100%;
}

@media (min-width: 320px) and (max-width: 480px) {
  .header {
    height: 50px;
  }
}

@media (min-width: 481px) and (max-width: 768px) {
  .header {
    height: 60px;
  }
}

@media (min-width: 769px) and (max-width: 1024px) {
  .header {
    height: 70px;
  }
}

@media (min-width: 1025px) and (max-width: 1200px) {
  .header {
    height: 80px;
  }
}

@media (min-width: 1201px) {
  .header {
    height: 100px;
  }
}

像这种媒体查询，这样写本身是没有错，但是一旦它出现了，那一定不是针对某一个选择器的样式，可能是针对你这个项目中 50% 甚至更多的样式。如果说这么多的选择器都需要写这么一大堆媒体查询的代码，这可是非常麻烦的且恶心的，并且可阅读性也不好。

那么怎么简化呢？我们要做媒体查询就确实得写这么一堆代码，但是我们可以利用 SCSS 来帮助我们简化操作，因为现在的开发基本都是组件化了，在组件里写代码的时候，我只关心这个组件的某个样式，就比方说这里的 .header 样式。

我们应该怎么来简化媒体查询呢？我们可以写一段伪代码来模拟一下：

.header {
  width: 100%;
    
  手机端 {
    // 具体的样式
  }
  平板端 {
    // 具体的样式
  }
}

如果能这样子写的话，我们阅读起来是不是友好一点，比直接查看原来的媒体查询代码要好很多。

以上的写法好处是：

  • 第一是样式都聚合在一起了，阅读起来方便。
  • 第二是我们写的代码也少很多，也不再需要去记媒体查询每一段的尺寸了。

SCSS 混合的基本使用

要想使用以上的写法，就需要用到 SCSS 中的 混合，混合指令（Mixin）用于定义可重复使用的样式，避免了使用无语意的 class。

比方说我们经常会用到的居中布局：

.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

这个居中布局应该很多地方都会用到，那么我们就可以把这个居中的样式抽离出来：

/*  @mixin 后添加混合的名称 */
@mixin center {
  display: flex;
  justify-content: center;
  align-items: center;
}

而当我们想要使用的时候，就可以通过 @include 混合名称 的形式进行使用：

.header {
  width: 100%;
    
  /* 使用混合，相当于 center 里边的那三行代码就写在这里了 */
  @include center;
}

除此之外，mixin 还可以传参，假设我们有一个 flex 布局，但是不确定对齐方向，可以每次调用混合时我们想要的对齐方向都不同，那么我们就可以使用传参的方式：

@mixin flex($layout) {
  display: flex;
  justify-content: $layout;
  align-items: $layout;
}

.header {
  width: 100%;
    
  /* 使用混合，并传入参数 */
  @include flex(start);
}

.header {
  width: 100%;
  @include flex(center);
}

这样我们就能适配 flex 布局中的各种对齐方式了。

SCSS 混合简化媒体查询

理解了 SCSS 混合的基本使用后，我们再回到之前的问题，如何简化媒体查询。

我们可以先定义一个混合 @mixin respond-to($breakname)：

@mixin respond-to($breakname) {
  /* scss 里边是可以写判断语句的 */
  @if $breakname == 'phone' {
    @media (min-width: 320px) and (max-width: 480px) {
      height: 50px;
    }
  }

  @else if $breakname == 'pad' {
    @media (min-width: 481px) and (max-width: 768px) {
      .header {
        height: 60px;
      }
    }
  }
}

.header {
  width: 100%;
  @include respond-to('phone');
}

.header {
  width: 100%;
  @include respond-to('pad');
}

按照这个思路我们就可以继续写出适配笔记本、适配桌面端、适配大屏等。但是，这样的写法还不够灵活、不够通用。因为目前就一个 height 的样式，如果还有其它样式呢？在我们无法确定每个媒体查询断点中需要有哪些样式的时候怎么办呢？

在混合里边，我们在使用混合的时候，可以在混合后面加上一个花括号，然后在里边写我们想要的具体样式：

.header {
  width: 100%;
    
  /* 在 {} 里边写我们想要的样式 */
  @include respond-to('phone') {
    height: 50px;
    background-color: red;
  };
}

上述的写法就有点类型于 vue 框架中 插槽 的使用。然后在定义混合的时候，我们就不要写死样式了，替换为插槽的内容就可以了：

@mixin respond-to($breakname) {
  /* scss 里边是可以写判断语句的 */
  @if $breakname == 'phone' {
    @media (min-width: 320px) and (max-width: 480px) {
      /* 到时候这里的 @content 就会替换为传递进来的具体样式 */
      @content; 
    }
  }

  @else if $breakname == 'pad' {
    @media (min-width: 481px) and (max-width: 768px) {
      @content;
    }
  }
}

那我们继续思考，如果说去写这个 if 判断的话，那这个分支会有很多，我们在 js 代码中，如果判断分支太多的话，一般会通过数据结构映射的关系进行简化，例如：

{
  phone: [320, 480],
  pad: [481, 768],
      
  ...以此类推
}

按照这样的想法，在混合里边也可以用类似的写法，首先要确定一个问题就是在 scss 里边能不能写对象的结构？其实是可以的，只是在 scss 里边它不叫对象，叫 map：

/* 在 scss 里边，map 的结构是使用 () 来表示的 */
$breakpoints: (
  /* 在 scss 里边，数组的形式还是用 () 来表示 */
  'phone': (320px, 480px),
  'pad': (481px, 768px),
  'notebook': (769px, 1024px),
  'desktop': (1025px, 1200px),
  'tv': 1201px
);

然后回到 混合 这里，我们就需要通过拿到传入的设备类型，去这个 map 结构里读取对应的属性值：

@mixin respond-to($breakname) {
  /* scss 里边有一个函数叫做 map-get，专门用来在映射里去取值 */
  /* 第一个参数是映射结构，第二个参数是属性名称 */
  /* 这里取出来之后，赋值给 $bp 变量 */
  $bp: map-get($breakpoints, $breakname);

  /* 然后又因为我们定义数据的时候，map 的属性值又两种情况，一种是数组，一种是单一的一个值 */
  /* 所以，这里还需要判断，通过 type-of 来判断 */
  @if type-of($bp) == 'list' {
    /* 如果是数组类型，就取出前两项，这时候有需要依托于另一个函数 nth() 来获取 */
    /* 取出第一项 */
    $min: nth($bp, 1);
    /* 取出第二项 */
    $max: nth($bp, 2);

    @media (min-width: $min) and (max-width: $max) {
      @content;
    }
  }

  @else {
    @media (min-width: $bp) {
      @content;
    }
  }
}

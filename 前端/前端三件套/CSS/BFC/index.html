BFC

前言

在学习 BFC 之前，先回顾一下两个概念：一个是 行元素、一个是 块元素。通常开发中，这两个元素什么时候用的多一点呢？通常 行元素 都是为了展示一些文字或者图片的，都是一些内容。比如说 span 标签、a 标签、img 标签等，而这些内容展示在哪里？我们通常是做一个整体的规划和布局，所以，块元素 通常用来做整体的规划和布局的。

所以，我们去研究 BFC，要搞清楚研究的是什么，研究的是普通文档留下的 块元素 的排布规则。因为 行元素 是内容，这个内容的展示其实得先有一个整体的布局规划，才能让内容展示出来。然后这个普通的 行元素 有哪些排布的问题？这个问题该如何解决？才引出了所谓的 BFC。

常规流块元素的排布规则

  1. 水平方向上，撑满整个包含块宽度或者撑满父元素的，垂直方向上，它的高度由内容撑开，并且多个块元素垂直方向上依次摆放。
  2. 普通常规流块元素的排布规则基础上存在的问题：
    • 垂直方向上，相邻的块元素，margin 会合并。
    • 父子关系的情况下，可能会产生 margin 塌陷。
    • 父子关系的情况下，如果子元素浮动，父元素会产生 高度坍塌。
    • 兄弟元素之间，正常元素可能会 被浮动元素覆盖（正常元素在浮动元素下方）。

这些问题的根源在哪？它的根源其实在浏览器上，因为我们写完了所有元素后，最后都是通过浏览器去解析的并渲染到页面上的，所以，之所以会产生这些问题是因为浏览器的渲染规则就是这样的。所以，我们才会去研究所谓的 BFC，这个 BFC 说白了就是一块空间，这个空间上有一个单独的浏览器渲染规则，只要把这个规则搞清楚了，那自然就不会有问题了。

BFC - 块级格式化上下文（Block Formatting Context）

块级格式化上下文是什么意思呢？这里总结了几点去理解：

  1. 之前说了，这些所有的元素在浏览器上展示，都是由浏览器渲染的，上述存在的问题就是浏览器渲染规则产生的。所以，BFC 就是开启一个区域，这个区域就是 BFC 区域，是一块 独立的渲染区域，是用来规定渲染规则的。（比如说我写了两个 div，这两个 div 在正常情况下，浏览器渲染规则都是一样的，但如果你想让第一个 div 的渲染跟其他的 div 不一样，我们就可以给第一个 div 开启一个 BFC 区域。就相当于给当前这个元素做了一个独立的说明，做了说明之后，它的规则就跟别人不一样。）
  2. 这个 BFC 的渲染规则是怎么样的呢？它开启了以后，当前 BFC 空间内部跟空间外部的隔开的，内部不会受到外部的影响，也不会影响到外部。
  3. 因为不会产生影响，所以 不同的 BFC 区域，渲染时也互不干扰。

接下来讨论两个问题：

  1. 第一个，这个 BFC 所谓的独立渲染空间，这个空间到底能解决什么问题？ BFC 空间解决的就是 我们之前说的 普通常规流块元素的排布规则基础上存在的问题。

    • 开启 BFC，其子元素 不会再产生 margin 塌陷问题（不会和它的子元素产生 margin 合并）。
    • 开启 BFC，就算子元素浮动，自身高度也不会坍塌（高度计算不再无视浮动元素）。
    • 开启 BFC，自己 不会被其他浮动元素所覆盖（不会与浮动元素重叠，会避开浮动元素排布）。

  2. 第二个，就是这个 BFC 空间到底怎么开启。

    • 当前元素是根元素（HTML），也就是说浏览器最大的那个 html 元素它本身就已经开启了 BFC 设置。

    • html 元素内部的其他元素可以设置以下属性开启 BFC：

    • 设置 float 属性。

    • 设置 position 属性（属性值需要是 absolute 或 fixed）。

    • 设置 overflow 属性（属性值不能为 visible 即可）。

    • 行内块元素（inline-block）。

    • 设置 display 为 flow-root 的元素（IE 浏览器不支持）。

    • 伸缩项目（即其父元素要设置 display 为 flex，成为伸缩盒，当前元素是其父元素的 item）。

    • 多列容器（设置 column-count）。

    • 表格元素（table、thead、tbody、tfoot、tr、th、td、caption）。

    • column-span 为 all 的元素（表格第一行横跨所有列）。

代码实践，看下 BFC 解决的问题

1、首先是解决 margin 塌陷问题，我们先来初始化一个父级盒子，以及内部的三个子元素：

<!-- 制造一个会产生 margin 塌陷的 div 盒子 -->
<!-- 再通过 css 属性解决 margin 塌陷问题 -->
<div class="container">
    <div class="item">box1</div>
    <div class="item">box2</div>
    <div class="item">box3</div>
</div>
* {
  margin: 0;
  padding: 0;
}

.container {
  width: 400px;
  background-color: #f0f2f5;
}

.item {
  width: 100px;
  height: 100px;
  color: #fff;
  text-align: center;
  line-height: 100px;
  font-size: 18px;
}

.item:nth-child(1) {
  background-color: burlywood;
}

.item:nth-child(2) {
  background-color: aquamarine;
}

.item:nth-child(3) {
  background-color: brown;
}

初始化的页面如下图：

然后我们给每个 item 盒子设置 margin：

.item {
  margin: 20px;
  width: 100px;
  height: 100px;
  color: #fff;
  text-align: center;
  line-height: 100px;
  font-size: 18px;
}

再观察页面结构，如下图所示：

根据上图我们可以发现 div 结构发生了塌陷，什么是塌陷呢？就是 box1 的上面的 margin 部分 和 box3 的下面 margin 部分没有了，被父元素抢走了。

那么产生了 margin 塌陷后怎么办呢？可以开启 BFC 区域。让开启了 BFC 区域的盒子有一个独立的渲染规则。就可以不受到这些影响，就可以解决 margin 塌陷问题。

根据之前讨论的两个问题我们可以知道：开启 BFC，其子元素 不会再产生 margin 塌陷问题。所以，我们应该给 父元素 开启 BFC 区域。

body {
  /* .container 盒子的父元素设置 display flex */
  /* 让 .container 盒子开启 BFC 区域 */
  display: flex;
}

.container {
  width: 400px;
  background-color: #f0f2f5;

  /* 设置以下任意一个属性，开启 BFC 区域 */
  /* float: left; */
  /* position: absolute; */
  /* overflow: hidden; */
  /* display: inline-block; */
  /* display: flow-root; */
  .....
}

开启 BFC 区域后的页面结构如下图所示：

根据上图所示，我们可以发现此时的页面结构中，box1 和 box3 的 margin-top 和 margin-bottom 距离又重新回到了对应的子元素身上。这就是通过开启 BFC 区域的独立渲染空间，解决的 margin 塌陷问题。

注意！！这里需要强调几点：

  • 第一个 你要关注的点是 当前这个父元素 container 盒子的空间设置了 BFC，它解决的是父元素内部的这种 margin 塌陷问题，它没有考虑父元素外部会不会受影响。比方说 父元素设置了 浮动，父元素外边的元素一定会有影响，这个你先得考虑到，得自己清除 浮动 后的影响。

  • 第二个 这么多个方式和手段都能解决父元素内部的 margin 塌陷问题，开发中到底用哪个？这得看你具体的时候场景，你要找一个解决方案，它既能解决 margin 塌陷问题，又能对于 父元素 外部的元素没有影响。

2、接下来我们再看一下 BFC 解决父元素开启 BFC，就算子元素浮动，自身高度也不会坍塌 的场景，我们重新写一个页面结构：

<!-- 制造一个会产生 高度坍塌 的 div 盒子 -->
<!-- 再通过 css 属性解决坍塌问题 -->
<div class="container">
    <div class="box">box1</div>
    <div class="box">box2</div>
    <div class="box">box3</div>
</div>
.container {
  width: 400px;
  background-color: #f0f2f5;
}

.box {
  width: 100px;
  height: 100px;
  color: #fff;
  text-align: center;
  line-height: 100px;
  font-size: 18px;
}

.box:nth-child(1) {
  background-color: burlywood;
}

.box:nth-child(2) {
  background-color: aquamarine;
}

.box:nth-child(3) {
  background-color: brown;
}

从上面的结构可以看出，父元素盒子是没有做高度设置，父盒子的高度完全是由子元素的高度撑开的，如下图所示：

然后再给每个子元素盒子设置 浮动：

.box {
  width: 100px;
  height: 100px;
  color: #fff;
  text-align: center;
  line-height: 100px;
  font-size: 18px;
  float: left;
}

然后再看页面结构：

我们可以发现，灰色区域的父级盒子不见了，其实不是不见了，而是因为子元素都浮动了，脱离了文档流，而父元素没有设置高度，此时它的内部没有任何子元素可以撑开高度，所以高度为 0。

然后我们再通过给父元素开启 BFC 区域：

.container {
  width: 400px;
  background-color: #f0f2f5;

  /* 设置以下任意一个属性，开启 BFC 区域 */
  /* float: left; */
  /* position: absolute; */
  /* overflow: hidden; */
  /* display: inline-block; */
  display: flow-root;
  ...
}

看下开启之后的页面结构：

我们可以看到，设置了 BFC 的父级盒子元素，可以解决了因为子元素浮动而造成的高度坍塌问题。给父元素设置了 BFC 后，父元素的渲染规则就会发生变化，在它计算自己的高度的时候，不会无视子元素中的浮动元素。

3、最后再来看一下 开启 BFC，自己 不会被其他浮动元素所覆盖 的场景，我们重新写一个页面结构：

<!-- 制造一个会产生 浮动的盒子 和 正常的盒子 -->
<!-- 再通过 css 属性解决浮动盒子覆盖正常盒子的问题 -->
<div class="brother1"></div>
<div class="brother2"></div>
.brother1, .brother2 {
  width: 200px;
  height: 200px;
  color: #fff;
  font-size: 18px;
  text-align: center;
  line-height: 200px;
}

.brother1 {
  background-color: aquamarine;
}

.brother2 {
  background-color: blueviolet;
}

页面如下图所示：

当我们给 brother1 盒子设置了浮动后：

.brother1 {
  float: left;
  background-color: aquamarine;
}

效果如下图所示：

可以发现浮动的元素覆盖到了正常元素的上方。

接下来要解决这个问题就是要开启 BFC，但是要这么开启 BFC 呢？现在这两个兄弟盒子没有父级元素，应该给谁设置呢？根据之前问题的描述：开启 BFC，自己 不会被其他浮动元素所覆盖。所以，应该给 brother2 元素进行设置。

.brother2 {
  background-color: blueviolet;

  /* 设置以下任意一个属性，开启 BFC 区域 */
  /* float: left; */
  /* position: absolute; */
  /* overflow: hidden; */
  /* display: inline-block; */
  display: flow-root;
  ...
}

总结

BFC 不是一个特别神奇的东西，我们就把 BFC 当作是一个设置，设置当前块级元素为一个独立的空间。这个空间渲染的时候告诉浏览器一声，当前的块级元素特殊，需要单独渲染它。所以在单独渲染它的时候，就不能跟其他的元素做重叠或者覆盖。

TS 内置工具类的使用

操作接口类型

Partial

Partail 的源码：

type Partial<T> = {
  [P in keyof T]?: T[P];
};

Partial 工具类型可以将一个定义好的对象类型（包含 必选 + 可选项）的 所有属性 全部转化为 可选 的，且该工具类型返回的类型是给定类型的所有子集。

// 声明 Person 类型，属性值都是必填的
type Person = {
  name: string
  age: number
  gender: 1 | 2
  address: string
}

// Partial<> 把 Person 的所有属性都变成可选，并返回所有子集，不会修改原 Person 类型
type newPerson = Partial<Person>

可以看到经过 Partial 修饰过的 Person 类型中的所有属性都变成了 可选。

Required

Required 的源码：

type Required<T> = {
  [P in keyof T]-?: T[P];
};

映射类型在键值的后面使用了一个 - 符号，- 与 ? 组合起来表示去除类型的可选属性，因此给定类型的所有属性都变为了 必填。

与 Partial 相反，Required 会把 所有属性 都变成 必填。

// 声明 Person 类型，属性值都是可选的
type Person = {
  name?: string
  age?: number
  gender?: 1 | 2
  address?: string
}

// Required<> 把 Person 的所有属性都变成必填，并返回所有子集，不会修改原 Person 类型
type newPserson = Required<Person>

效果如下：

可以看到经过 Required 修饰过的 Person 类型中的所有属性都变成了 必填。

Readonly

Readonly 的源码：

type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

Readonly 工具类型可以将给定类型的 所有属性 设为 只读，这意味着给定类型的属性不可以被重新赋值。

// 声明一个 Person 类型
type Person = {
  name: string
  age: number
  gender: 1 | 2
}

// 通过 Readonly<> 把 Person 里的所有属性都变成 只读
type newPerson = Readonly<Person>

// 常见一个类型为 newPerson 的对象
const p: newPerson = {
  name: '张三',
  age: 18,
  gender: 1
}

// 因为通过 Readonly 把 Person 的属性都设置为了只读
// 当我们在创建这个 p 对象并初始化后，就不能再对这个对象的
// name，age，gender 属性进行修改
p.name = '李四' // 报错

Pick

Pick 的源码：

type Pick<T, K extends keyof T> = {
  [P in K]: T[P];
};

Pick 工具类型接收了 两个泛型参数，第一个 T 为给定的参数类型，而第二个参数为需要提取的键值 key。有了参数类型和需要提取的键值 key，我们就可以通过映射类型很容易地实现 Pick 工具类型的功能。

Pick 工具类型可以从给定的类型中选取出指定的键值，然后组成一个新的类型。

type Person = {
  name: string
  age: number
  gender: 1 | 2,
  address: string
}

// 从 Person 类型中提出 name 和 age 属性，并组成新的类型返回给 newPerson
type newPerson = Pick<Person, 'name' | 'age'>

Omit

Omit 的源码：

type Omit<T, K extends keyof any> = Pick<T, Exclude<keyof T, K>>;

从 Omit 源码可以发现，Omit 类型的实现使用了前面介绍的 Pick 类型。我们知道 Pick 类型的作用是选取给定类型的指定属性，那么这里的 Omit 的作用应该是选取除了指定属性之外的属性，而 Exclude 工具类型的作用就是从入参 T 属性的联合类型中排除入参 K 指定的若干属性。

与 Pick 类型相反，Omit 工具类型的功能是返回去除指定的键值之后返回的新类型。

type Person = {
  name: string
  age: number
  gender: 1 | 2,
  address: string
}

// 通过 Omit 工具类型把 Person 的 gender 和 address 属性去除并组成新的类型返回
type newPerson = Omit<Person, 'gender' | 'address'>

联合类型

Exclude

在介绍 Omit 类型的实现中，我们使用了 Exclude 类型。通过使用 Exclude 类型，我们从接口的所有属性中去除了指定属性，因此，Exclude 的作用就是从联合类型中去除指定的类型。

Exclude 的源码：

type Exclude<T, U> = T extends U ? never : T

上述源码表示：Exclude 的实现使用了条件类型。如果类型 T 可以继承类型 U ，则不返回类型 T，否则返回此类型 T ，这样我们就从联合类型中去除了指定的类型。

// 创建一个联合类型
type status = 'a' | 'b' | 'c'

// 把联合类型 status 中的 'b' 去除，并返回新的联合类型给 newStatus
type newStatus = Exclude<status, 'b'>

Extract

Extrace 的源码：

type Extract<T, U> = T extends U ? T : never;

Extract 类型的作用与 Exclude 正好相反，Extract 主要用来从联合类型中提取指定的类型，类似于操作接口类型中的 Pick 类型。

type status = 'a' | 'b' | 'c'

// 通过 Extract 工具类，把联合类型 status 中的 a 和 b 提取出来
type newStatus = Extract<status, 'a' | 'b'>

NonNullable

NonNullable 的源码：

type NonNullable<T> = T extends null | undefined ? never : T;

// 等同于使用 Exclude
type NonNullable<T> = Exclude<T, null | undefined>;

NonNullable 的作用是从 联合类型 中去除 null 或者 undefined 的类型。

type status = 'a' | 'b' | 'c' | undefined | null

// 去除 undefined 和 null
type newStatus = NonNullable<status>

Record

Record 源码：

type Record<K extends keyof any, T> = {
  [P in K]: T;
};

Record 的作用是生成 接口类型，Record 类型接收了两个泛型参数：第一个参数作为接口类型的 属性，第二个参数作为接口类型的 属性值。

// 定义两个联合类型
type Key = 'name' | 'age'
type Value = string | number

// 通过 Record 把 key 和 Value 组装成一个 接口类型
type Person = Record<Key, Value>

// 正常
const p: Person = {
  name: '张三',
  age: 18,
}

// 报错，因为 value 的类型只能是 string 或 number
const p2: Person = {
  name: '张三',
  age: true,
}

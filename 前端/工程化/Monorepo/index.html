Monorepo

介绍

Monorepo 是一种项目代码管理方式，指单个仓库中管理多个项目，有助于简化代码共享、版本控制、构建和部署等方面的复杂性，并提供更好的可重用性和协作性。

本文主要是通过 pnpm 的 workspace 工作空间来实现 Monorepo 管理 vue3 + vite + ts 的组件库。

项目搭建

  1. 执行 pnpm create vite，创建 vue + vite + ts 项目。没有安装 pnpm 的需要先安装。执行 npm i -g pnpm 全局安装 pnpm。

  2. 修改 根目录 下的 pakage.json 文件：

    {
  "name": "simple-space",
  // 防止最外层的包被发布出去，设置为 true 以后发布时会提醒你
  "private": true, // 设置为私有，防止被发布
  "version": "0.0.0",
  "type": "module",
    
  // 删除 scripts 属性中的默认脚本命令
  // 新增 "preinstall": "npx only-allow pnpm" 表示只允许通过 pnpm 来开发
  // preinstall 是在 install 之前（首次）执行的。
  // 还可以设置 postinstall，表示在 install 之后（首次）执行
  "scripts": {
    "preinstall": "npx only-allow pnpm",
  },
  "dependencies": {
    "vue": "^3.4.31"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^5.0.5",
    "typescript": "^5.2.2",
    "vite": "^5.3.4",
    "vue-tsc": "^2.0.24"
  }
}

  3. 项目根目录中新增 pnpm-workspace.yaml 文件，并编辑：

    # 指定哪些目录作为独立的工作空间，这个工作空间可以理解为一个子模块或者 npm 包
packages:
 # 各种子包的项目目录，包括组件库和一些工具库
 - 'packages/*' 
 
 # 文档项目，存放文档的目录，用于展示组件的使用方法
 - 'docs'
 
 # 用于测试组件库的项目
 - 'example'

  4. 因为在上一步中，指定了 packages 目录，所以需要在项目根目录中，创建 packages 目录，用来存放各种子包。

  5. 进入 packages 目录，执行 pnpm create vite bit-form 创建 bit-form 组件库项目，用于编写将来我们要写的相关组件。并修改 bit-form 目录下的 package.json：

    {
  "name": "@simple-space/bit-form", // 修改包名
  "private": false, // 设置非私人包
  "version": "0.0.1", // 修改版本
  
  // 省略其它...
}

  6. 继续进入到 packages 目录，创建 utils 目录用于编写一些公共工具库，在 utils 目录下执行 pnpm init 初始化，并修改 utils 目录下的 packge.json：

    {
  "name": "@simple-space/utils", // 修改包名
  "private": false, // 设置非私人包
  "version": "0.0.1", // 修改版本
  
  // 省略其它...
}

  7. 分别修改 packages/bit-form 和 packages/utils 目录下的 tsconfig.json：

    {
  // extends 表示当前目录下的 tsconfig.json 的配置全部都继承自根目录下的 tsconfig.json 文件。
  "extends": ["../../tsconfig.json"],
    
  // 并且还能在当前目录下的 tsconfig.json 中重写或新增一些配置。
  "include": ["src/**/*.ts", "src/**/*.tsx", "src/**/*.vue"]
}

  8. 修改根目录下的 tsconfig.json 文件，把 include 配置去掉，如果本来就没有配置就不用理会。

  9. 删除项目根目录中用不到的文件和目录，这里删除 public 目录、src 目录、index.html 文件。

  10. 删除 packages/bit-form 中用不到的文件和目录，这里要删除的目录和文件比较多，直接下面的目录结构。

全局挂载组件

有时候我们希望在 vue3 项目中，使用组件库时，可以在 main.ts 中全局引入，然后在组件页面中就可以直接使用，例如：

// main.ts

import { createApp } from 'vue'
import App from './App.vue'
import BitForm from '@simple-space/bit-form' // 全局引入组件库

const app = createApp(App)
app.use(BitForm) // 通过 app.use() 挂载组件
app.mount('#app')

组件页面：

<template>
  <!-- 直接使用组件，无需引入 -->
  <bit-button></bit-button>
</template>

其实使用 app.use() 的时候它会调用传入参数的 install 方法，因此首先我们给每个组件添加一个 install 方法，然后再导出整个组件库。

我们将 packages/bit-form/src/Button/index.ts 改为：

// import BitButton from "./index.vue";
// export { BitButton };
// export default BitButton;

// 用以下方式注册组件并导出
import Button from "./index.vue";
import type { App, Plugin } from "vue";

type SFCWithInstall<T> = T & Plugin;

const withInstall = <T>(comp: T) => {
  (comp as SFCWithInstall<T>).install = (app: App) => {
    const name = (comp as any).name;
    // 注册组件
    app.component(name, comp as SFCWithInstall<T>);
  };
  return comp as SFCWithInstall<T>;
};

export const BitButton = withInstall(Button);
export default BitButton;

Upload 组件同理同样的修改，这里就不展示了。

然后将 packages/bit-form/index.ts 改为：

// 在这里统一导出所有组件
// export * from './src/index'

import * as components from "./src/index";
export * from "./src/index";
import { App } from "vue";

export default {
  install: (app: App) => {
    for (let c in components) {
      // 这里会包 ts 错误提示：
      // 元素隐式具有 "any" 类型，因为类型为 "string" 的表达式不能用于索引类型 "typeof"
      // 虽然不影响使用，但是我们还是应该要解决这个问题。
      app.use(components[c]);
    }
  }
}

我们把 packages/bit-form/index.ts 改为：

// 在这里统一导出所有组件
// export * from './src/index'

import * as components from "./src/index";
export * from "./src/index";
import { App } from "vue";

// 获取我们导入的 components 的类型
type Component = typeof components

// 通过 components 类型得到对应的联合类型
type Key = keyof Component

export default {
  install: (app: App) => {
    for (let c in components) {
      // 声明一个 key，类型为 Key
      const key: Key = c as Key
      app.use(components[key]);
    }
  }
}

抽离封装组件注册方法

这里可以有优化空间！因为我们将来肯定会有很多的组件，并且每个组件的目录下的 index.ts 都会写一遍 withInstall 方法，这会很繁琐，所以我们可以新建 /packages/bit-form/utils/install.ts 文件，然后在 install.ts 中封装 withInstall 方法：

import type { App, Directive } from 'vue'
import type { SFCInstallWithContext, SFCWithInstall } from './typescript'

// 注册组件
export const withInstall = <T, E extends Record<string, any>>(
  main: T,
  extra?: E
) => {
  (main as SFCWithInstall<T>).install = (app: App): void => {
    for (const comp of [main, ...Object.values(extra ?? {})]) {
      app.component(comp.name, comp)
    }
  }

  if (extra) {
    for (const [key, comp] of Object.entries(extra)) {
      (main as any)[key] = comp
    }
  }
  return main as SFCWithInstall<T> & E
}

// 注册方法
export const withInstallFunction = <T>(fn: T, name: string) => {
  (fn as SFCWithInstall<T>).install = (app: App) => {
    (fn as SFCInstallWithContext<T>)._context = app._context
    app.config.globalProperties[name] = fn
  }

  return fn as SFCInstallWithContext<T>
}

// 注册指令
export const withInstallDirective = <T extends Directive>(
  directive: T,
  name: string
) => {
  (directive as SFCWithInstall<T>).install = (app: App): void => {
    app.directive(name, directive)
  }

  return directive as SFCWithInstall<T>
}

然后新建 /packages/bit-form/utils/typescript.d.ts 用于编写对应的 withInstall 用到的类型：

import type { AppContext, Plugin } from 'vue'

export type SFCWithInstall<T> = T & Plugin

export type SFCInstallWithContext<T> = SFCWithInstall<T> & {
  _context: AppContext | null
}

然后修改 /packages/bit-form/src/Button/index.ts，调用我们封装的 withInstall 方法：

import Button from "./index.vue";
import { withInstall } from "../../utils/install";

配置 TS 类型提示

我们先来看一个现象，当我们通过 局部引入 的方式引入组件并使用，当我们鼠标移动到组件名称时，发现会弹出该组件对应的属性提示：

而当我们通过 全局引入 并使用组件时，发现并不会提供任何的提示：

我们新建 packages/bit-form/src/components.d.ts 文件：

import * as components from './index'

// Volar 插件说明：https://github.com/vuejs/language-tools
// 声明了组件类型后，在页面中调用组件时，鼠标悬停组件标签，
// 会显示组件相应的属性值，方便开发人员知道该组件有哪些属性。
declare module 'vue' {
  export interface GlobalComponents {
    BitButton: typeof components.BitButton
    BitUpload: typeof components.BitUpload
  }
}

export {}

vite 打包组件库

初步配置打包

vite 专门提供了库模式的打包方式,配置其实非常简单，首先安装 vite 以及 @vitejs/plugin-vue，不过因为我这里组件库是直接通过 pnpm create vite 创建的 vue3 + vite 项目，所以这里就不需要额外安装了。

修改 /packages/bit-form/vite.config.ts：

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [vue()],

  build: {
    // 压缩
    minify: false,

    // 自定义底层的 Rollup 打包配置。这与从 Rollup 配置文件导出的选项相同，并将与 Vite 的内部 Rollup 选项合并。
    rollupOptions: {
      // 忽略打包 vue 文件
      external: ["vue"],

      // 指定 bundle 的入口文件（也就是组件库的入口文件）
      input: "./index.ts",

      output: {
        // 该选项用于在 umd / iife bundle 中，使用 id: variableName 键值对指定外部依赖。
        globals: {
          vue: "Vue",
        },
          
        // 打包的输出目录
        dir: "dist",
      },
    },

    lib: {
      entry: "./index.ts",
      name: "bit-form",
      fileName: "bit-form",
      formats: ["es", "umd", "cjs"],
    }
  }
})

在 /packages/bit-form/package.json 中的 scripts 字段中添加 build 脚本命令：

{
  // 省略其它...
  "scripts": {
    // ... 省略其它
     
    "build": "vite build"
  }
}

但是因为我这里的组件库一开始就是用的 pnpm create vite 创建的 vue3 + vite 项目，所以 package.json 文件中已经有了 build 的编译脚本命令，那就不需要添加了，只需要把 vue-tsc -b && vite build 修改成 vite build 即可，否则执行 pnpm run build 会报错。

然后 cd 到 /packages/bit-form 目录中，在终端执行 pnpm run build 命令进行打包，打包后的结果如下：

配置按需引入的打包

但是上述的打包方式最终会将整个组件库打包到一个文件中，并且样式文件也 不能按需加载，所以我们需要修改一下配置让打包后的结构和我们开发的结构一致，修改 /packages/bit-form/vite.config.ts：

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [vue()],

  build: {
    // 压缩
    minify: false,

    // 自定义底层的 Rollup 打包配置。这与从 Rollup 配置文件导出的选项相同，并将与 Vite 的内部 Rollup 选项合并。
    rollupOptions: {
      // 忽略打包 vue 文件
      external: ["vue"],

      // 指定 bundle 的入口文件（也就是组件库的入口文件）
      input: "./index.ts",

      output: [
        {
          format: "es", // 打包格式
          entryFileNames: "[name].mjs", // 打包后文件名
          preserveModules: true, // 让打包目录和我们目录对应
          exports: "named", // 该选项用于指定导出模式，'named' 适用于使用命名导出的情况，老实说你删了这个配置项都行
          dir: "./dist/es", // 配置打包根目录
        },
        {
          format: "cjs", // CommonJS 模式
          entryFileNames: "[name].js",
          preserveModules: true,
          exports: "named",
          dir: "./dist/lib",
        }
      ]
    },

    lib: {
      entry: "./index.ts",
    }
  }
})

打包结果如下：

这样，当我们在使用组件库的时候就可以进行 按需引入 了。但是此时的所有样式文件还是会统一打包到 style.css 中，还是不能进行样式的按需加载，后续我们再继续实现样式文件的按需加载。

声明文件

到这里其实打包的组件库只能给 js 项目使用，在 ts 项目下运行会出现一些错误，而且使用的时候还会失去代码提示功能，这样的话我们就失去了用 ts 开发组件库的意义了。所以我们需要在打包的库里加入 声明文件 (.d.ts)。

首先要安装 vite-plugin-dts 插件，我这里直接安装在根目录中：

pnpm add vite-plugin-dts -D -w

然后在 /packages/bit-form/vite.config.ts 中，使用该插件：

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import dts from "vite-plugin-dts"

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    vue(),
     
    // 使用生成 d.ts 声明文件的插件
    dts({
      entryRoot: "./src",

      // 指定 d.ts 类型声明文件的输出位置
      outDir: ["./dist/es/src", "./dist/lib/src"],

      // 指定使用的根目录的 tsconfig.json，也可以单独为 /packages/bit-form 指定 tsconfig.json
      tsconfigPath: "./tsconfig.json",
    }),
  ],

  // ...省略其它
})

在 /packages/bit-form/ 路径下，执行 pnpm run build 重新执行打包命令，打包结果如下：

从上图可以看到，我们已经成功地生成了 .d.ts 文件。

gulp 打包组件库

当我们使用 Vite 库模式打包的时候，vite 会将样式文件全部打包到同一个文件中，这样的话我们每次都要全量引入所有样式文件做不到按需引入的效果。所以打包的时候我们可以不让 vite 打包样式文件，样式文件将使用 gulp 进行打包。

安装 gulp

我们需要使用 ts 以及新的 es6 语法，而 gulp 是不支持的，所以我们需要安装一些依赖使得 gulp 支持这些，其中 sucras 让我们执行 gulp 可以使用最新语法并且支持 ts，同样的，我们把包安装在根目录：

pnpm i gulp @types/gulp sucrase -D -w

gulp 打包样式按需加载

因为我们用的是 scss 写的样式，所以需要安装 gulp-sass，同时在安装一个自动补全 css 前缀插件 gulp-autoprefixer 以及它们对应的类型文件：

pnpm add gulp-sass @types/gulp-sass gulp-autoprefixer @types/gulp-autoprefixer -D -w

如果是使用的 less 写的样式，则安装以下：

pnpm add gulp-less @types/gulp-less gulp-autoprefixer @types/gulp-autoprefixer -D -w

我们新建 /packages/bit-form/scripts/build/index.ts 文件，用来存放跟打包执行脚本相关的内容，我们先写一个打包样式的函数：

import { src, dest } from "gulp";
import glupSass from 'gulp-sass';
import sassLang from 'sass';
import autoprefixer from "gulp-autoprefixer";
import { resolve } from 'path'

// 组件库根目录
const componentPath = resolve(__dirname, '../../')

// packages 根目录
const pkgPath = resolve(__dirname, '../../../')

const sass = glupSass(sassLang);

// 打包样式
export const buildStyle = () => {
  return src(`${componentPath}/src/**/*.scss`)
    .pipe(sass())
    .pipe(autoprefixer())
    .pipe(dest(`${pkgPath}/bit-form/dist/lib/src`))
    .pipe(dest(`${pkgPath}/bit-form/dist/es/src`));
};

这里使用到了 gulp 的 dest 和 src 函数，不知道什么意思的可以看 gulp 的介绍。

然后在 /packages/bit-form/package.json 中的 scripts 属性新增新的编辑脚本命令：

{
  // ...省略其它
    
  "scripts": {
    // ...省略其它
      
    "build:bit-form": "gulp -f scripts/build/index.ts"
  }
}

然后在 /packages/bit-form/ 路径下执行 pnpm run build:bit-form，查看打包结果，如果打包结果像下面一样报错的：

只需要把 gulp-autoprefixer 版本号降为 ^8.0.0 即可：

pnpm add gulp-autoprefixer@8.0.0 -D -w

然后再执行一次 pnpm run build:bit-form，发现能正常编译，并且打包后的目录中每个组件的下面都有对应的样式文件了：

这里我们仅仅是打包了样式而已，还没有通过 gulp 打包组件，所以打包的目录中只有样式文件，没有对应的组件文件，接下来就要使用 gulp 来打包组件了。

gulp 打包组件

再写一个打包组件的函数，这里需要写一个执行命令的工具函数，我们写在 /packages/bit-form/scripts/utils/run.ts 中：

import { spawn } from "child_process";

export default async (command: string, path: string) => {
  //cmd表示命令，args代表参数，如 rm -rf  rm就是命令，-rf就为参数
  const [cmd, ...args] = command.split(" ");
  return new Promise((resolve, reject) => {
    const app = spawn(cmd, args, {
      cwd: path, //执行命令的路径
      stdio: "inherit", //输出共享给父进程
      shell: true, //mac不需要开启，windows下git base需要开启支持
    });
    //执行完毕关闭并resolve
    app.on("close", resolve);
  });
};

然后在 /packages/bit-form/scripts/build/index.ts 中引入：

// 省略其它..
import run from '../utils/run'

// 省略其它..

// 打包组件
export const buildComponent = async () => {
  run("pnpm run build", componentPath); // 其实就是通过 gulp 命令调用了 vite 的打包
}

因为打包样式和打包组件可以并行，所以最后的 /packages/bit-form/scripts/build/index.ts 为：

import { src, dest, series, parallel } from "gulp";
import glupSass from 'gulp-sass';
import sassLang from 'sass';
import autoprefixer from "gulp-autoprefixer";
import { resolve } from 'path'
import run from '../utils/run'

// 组件库根目录
const componentPath = resolve(__dirname, '../../')

// packages 根目录
const pkgPath = resolve(__dirname, '../../../')

const sass = glupSass(sassLang);

// 打包样式
export const buildStyle = () => {
  return src(`${componentPath}/src/**/style/**.scss`)
    .pipe(sass())
    .pipe(autoprefixer())
    .pipe(dest(`${pkgPath}/bit-form/dist/lib/src`))
    .pipe(dest(`${pkgPath}/bit-form/dist/es/src`));
};

// 打包组件
export const buildComponent = async () => {
  run("pnpm run build", componentPath);
};

// 同时执行打包组件和打包样式
export default series(
  parallel(
    async () => buildStyle(),
    async () => buildComponent()
  )
)

然后修改 /packages/bit-form/vite.config.ts，设置 emptyOutDir 为 false：

import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import dts from "vite-plugin-dts"

// https://vitejs.dev/config/
export default defineConfig({
  // ...省略其它

  build: {
    // ... 省略其它

    emptyOutDir: false
  }
})

发布组件库

发布前的配置

我们要发布的是打包后的 /packages/bit-form/dist 目录，所以在 dist 目录中执行 pnpm init 初始化生成 package.json，然后修改 package.json：

{
  "name": "bit-form",
  "version": "0.0.1",
  "description": "",
  "main": "lib/index.js",
  "module": "es/index.mjs",
  "keywords": [
    "bit-form",
    "vue3 + vite + ts 组件库"
  ],
  "files": [
    "es",
    "lib"
  ],
  "author": "wr",
  "license": "ISC",
  "typings": "lib/src/index.d.ts",
  "sideEffects": [
    "**/*.css"
  ]
}

  • main：组件库入口文件。

  • module：如果使用组件库的环境支持 esmodule 则入口文件变成这个字段。

  • files：发布到 npm 上的文件目录。

  • sideEffects：忽略 tree shaking 带来副作用的代码。

    比如打包后组件代码中包含了：

    import "./xxx.css"

    这样会使得构建工具无法知道这段代码是否有副作用（也就是会不会用到其它引入的文件中的代码），所以构建的时候就会全量打包代码从而失去 esmodule 的自动按需引入功能。因此加上 sideEffects 字段就可以告诉构建工具这段代码不会产生副作用，可以放心的 tree shaking。

  • typings：声明文件入口。

手动发布组件库

在打包目录下执行 npm login 登录你的 npm 账号，然后 pnpm publish 发布组件库。

然后在我们的 npm 仓库中也看到了对应的组件：

release-it 自动发布

上面的发布每次更新我们需要手动去提升版本,手动去打 tag 等,非常不方便,接下来我们将使用 release-it 来管理这些。首先在根目录中安装：

pnpm add release-it -D -w

然后在打包后文件 /packages/bit-form/dist 下的 package.json 中加入 script 脚本以及 git 仓库地址：

{
  // 省略其它..
  "scripts": {
    "release": "release-it"
  },
  "repository": {
    "type": "git",
    "url": "https://www.npmjs.com/package/bit-form" // 这里换成你的 npm 组件库地址
  }
}

在 /packages/bit-form/scripts/ 目录下新建 publish/index.ts 用于发布任务：

import run from "../utils/run";
import { series } from "gulp";
import { resolve } from 'path'

// packages 根目录
const pkgPath = resolve(__dirname, '../../../')

// 发包方法
export const publishComponent = async () => {
  run("release-it", `${pkgPath}/dist`);
};
export default series(async () => publishComponent());

在 /packages/bit-form/ 目录的 package.json 文件中新增 scripts 命令 gulp 执行 publish/index.ts：

{
  // 省略其它...
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview",
    "build:bit-form": "gulp -f scripts/build/index.ts", // gulp 打包组件和样式
    "publish:bit-form": "gulp -f script/publish/index.ts" // 发布
  }
}

然后将我们的改动提交后执行pnpm run publish:bit-form,就会发现他让我们选择如何提升版本,是否发布,是否加个 tag 等等，选择完之后我们的组件库就发布成功了。

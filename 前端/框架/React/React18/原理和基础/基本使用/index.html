React 介绍

React 由 Meta 公司开发，是一个用于 构建 Web 和 原生交互界面的库

相较于传统基于 DOM 开发的优势

  1. 组件化的开发方式
  2. 不错的性能

相较于其它前端框架的优势

  1. 丰富的生态
  2. 跨平台支持

React 的市场情况

全球最流行，大厂必备

开发环境创建

create-react-app 是一个快速创建 React 开发环境的工具，底层由 Webpack 构件，封装了配置细节，开箱即用 执行命令：

npx create-react-app react-basic
  1. npx - Node.js工具命令，查找并执行后续的包命令
  2. create-react-app - 核心包（固定写法），用于创建React项目
  3. react-basic React 项目的名称（可以自定义）
  4. 创建 React 项目的更多方式 https://zh-hans.react.dev/learn/start-a-new-react-project

React 脚手架中的 webpack 配置

使用 create-react-app 生成项目，不会有 webpack 的配置项，要导出 webpack，必须使用 react-script eject 暴露出 create-react 中的相关 webpack、babel 等配置文件在新增的 config 目录下，同时新增scripts 目录，该目录下会有新的命令文件更新，package.json 文件中 scripts 命令同步更新。

但这是一个单向操作 react-scripts eject 后，就无法恢复了。如果只是修改一些简单的配置，eject 是没有必要的。

注意：！！如果真的执行了 npm run eject，那么你将无法跟随官方的脚步去升级项目的 react-script 版本。

我们可以通过插件来配置 webpack，而不执行 react-script eject。

react-scripts eject 的副作用

虽然执行 eject 后可以根据自己的需要添加配置项做一些更高阶的配置，但是由于这个操作是不可逆的，所以也会带来一些问题：

  • 维护性。因为 eject 的不可逆性，之后复杂的 webpack 等配置由框架本身转交给用户自己进行维护了。一些我们不需要修改的 react 官方默认配置可能就会丢失。其次，在版本迭代时，如果更新了 react、react-scripts、eslint、tsconfig 等依赖，有可能会引起版本依赖的问题，即使我们按错误信息修复了之后，项目还是无法运行。
  • 代码简洁性。create-react-app 其中的一个目的就是可以让你专注与项目的开发，不用去关心其他的配置与服务，然而在 eject 之后，package.json就写入了很多依赖项，还有其他的脚本和配置文件，这在维护依赖项的时候会十分不友好。

通过插件修改 webpack 配置

通过上述我们可以知道执行 eject 的副作用，那么如果我们需要修改相关 webpack 配置，应该怎么做呢？

我们可以通过 react-app-rewired 和 @craco/craco 两个插件优雅的修改 create-react-app 生成的项目配置。两个插件的使用方法都差不多，也都可以实现我们的需求，至于使用哪一个见仁见智。我建议是用 @craco/craco，比较新，react-app-rewired 最后的更新已经是两年前了。

跨域的配置

要想处理跨域，我们需要在 /src 目录下，新建一个 setupProxy.ts 文件，并且代理还需要安装 http-proxy-middleware 依赖：

/**
 * 处理反向代理请求的模块，必须创建在 src/setUpProxy.js

 */
// 通过 npm install http-proxy-middleware --save 安装依赖
// 导入创建 Proxy 的中间件
const { createProxyMiddleware } = require('http-proxy-middleware');

// 导出的函数中可以拿到当前 应用 app
module.exports = function (app: any) {
  // 注册中间件
  app.use(
    // 以 /proxy 开头请求的路径，如：/proxy/list、/proxy/detail 等请求路径，让 /proxy 开头的路径进行代理配置
    '/proxy',
    createProxyMiddleware({
      // 让以 /proxy 开头的路径请求，拼接到 target 指定路径的后面，然后让服务器帮我们发送请求，
      // 这样不是以浏览器发起请求，就不会存在跨域现象。
      target: 'https://i.maoyan.com',
      changeOrigin: true,
    })
  );
}

JSX 基础

什么是 JSX

概念：JSX 是 JavaScript 和 XMl(HTML) 的缩写，表示在 JS 代码中编写 HTML 模版结构，它是 React 中 构建UI 的方式

const message = 'this is message'

function App(){
  return (
    <div>
      <h1>this is title</h1>
      { message }
    </div>
  )
}

优势：

  1. HTML 的声明式模版写法
  2. JavaScript 的可编程能力

JSX 的本质

JSX 并不是标准的 JS语法，它是 JS的语法扩展，浏览器本身不能识别，需要通过解析工具做解析之后才能在浏览器中使用

JSX 高频场景 - JS表达式

在JSX中可以通过 大括号语法{} 识别 JavaScript中 的表达式，比如常见的变量、函数调用、方法调用等等

  1. 使用引号传递字符串
  2. 使用JS变量
  3. 函数调用和方法调用
  4. 使用 JavaScript 对象
  5. 注意：if 语句、switch 语句、变量声明不属于表达式，不能出现在 {} 中
const message = 'this is message'

function getAge(){
  return 18
}

function App(){
  return (
    <div>
      <h1>this is title</h1>
      {/* 字符串识别 */}
      {'this is str'}
      {/* 变量识别 */}
      {message}
      {/* 变量识别 */}
      {message}
      {/* 函数调用 渲染为函数的返回值 */}
      {getAge()}
    </div>
  )
}

React 的事件绑定

基础实现

React 中的事件绑定，通过语法 on + 事件名称 = { 事件处理程序 } 绑定到元素标签中，整体上遵循驼峰命名法：

function App(){
  const clickHandler = ()=>{
    console.log('button按钮点击了')
  }
  return (
    <button onClick={ clickHandler }>click me</button>
  )
}

使用事件参数

在事件回调函数中设置 形参 e 即可

function App(){
  const clickHandler = (e)=>{
    console.log('button按钮点击了', e)
  }
  return (
    <button onClick={ clickHandler }>click me</button>
  )
}

传递自定义参数

语法：事件绑定的位置改造成箭头函数的写法，在执行 clickHandler 实际处理业务函数的时候传递实参

function App(){
  const clickHandler = (name)=>{
    console.log('button按钮点击了', name)
  }
  return (
    <button onClick={ () => clickHandler('jack') }>click me</button>
  )
}

注意：不能直接写函数调用，如果直接写函数调用，还没有点击就直接调用了函数，这里事件绑定需要一个函数引用。

同时传递事件对象和自定义参数

语法：在事件绑定的位置传递事件实参e和自定义参数，clickHandler 中声明形参，注意顺序对应

function App(){
  const clickHandler = (name,e)=>{
    console.log('button按钮点击了', name,e)
  }
  return (
    <button onClick={ (e) => clickHandler('jack', e) }>click me</button>
  )
}

组件是什么

概念：一个组件就是一个用户界面的一部分，它可以有自己的逻辑和外观，组件之间可以互相嵌套，也可以多次复用。

组件状态管理 - useState

基础使用

useState 是一个 React Hook（函数），它允许我们向组件添加一个 状态变量，从而控制影响组件的渲染结果 和 普通JS变量 不同的是，状态变量一旦发生变化组件的视图 UI 也会跟着变化（数据驱动视图）。

function App(){
  // useState(initialState) 是一个函数，initialState 表示传递一个初始值
  // [count, setCount] 中的 count 表示接收初始值状态的变量名称，setCount 表示改变这个状态的方法，这两个变量名可以自定义
  const [ count, setCount ] = React.useState(0)
  return (
    <div>
      <button onClick={()=>setCount(count+1)}>{ count }</button>
    </div>
  )
}

状态的修改规则

在 React 中 状态 被认为是 只读 的，我们应该始终 替换它而不是修改它，直接修改状态不能引发视图更新。

如下图中：

左侧图中，直接通过修改 count 变量，无法引起视图更新，count++ 后，视图绑定的 count 变量仍然是旧值。

右侧图中，通过调用 useState() 返回的 setCount 方法来，传入一个 新的值，替换 count，才会引发视图的变化。

修改对象状态

对于对象类型的状态变量，应该始终给 set 方法一个 全新的对象 来进行修改。

useState 的异步更新机制

注意：useState 返回的 set 方法是异步更新数据的。

function App() {
  const [num, setNum] = useState(0)

  function handleNum() {
      // 第一次点击按钮执行后
      setNum(num++)
      
      // 打印：0，说明 set 方法是异步更新的
      console.log(num)
  }

  return (
    <div className="App">
          {/* 第一次点击按钮后，显示 1 */}
          <span>{ num }</span>
          <button onClick={ () => handleNum() }>点击+1</button>
    </div>
  );
}

React 表单控制

受控表单绑定

概念：使用 React 组件的状态（useState）控制表单的状态。

下图中：

左侧是 React 中组件的状态，右侧是 input 输入框本身的状态，中间的两个箭头方向是这两个状态交互一个流程。

我们先来认识一下什么是 React 的 state 状态，我们在之前学到了 React 的一个 hooks 函数 useState，通过 useState 可以生成一个状态值。这个状态值有一个特点：就是它的变化会引发页面视图的更新。

那什么是 input 输入框的状态呢？我们在基础的 JS 中应该学到过，通过 input 输入框的 input 事件，我们可以通过事件参数 e.target.value 拿到输入框中的值，我们可以简单地理解为 e.target.value 就是输入框本身的状态。

这两个状态之间有什么关系呢？就是 完全由左边的 React 状态来控制右边 input 框的状态的显示。也就说 React 的状态是什么，右侧的 input 框状态就是什么。这个过程就叫做 受控表单绑定。

那么具体该如何实现这个 受控表单绑定 呢？根据上图：

  1. 首先把 React 中的 状态 绑定到 input 中的 value 属性。value 属性决定了 input 框显示什么内容，React 的 state 一旦完成了绑定，相当于是 state 是什么，input 的 value 就是什么。
  2. 反之，当用户在输入框中输入了最新的数据后，我们可以通过 e.target.value 拿到输入的最新数据，并通过 useState 返回的 set 方法把最新的 value 设置给 state，state 一旦变化就会引起页面的重新渲染，又相当于重新走了 state 绑定到 input 的 value 属性这一流程。
  3. 具体代码如下所示：
function App(){
  // 定义了一个 reac 的状态 value
  const [value, setValue] = useState('')
  
  // 把这个 react 的状态绑定到 input 输入框的 value 属性上
  // 一旦 input 输入框的内容发生变化，触发了 onChange 事件，事件中调用 useState 的 set 方法，
  // set 方法改变 react 状态，引起页面视图重新渲染，value 状态又重新绑定到了 input 输入框，
  // 所以，react 状态控制着 input 的状态的显示。
  return (
    <input 
      type="text" 
      value={ value } 
      onChange={ e => setValue(e.target.value) }
    />
  )
}

非受控绑定 - useRef 获取 DOM

概念：通过获取 DOM 的方式获取表单的输入数据。

useRef 是一个 React Hook，它能帮助引用一个不需要渲染的值。使用 ref 操作 DOM 是非常常见的行为。

在 React 组件中，获取/操作 DOM，需要使用 useRef 钩子函数，分为两步：

  1. 使用 useRef 创建 ref 对象，并与 JSX 绑定。
  2. 在 DOM 可用时，通过 ref对象.current 拿到 DOM 对象。
  3. 注意：ref对象.current 需要在 DOM 元素渲染完成后才能获取到。
function App(){
  // 创建 ref 对象
  const inputRef = useRef(null)

  const onChange = ()=>{
    // 通过 ref 对象拿到 input 输入框的值
    console.log(inputRef.current.value)
  }
  
  return (
    {/* 把 ref 对象绑定到 input 元素的 ref 属性上 */}
    <input 
      type="text" 
      ref={ inputRef }
      onChange={ onChange }
    />
  )
}

React 组件通信

概念：组件通信就是 组件之间的数据传递，根据组件嵌套关系的不同，有不同的通信手段和方法。

下图中，左侧是一个完整的 web 页面，右侧就是把左侧的 web 页面中各功能模块拆分成的一个个的组件。

  1. A - B 父子通信
  2. B - C 兄弟通信
  3. A - E 跨层通信

父子通信 - 父传子

上图中，最外层有一个 App 组件，我们把它称之为 父组件，内容有一个 Son 组件，我们把它称之为 子组件。现在在 父组件 中有一个 name 变量，父组件 想要把这个 name 的数据传到 子组件 中，这就是一个 父子通信。

基础实现

**实现步骤 **

  1. 父组件传递数据 - 在 子组件标签上绑定属性名称，这个属性名称可以自定义，并且给该属性绑定父组件要传给子组件的值 。
  2. 子组件接收数据 - 子组件内部通过 props 参数接收数据。
// 定义一个 子组件
// 子组件内部中，可以通过函数的参数接收一个形参，
// 这个 props 包含所有父组件传递过来的属性，
// 这个形参的名称可以自定义，但是一般都叫 props。
function Son(props){
  // 读取属性中的 父组件 传递过来的 name 变量
  return <div>{ props.name }</div>
}

// 定义一个 父组件
function App(){
  // 父组件中的变量数据
  const parentValue = 'this is app name'
  
  return (
    <div>
       {/* 在 子组件 标签上绑定属性，这个属性名称可以自定义 */}
       {/* 给子组件的 name 属性绑定要传递的值，值为 parentValue */}
       <Son name={ name }/>
    </div>
  )
}

props 说明

props 可以传递任意的合法数据，比如数字、字符串、布尔值、数组、对象、函数、JSX...

注意：！！props 是只读对象！！ 子组件只能读取 props 中的数据，不能直接进行修改, 父组件的数据只能由父组件修改 。

特殊的 prop - chilren（插槽）

场景：当我们把内容嵌套在组件的标签内部时，组件会自动在名为 children 的 prop 属性中接收该内容。

父子通信 - 子传父

在上图中，子组件 Son 内有一个 msg 状态，当我们点击 send 按钮的时候，希望可以把 msg 状态传递给 父组件 App。

核心思路：在子组件中调用父组件中的函数并传递参数。

// 子组件
function Son({ onGetMsg }){
  const sonMsg = 'this is son msg'
  return (
    <div>
      {/* 在子组件中执行父组件传递过来的函数，并把 Son 子组件的数据 sonMsg 作为传递传递给 父组件 APP */}
      <button onClick={ () => onGetMsg(sonMsg) }>send</button>
    </div>
  )
}

// 父组件
function App(){
  // 父组件中的函数，用于传递给子组件，参数是接收 子组件 传递过来的数据
  const getMsg = (msg) => console.log(msg)
  
  return (
    <div>
      {/* 传递父组件中的函数到子组件 */}
       <Son onGetMsg={ getMsg }/>
    </div>
  )
}

兄弟组件通信

实现思路：借助 状态提升 机制，通过 共同的父组件 进行兄弟之间的数据传递。

  1. A组件 先通过 子传父 的方式把数据传递给 父组件App。
  2. App 拿到数据之后通过 父传子 的方式再传递给 B组件。
// 需求：把 A 的数据 -> 传递给 B
// 1. 通过子传父 A -> App
// 2. 通过父传子 App -> B

import { useState } from "react"

// 子组件A
function A ({ onGetAName }) {
  // 子组件A 中的数据
  const name = 'this is A name'
  return (
    <div>
      this is A compnent,
          
      {/* 调用 父组件App 的方法，并且把 子组件A 的数组传递给 父组件 */}
      <button onClick={ () => onGetAName(name) }>send</button>
    </div>
  )
}

// 子组件B
function B ({ name }) {
  return (
    <div>
      this is B compnent,
      {name}
    </div>
  )
}

// 父组件App
function App () {
  const [name, setName] = useState('')
  
  // 2. 通过该函数，拿到 子组件A 传递过来的参数
  const getAName = (name) => {
    // 更新父组件的状态 name
    setName(name)
  }
  
  return (
    <div>
      this is App
      {/* 1. 通过给 子组件A 绑定属性，传递父组件的函数 */}
      <A onGetAName={ getAName } />
          
      {/* 3. 把父组件的状态 name 作为属性传递给 子组件B */}
      <B name={ name } />
    </div>
  )
}

export default App

实现步骤：

  2. 在 **顶层组件（App）**中通过 ctx.Provider 组件提供数据。
  3. 在 **底层组件（B）**中通过 useContext 钩子函数获取消费数据。

React 副作用管理 - useEffect

概念理解

useEffect 是一个 React Hook 函数，用于在 React 组件中创建 不是由事件引起而是由渲染本身引起的操作（副作用）, 比如发送 AJAX 请求，更改 DOM 等等。

上面的解释可能有些抽象，我们通过下图进行理解：

在上图中，左侧是一个组件，组件内有两个元素：title 和 article，它是一个未渲染完成的样子。而右侧也是一个组件，左侧组件和右侧组件是同一个组件，只不过右侧组件是已经渲染完成的样子。当这个组件渲染完毕后，会向服务器请求一些数据。

说明：上面的组件中没有发生任何的用户事件，组件渲染完毕之后就需要和服务器要数据，整个过程属于 “只由渲染引起的操作”。

基础使用 - 页面初始化后发送请求

需求：在组件渲染完毕之后，立刻从服务端获取平道列表数据并显示到页面中。

说明：

  1. 参数1 是一个函数，可以把它叫做 副作用函数，在函数内部可以放置要执行的操作。
  2. 参数2 是一个数组（可选参），在数组里放置依赖项，不同依赖项会影响第一个参数函数的执行时机，当是一个空数组的时候，副作用函数只会在组件渲染完毕之后执行一次。

如下代码示例，在组件初始化完成后，发送列表请求：

import React, { useState, useEffect } from 'react';
import axios from 'axios'

interface listType {
  id: number,
  name: string
}

function App() {
  let [list, setList] = useState<listType[]>([])

  useEffect(() => {
    axios.get('http://geek.itheima.net/v1_0/channels')
      .then(res => {
        setList(res.data.data.channels)
      })
  }, [])

  return (
    <div>
      {
        list.map(item => {
          return (
            <div key={ item.id }>
              <span>{ item.name }</span>
            </div>
          )
        })
      }
    </div>
  );
}

export default App;

useEffect 依赖说明

useEffect 副作用函数的执行时机存在多种情况，根据传入 依赖项的不同，会有不同的执行表现

依赖项副作用功函数的执行时机
没有依赖项组件初始渲染 + 组件更新时执行
空数组依赖只在初始渲染时执行一次
添加特定依赖项组件初始渲染 + 依赖项变化时执行

下面根据代码示例分别展示上述另外两种情况：

  • 没有依赖项的情况：
import React, { useState, useEffect } from 'react';

function App() {

  const [count, setCount] = useState(0)

  // 此时 useEffect 针对是的 组件初始化 和 任何原因导致组件重新渲染时，才会执行
  useEffect(() => {
    // 在页面初始化的时候执行了打印语句
    // 并且当前点击了页面的按钮后，也触发了打印语句
    console.log('副作用函数执行了');
  })

  function add() {
    setCount(count + 1)
  }

  return (
    <div>
      <button onClick={ () => add() }>{ count }</button>
    </div>
  );
}

export default App;
  • 添加特定一来的情况

    setCount(count + 1)
  }

    <div>
      <button onClick={ () => add() }>点击</button>
    </div>
  );
}

清除副作用

概念：在 useEffect 中编写的 由渲染本身引起的对接组件外部的操作，社区也经常把它叫做 副作用操作，比如在 useEffect 中开启了一个定时器，我们想在组件卸载时把这个定时器再清理掉，这个过程就是清理副作用

说明：清除副作用的函数最常见的执行时机是在组件卸载时自动执行。

下面代码示例，需求是：在 Son 组件渲染时开启一个定时器，卸载时清除这个定时器：

import { useEffect, useState } from "react"

// 子组件
function Son () {
  // 1. 渲染时开启一个定时器
  useEffect(() => {
    const timer = setInterval(() => {
      console.log('定时器执行中...')
    }, 1000)

    // 组件销毁时触发
    return () => {
      // 清除副作用(组件卸载时)
      clearInterval(timer)
    }
  }, [])
  return <div>this is son</div>
}

// 父组件
function App () {
  // 通过条件渲染模拟组件卸载
  const [show, setShow] = useState(true)
  
  return (
    <div>
      {/* 根据条件是否渲染 子组件 */}
      {show && <Son />}
      <button onClick={() => setShow(false)}>卸载Son组件</button>
    </div>
  )
}

export default App

自定义 Hook 实现

概念：自定义 Hook 是以 use 开头的函数，通过自定义 Hook函数 可以用来 实现逻辑的封装和复用。

按照上图所示，实现以下需求：

  1. 左侧的图显示的是一个组件，组件内有一个盒子和一个按钮，当点击 toggle 按钮的时候，控制 this is div 盒子的显示和隐藏。
  2. 我们分别使用 不封装实现、封装自定义 Hook 实现 以及 抽离实现的通用逻辑。

不封装实现：

// 不封装 Hook
import { useState } from "react"

function App () {
  const [value, setValue] = useState(true)
  const toggle = () => setValue(!value)
  
  return (
    <div>
      { value && <div>this is div</div> }
      <button onClick={ toggle }>toggle</button>
    </div>
  )
}

export default App

封装自定义 Hook 实现：

  • 封装 useToggle.ts 通用 Hooks 函数：
// useToggle.ts 通用 Hooks 函数
import { useState } from "react"

function useToggle () {
  // 可复用的逻辑代码
  const [value, setValue] = useState(true)

  const toggle = () => setValue(!value)

  // 哪些状态和回调函数需要在其他组件中使用 return
  return {
    value,
    toggle
  }
}

export default useToggle
  • 使用该 Hooks 的组件：
import useToggle from './hooks/useToggle' // 引入自定义 hooks

function App () {
  const { value, toggle } = useToggle()
  
  return (
    <div>
      { value && <div>this is div</div> }
      <button onClick={ toggle }>toggle</button>
    </div>
  )
}

export default App

封装 自定义 Hook 通用思路：

  1. 声明一个以 use 开头的函数。
  2. 在函数体内封装可复用的逻辑（只要是可复用的逻辑）。
  3. 把组件中用到的 状态 或者 回调 return 出去（以对象或者数组）。
  4. 在哪个组件中要用到这个逻辑，就执行这个函数，解构出来状态和回调进行使用

React Hooks 使用规则

  1. 只能在组件中或者其他自定义 Hook 函数中调用。
  2. 只能在组件的顶层调用，不能 嵌套在 if、for、其它的函数中。
  3. 必须以 “use” 开头吗？必须如此。这个约定非常重要。不遵循的话，由于无法判断某个函数是否包含对其内部 Hook 的调用，React 将无法自动检查你的 Hook 是否违反了 Hook 的规则。

// 封装高阶组件 -- 权限路由组件
// 核心逻辑：有 token 正常跳转，无 token 去登陆页面

import { getToken } from "@/utils";

/**
 * 权限路由组件
 * @param props 接收的子组件
 * @returns 组件

function AuthRoute(props: any) {
  const childrenComponent = props.children
  const token = getToken()

  if (token) return childrenComponent
  else {
    // 重定向到登录
    // replace 表示替换之前的路由记录
    return <Navigate to={'/login'} replace />
  }
}

useReducer

作用：和 useState 的作用类似，用来管理 相对复杂 的状态数据。

基础使用

  1. 定义一个 reducer 函数（根据不同的 action 返回不同的新状态）。
  2. 在组件中调用 useReducer，并传入 reducer 函数和 状态的初始值。
  3. 事件发生时，通过 dispatch 函数分派一个 action 对象（通知 reducer 要返回哪个新状态并渲染UI）
import { useReducer } from "react"

type Action = {
  type: string
}

// 1. 定义 reducer 函数，根据不同的 action 返回不同的新状态
function reducer(state: number, action: Action) {
  switch (action.type) {
    case 'INC':
      return state + 1
    case 'DEC':
      return state - 1
    default:
      return state
  }
}

function Layout () {
  // 2. 使用 useReducer 分派 action
  const [state, dispatch] = useReducer(reducer, 0)
  return (
    <div>
      {/* 3. 调用 dispatch 函数传入 action 对象 触发 reducer 函数，分派 action 操作，使用新状态更新视图 */}
      <button onClick={ () => dispatch({ type: 'DEC' }) }>-</button>
      {state}
      <button onClick={ () => dispatch({ type: 'INC' }) }>+</button>
    </div>
  )
}

useMemo

作用：它在每次重新渲染的时候能够缓存计算的结果。

看个场景

上图中，我们本来的用意是想 基于 count1 的变化计算斐波那契数列之和，但是当我们修改 count2 状态的时候，斐波那契求和函数也会被执行，显然是一种浪费。

useMemo 的基础语法

// 第一个参数是函数，并且是纯函数，必须要有返回值，返回计算后的结果。
// 第二个参数是数组，表示依赖哪些数据，只有这里的数据发生变化了，第一个参数中的回调才会重新运行。
useMemo(() => {
  // 做一些处理，返回计算之后的结果
  // count1 变化了才会重新执行
}, [count1])

说明：

  • useMemo 接收的第一个参数是 纯函数。
  • 使用 useMemo 做缓存之后可以保证只有 count1 依赖项发生变化时，才会重新运行第一个参数中的函数进行计算。否则永远使用的都是之前缓存的计算结果。

useMemo 缓存计算结果

// useMemo
// 作用：在组件渲染时缓存计算的结果

import { useMemo, useState } from 'react'

// 计算斐波那契数列之和
function fib(n: number): number {
  console.log('斐波那契函数执行了')
  return n <= 0 ? 1 : n * fib(n - 1)
}

function App() {
  const [count1, setCount1] = useState(0)
  const [count2, setCount2] = useState(0)

  // 计算斐波那契之和
  // 我们可以发现，不管是 count1 发生变化，还是 count2 发生变化
  // fib 函数都会重新运行。
  // const result = fib(count1)

  // 通过使用 useMemo 来得到缓存的结果，只有当 count1 发生变化才会重新计算
  const result =useMemo(() => {
    return fib(count1)
  }, [count1])

  return (
    <div>
      { result }
      <button onClick={() => setCount1(count1 + 1)}> count1 ++：{count1} </button>
      <button onClick={() => setCount2(count2 + 1)}> count2 ++：{count2}</button>
    </div>
  )
}

import React, { useMemo, useState } from 'react'

  // 返回一个固定的 [1, 2, 3] 引用数据，它不随着依赖项的变化而

}

useCallback

作用：在组件多次重新渲染的时候缓存函数。

看个场景

上一小节我们说到，当给子组件传递一个 引用类型 prop 的时候，即使我们使用了 memo 函数 依旧无法阻止 子组件 的渲染，其实传递 prop 的时候，往往传递一个回调函数更为常见，比如实现子传父，此时如果想要避免子组件渲染，可以使用 useCallback 缓存 回调函数。

先看下以下不用 useCallback 缓存 的例子：

// useCallBack

import { memo, useState } from 'react'

const MemoSon = memo(function Son(props: any) {
  return (
    <div>{ props.onGetSonMessage('子组件重新渲染了') }</div>
  )
})

function App() {
  const [count, setCount] = useState(0)
  console.log('父组件重新渲染了')

  const onGetSonMessage = (message: string) => {
    console.log(message)
  }

  return (
    <div>
      父组件 -- { count }
      <MemoSon onGetSonMessage={onGetSonMessage} />
      <button onClick={() => setCount(Math.random())}>update</button>
    </div>
  )
}

export default App

从上可以发现，即使 父组件 传递了一个 引用数据类型的函数给 子组件，每次 父组件 的状态发生变化，子组件 都会重新渲渲染。

useCallback 的基本使用

基本语法：

useCallback((value) => {
    // ... doing somthing
}, [])

说明：

  • useCallback 会返回一个新的具有缓存的函数。

  • useCallback 的第一个参数是要缓存的函数。

  • useCallback 的第二个参数是依赖项数组，当数组内容的某个依赖项发生变化，会重新执行 useCallback。传递空数组表示只执行一次。

使用 useCallback 缓存函数

说明：useCallback 包裹函数之后，函数可以保证在 App 重新渲染的时候保持引用稳定，也就是返回同一个引用。

// useCallBack

import { memo, useCallback, useState } from 'react'

const MemoSon = memo(function Son(props: any) {
  return (
    <div>{ props.onGetSonMessage('子组件重新渲染了') }</div>
  )
})

function App() {
  const [count, setCount] = useState(0)
  console.log('父组件重新渲染了')

  // 使用 useCallback 缓存函数
  // 第一个参数为要缓存的函数
  // 第二个参数是一个空数组，表示不需要更新
  const onGetSonMessage = useCallback((message: string) => {
    console.log(message)
  }, []) 

  return (
    <div>
      父组件 -- { count }
      <MemoSon onGetSonMessage={onGetSonMessage} />
      <button onClick={() => setCount(Math.random())}>update</button>
    </div>
  )
}

export default App

从上可以看出，每次 父组件 的状态发生变化时，子组件 都不会重新渲染，因为传递的函数都是上一个缓存的函数 引用。

    // 通过 setState 方法修改组件内的状态
    this.setState({
      count: this.state.count + 1
    })
  }

  // render 方法渲染模板页面
  render() {
    return (
      <div>
        { this.state.count }
        <button onClick={ () => this.clickHandler() } >+1</button>
      </div>
    )
  }
}

类组件的组件通信

概念：类组件和 Hooks 编写的组件在组件通信的思想上完全一致。

  1. 父传子：通过 prop 绑定数据。
  2. 子传父：通过 prop 绑定父组件中的函数，子组件调用。
  3. 兄弟通信：状态提升，通过 父组件 做桥接。

父传子 的示例：

import { Component } from "react";

type Props = {
  msg: string
}

// 子组件
class Son extends Component<Props> {
  render() {
    return (

  }

子传父 的示例：

import { Component } from "react";

type Props = {
  msg: string
  getMsg: (msg: string) => void
}

// 子组件
class Son extends Component<Props> {
  render() {
    return (
      <div>
        <div>子组件</div>
        <span>接收父组件的消息：{ this.props.msg }</span>
        <button onClick={ () => this.props.getMsg('子组件的消息') }>点击向父组件传递消息</button>
      </div>
    )
  }
}

class Parent extends Component {
  state = {
    message: '父组件的消息',
    childMsg: ''
  }

  getMsg = (msg: string) => {
    this.setState({
      childMsg: msg
    })
  }

  render() {
    return (
      <div>
        <span>父组件</span>
        <span>接收子组件的消息：{ this.state.childMsg }</span>
        {/* 父组件 向 子组件 传递数据 */}
        <Son msg={ this.state.message } getMsg={ this.getMsg } />
      </div>
    )
  }
}

export default Parent

React 和 TypeScript

useState 自动推导

简单场景

通常 React 会根据传入的 useState 的默认值来自动推导 TS 类型，不需要显式标注类型。

使用场景：一般适用于 明确的初始值。

import { useState } from "react"

function App () {
  // val 会被自动推断为布尔类型
  // setVal 方法调用时只能传入布尔类型
  const [val, setVal] = useState(false)

  setVal(true) // 调用时就只能传入 boolean 类型
  // setVal(100) // 报错

  // list 会被自动推导为 number[] 类型
  const [list, setList] = useState([1, 2, 3])

  setList([4, 5, 6]) // 调用时，只能传入 number[] 类型
  // setList(['1', '2', '3']) // 报错

  return (
    <div>APP</div>
  )
}

export default App

复杂场景

useState 本身是一个 泛型函数，可以传入具体的 自定义类型。

复杂数据类型，useState 支持通过 泛型参数 指定初始参数类型以及 setter 函数的入参类型。

import { useState } from "react"

function App () {
  // 定义 User 数据类型  
  type User = {
    name: string
    age: number
  }

  // 传入一个泛型，并限制数据类型为 User
  const [user, setUser] = useState<User>()

  setUser({
    name: '张三',
    age: 18
  })

  return <div>APP</div>
}

export default App

说明：

  1. 限制 useState 函数参数的 初始值 必须满足对应 User 类型。
  2. 限制 setUser 函数的 参数 必须满足对应 User 类型。
  3. 使 user 状态具有 User 类型的相关提示。

没有具体默认值

实际开发时，有些时候 useState 的初始值可能为 null 或者 undefined，按照 泛型 的写法是不能通过类型校验的，此时可以通过完整的 类型联合 null 或者 undefined 类型即可。

import { useState } from "react"

function App () {
  type User = {
    name: string
    age: number
  }

  const [user, setUser] = useState<User | null>(null)

  setUser({
    name: '张三',
    age: 18
  })

  return (
    <div>APP</div>
  )
}

export default App

说明：

  1. 限制 useState 函数参数的初始值可以是 User | null。
  2. 限制 setUser 函数的参数类型可以是 User | null。

type Props = {

const Button = (props: Props) => {

useRef 和 TypeScript

在 TypeScript 的环境下，useRef 函数返回一个 只读 或者 可变 的 引用，只读 的场景常见于 获取真实 dom，可变 的场景，常见于 缓存一些数据，不跟随组件渲染，下面分俩种情况说明。

获取 DOM

获取 DOM 时，通过 泛型参数 指定具体的 DOM元素类型 即可。

import { useEffect, useRef } from "react"

function App () {
  // 尽可能提供一个具体的 dom type, 可以帮助我们在用 dom 属性时有更明确的提示
  // inputRef 的类型为 HTMLInputElement
  const inputRef = useRef<HTMLInputElement>(null)
  
  useEffect(() => {
    inputRef.current && inputRef.current.focus()
  }, [])

  return (
    <div>
      <input type="text" ref={ inputRef } />
    </div>
  )
}

export default App如果你可以确保`divRef.current` 不是null，也可以在传入初始值的位置

说明：如果想要获取 div 的类型则是 HTMLDivElement，如果想要获取 span 的类型，则是 HTMLSpanElement，以此类推。

稳定引用存储器

useRef 是一个 React Hook，它能帮助你操作 DOM，也可以引用一个不需要渲染的值。

把 useRef 当做为 可变存储容器（不需要渲染的值） 使用的时候，可以通过 泛型参数 指定容器存入的 数据类型，在还未存入实际内容时通常把 null 作为 初始值，所以依旧可以通过联合类型做指定，比如定时器管理的场景，如下所示：

import { useEffect, useRef } from "react"

function App () {
  const timerRef = useRef<number | undefined | NodeJS.Timer>(undefined)
  
  useEffect(() => {
    timerRef.current = setInterval(() => {
      console.log('1');
    }, 1000)

    return () => clearInterval(timerRef.current)
  }, [])

  return (
    <div>App</div>
  )
}

export default App

封装 API 接口 - axios 和 ts 的配合使用

场景：axios 提供了 request 泛型方法，方便我们传入类型参数推导出接口返回值的类型。

![axios.request 泛型方法](assets/axios.request 泛型方法.png)

说明：泛型参数 Type 的类型决定了 res.data 的类型。这里的 Type 是我们自己根据接口文档的返回内容自己定义的。

步骤：

  1. 根据接口文档创建 通用的泛型接口类型（多个接口返回值的结构是相似的）。
  2. 根据接口文档创建 特有的接口数据类型（每个接口有自己特殊的数据格式）。
  3. 组合 1 和 2 的类型，得到最终传给 request泛型 的 参数类型。

根据上图，我们来定义一下对应的数据结构类型：

定义 通用的响应数据结构 类型：

type Response<T> = {
  message: string
  data: T
}

export {
  Response

然后定义 列表的数据结构类型：

  name: string

  channels: ChannelItem[]

封装获取列表的 API 接口，并 组合通用的响应数据类型 和 列表特有的数据结构 限制返回的数据结构：

import { http } from '@/utils/index'
import type { Channels } from './types'
import type { Response } from '../types'

// 请求频道列表
function fetchChannelsAPI() {
  return http.request<Response<Channels>>({
    url: '/v1_0/channels',
    method: 'get'
  })
}

export {
  fetchChannelsAPI
}

注意：通过 axios.request<T>() 形式传入的泛型参数，在封装配置 axios 的相应拦截器时，需要直接返回 response，而不是返回 response.data，否则类型会判断错误。

然后在使用时，通过 res.data. 就可以关联对应的数据类型：

Tips：如果不想在调用接口获取接口返回的数据时，通过 res.data.xxx 去拿接口返回的数据，而是直接通过 res.xxx 去拿到对应的接口返回的数据时，可以在封装 API 时，改变泛型的传递方式，把 axios.request<T>() 改成 axios.post<any, T>()：

import { http } from '@/utils/index'
import type { Channels } from './types'
import type { Response } from '../types'

// 请求频道列表
function fetchChannelsAPI() {
  return http.request<any, Response<Channels>>({
    url: '/v1_0/channels',
    method: 'get'
  })
}

export {
  fetchChannelsAPI
}

并且，封装的 axios 中的 响应拦截器中，要返回 response.data：

import axios from 'axios'
import type { AxiosResponse } from 'axios'

// 创建一个 axios 实例
const httpInstance = axios.create({
  baseURL: 'http://geek.itheima.net',
  timeout: 5000
})

// 请求拦截器
httpInstance.interceptors.request.use(
  config => {
    return config
  },
  error => {
    return Promise.reject(error)
  }
)

// 相应拦截器
httpInstance.interceptors.response.use(
  (response: AxiosResponse) => {
    return response.data
  },
  error => {
    return Promise.reject(error)
  }
)

export default httpInstance

AxiosResponse 的类型如下：

路由懒加载

什么是 路由懒加载？

路由懒加载 是 指路由的 JS资源 只有在被访问时才会动态获取，目的是为了优化项目首次打开的时间。

从上图可以看到，当前正在预览的页面是首页，在没有没使用 路由懒加载前，所有菜单路由的 JS 资源都会加载，而使用了 路由懒加载后，只会加载当前被访问的路由的 JS资源。

如何配置路由懒加载？

  1. 把路由修改为由 React 提供的 lazy 函数 进行动态导入。
  2. 使用 React 内置的 Suspense组件 包裹路由中 element 选项对应的组件。
import { createBrowserRouter } from 'react-router-dom';
// import Home from '@/pages/Home'
import { Suspense, lazy } from 'react';
import Loading from '@/components/Loading'; // 路由页面加载组件

// react 的 lazy api 能够让你在组件第一次被渲染之前延迟加载组件的代码（路由懒加载）
const Home = lazy(() => import('@/pages/Home'))

const router = createBrowserRouter([
  // 使用 路由懒加载前 的编写方式
  // {
  //   path: '/',
  //   element: <Home />
  // },
  // 
  {
    path: '/',
    // Suspense 是 React 的内部组件，可以提供异步渲染的能力。
    // Suspense 可以接收一个 fallback 属性，因为现在的路由是一个异步的路由，
    // 所以，fallback 是提供一个占位，相当于是当前组件渲染之前，要显示什么内容。
    // fallback 可以接收一个组件。
    element: <Suspense fallback={<Loading />}><Home /></Suspense>
  }
])

export default router

CDN优化

什么是 CDN？

CDN 是一种内容分发网络服务，当用户请求网站内容时，由 离用户最近 的服务器将缓存的资源内容传递给用户。

哪些资源可以放到 CDN 服务器？体积较大的 非业务 JS 文件, 比如 react、react-dom 等。

  1. 体积较大，需要利用 CDN 文件在浏览器的 缓存特性，加快加载时间。
  2. 非业务 JS 文件，不需要经常做变动，CDN 不用频繁更新缓存。

项目中怎么做？

  1. 把需要做 CDN缓存 的文件 排除在打包之外（react、react-dom）。

使用 craco 来修改项目的 webpack配置， 在根目录的 craco.config.js 文件中：

const path = require('path')
// 引入辅助函数
const { whenProd, getPlugin, pluginByName } = require('@craco/craco')

module.exports = {
  // webpack 配置
  webpack: {
    // 配置别名
    alias: {
      "@": path.resolve(__dirname, 'src')
    },

    // 配置CDN
    configure: (webpackConfig) => {
      // 定义一个 cdn 变量，用于存储现成资源的
      let cdn = {
        js: []
      }

      // whenProd 是一个辅助函数，用来判断当前是否为生产环境
      // 只有生产环境时，才做这一套配置。
      whenProd(() => {
        // webpackConfig.externals 表示排除打包文件。排除掉的打包文件通过 CDN 引入。
        // key: 不参与打包的包（由 dependencies依赖项中的 key 决定）
        // value: cdn 文件中 挂载于全局的变量名称 为了替换之前在开发环境下
        webpackConfig.externals = {
          react: 'React',
          'react-dom': 'ReactDOM'
        }
        // 配置现成的 cdn 资源地址，也就是要引入的资源文件
        // 这里用的是免费的资源文件，
        // 实际开发的时候 用公司自己花钱买的cdn服务器
        cdn = {
          js: [
            'https://cdnjs.cloudflare.com/ajax/libs/react/18.1.0/umd/react.production.min.js',
            'https://cdnjs.cloudflare.com/ajax/libs/react-dom/18.1.0/umd/react-dom.production.min.js',
          ]
        }
      })

      // 通过 htmlWebpackPlugin 插件 在 public/index.html 注入 cdn 资源 url
      // getPlugin 方法表示要找某个插件，这里表示要找 HtmlWebpackPlugin 插件。
      const { isFound, match } = getPlugin(
        webpackConfig,
        pluginByName('HtmlWebpackPlugin')
      )

      // 如果找到了 HtmlWebpackPlugin 的插件
      if (isFound) {
        // 修改它的配置项，把我们之前声明好的 CDN 资源放到配置项里边
        // match.userOptions.cdn = cdn
          
        // 视频教程里是按照上面的 userOptions 进行添加 cdn 的，
        // 但是，在本地运行和打包都会报错，index.html 无法找到 options.cdn.js，
        // 所以，这里添加到 options 的 cdn 中。
        match.options.cdn = cdn
      }

      return webpackConfig
    }
  }
};

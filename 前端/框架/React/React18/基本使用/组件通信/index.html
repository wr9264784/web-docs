组件通信

概念：组件通信就是 组件之间的数据传递，根据组件嵌套关系的不同，有不同的通信手段和方法。

下图中，左侧是一个完整的 web 页面，右侧就是把左侧的 web 页面中各功能模块拆分成的一个个的组件。

  1. A - B 父子通信
  2. B - C 兄弟通信
  3. A - E 跨层通信

父子通信 - 父传子

上图中，最外层有一个 App 组件，我们把它称之为 父组件，内容有一个 Son 组件，我们把它称之为 子组件。现在在 父组件 中有一个 name 变量，父组件 想要把这个 name 的数据传到 子组件 中，这就是一个 父子通信。

基础实现

  1. 父组件传递数据 - 在 子组件标签上绑定属性名称，这个属性名称可以自定义，并且给该属性绑定父组件要传给子组件的值 。
  2. 子组件接收数据 - 子组件内部通过 props 参数接收数据。
// 定义一个 子组件
// 子组件内部中，可以通过函数的参数接收一个形参，
// 这个 props 包含所有父组件传递过来的属性，
// 这个形参的名称可以自定义，但是一般都叫 props。
function Son(props){
  // 读取属性中的 父组件 传递过来的 name 变量
  return <div>{ props.name }</div>
}

// 定义一个 父组件
function App(){
  // 父组件中的变量数据
  const parentValue = 'this is app name'
  
  return (
    <div>
       {/* 在 子组件 标签上绑定属性，这个属性名称可以自定义 */}
       {/* 给子组件的 name 属性绑定要传递的值，值为 parentValue */}
       <Son name={ name }/>
    </div>
  )
}

props 说明

props 可以传递任意的合法数据，比如数字、字符串、布尔值、数组、对象、函数、JSX...

注意：！！props 是只读对象！！ 子组件只能读取 props 中的数据，不能直接进行修改, 父组件的数据只能由父组件修改 。

特殊的 prop - chilren（插槽）

场景：当我们把内容嵌套在组件的标签内部时，组件会自动在名为 children 的 prop 属性中接收该内容。

父子通信 - 子传父

在上图中，子组件 Son 内有一个 msg 状态，当我们点击 send 按钮的时候，希望可以把 msg 状态传递给 父组件 App。

核心思路：在子组件中调用父组件中的函数并传递参数。

// 子组件
function Son({ onGetMsg }){
  const sonMsg = 'this is son msg'
  return (
    <div>
      {/* 在子组件中执行父组件传递过来的函数，并把 Son 子组件的数据 sonMsg 作为传递传递给 父组件 APP */}
      <button onClick={ () => onGetMsg(sonMsg) }>send</button>
    </div>
  )
}

// 父组件
function App(){
  // 父组件中的函数，用于传递给子组件，参数是接收 子组件 传递过来的数据
  const getMsg = (msg) => console.log(msg)
  
  return (
    <div>
      {/* 传递父组件中的函数到子组件 */}
       <Son onGetMsg={ getMsg }/>
    </div>
  )
}

兄弟组件通信

实现思路：借助 状态提升 机制，通过 共同的父组件 进行兄弟之间的数据传递。

  1. A组件 先通过 子传父 的方式把数据传递给 父组件App。
  2. App 拿到数据之后通过 父传子 的方式再传递给 B组件。
// 需求：把 A 的数据 -> 传递给 B
// 1. 通过子传父 A -> App
// 2. 通过父传子 App -> B

import { useState } from "react"

// 子组件A
function A ({ onGetAName }) {
  // 子组件A 中的数据
  const name = 'this is A name'
  return (
    <div>
      this is A compnent,
          
      {/* 调用 父组件App 的方法，并且把 子组件A 的数组传递给 父组件 */}
      <button onClick={ () => onGetAName(name) }>send</button>
    </div>
  )
}

// 子组件B
function B ({ name }) {
  return (
    <div>
      this is B compnent,
      {name}
    </div>
  )
}

// 父组件App
function App () {
  const [name, setName] = useState('')
  
  // 2. 通过该函数，拿到 子组件A 传递过来的参数
  const getAName = (name) => {
    // 更新父组件的状态 name
    setName(name)
  }
  
  return (
    <div>
      this is App
      {/* 1. 通过给 子组件A 绑定属性，传递父组件的函数 */}
      <A onGetAName={ getAName } />
          
      {/* 3. 把父组件的状态 name 作为属性传递给 子组件B */}
      <B name={ name } />
    </div>
  )
}

export default App

跨层组件通信

实现步骤：

  1. 使用 createContext 方法创建一个上下文对象 ctx，这个变量名可以自定义。
  2. 在 顶层组件（App） 中通过 ctx.Provider 组件提供数据。
  3. 在 底层组件（B） 中通过 useContext 钩子函数获取消费数据。

注意：这里并不是说只有祖孙组件之间才能用 上下文对象 进行通信。

如右图所示：在 组件树 里面其实可以形成为 N 个顶层和底层的关系，其中只要存在有嵌套的关系，就可以使用这套机制来完成通信。比方说：父子组件、爷孙组件。

// 顶层组件APP
import React, { createContext } from 'react';
import Acom from './components/A'

const msg = 'this is app msg'
const MsgCtx = createContext(msg) // 创建上下文对象

function App() {
  const msg = 'this is app msg'
  
  return (
    <div>
      {/* 通过上下文对象提供的 Provider 组件，提供全局变量 */}
      {/* Provider 组件绑定属性名称 value 是固定的 */}
      <MsgCtx.Provider value={ msg }>
        这是 APP组件
        <Acom />
      </MsgCtx.Provider>
    </div>
  );
}

export default App;
export { MsgCtx } // 还需要导出上下文对象
// 子组件A
import React, { useState } from 'react';
import Bcom from './B' // 导入 孙子组件B

function Acom() {
  
  return (
    <div>
      这是 组件A
          
      {/* 调用 孙子组件B */}
      <Bcom />
    </div>
  );
}

export default Acom;
// 孙子组件B
import React, { useState, useContext } from 'react';
import { MsgCtx } from '../App'

function Bcom() {
  
  return (
    <div>
      这是B
      <div>
        get app msg: { useContext(MsgCtx) }
      </div>
    </div>
  );
}

export default Bcom;

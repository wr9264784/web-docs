React 和 TypeScript

useState 自动推导

简单场景

通常 React 会根据传入的 useState 的默认值来自动推导 TS 类型，不需要显式标注类型。

使用场景：一般适用于 明确的初始值。

import { useState } from "react"

function App () {
  // val 会被自动推断为布尔类型
  // setVal 方法调用时只能传入布尔类型
  const [val, setVal] = useState(false)

  setVal(true) // 调用时就只能传入 boolean 类型
  // setVal(100) // 报错

  // list 会被自动推导为 number[] 类型
  const [list, setList] = useState([1, 2, 3])

  setList([4, 5, 6]) // 调用时，只能传入 number[] 类型
  // setList(['1', '2', '3']) // 报错

  return (
    <div>APP</div>
  )
}

export default App

复杂场景

useState 本身是一个 泛型函数，可以传入具体的 自定义类型。

复杂数据类型，useState 支持通过 泛型参数 指定初始参数类型以及 setter 函数的入参类型。

import { useState } from "react"

function App () {
  // 定义 User 数据类型  
  type User = {
    name: string
    age: number
  }

  // 传入一个泛型，并限制数据类型为 User
  const [user, setUser] = useState<User>()

  setUser({
    name: '张三',
    age: 18
  })

  return <div>APP</div>
}

export default App

说明：

  1. 限制 useState 函数参数的 初始值 必须满足对应 User 类型。
  2. 限制 setUser 函数的 参数 必须满足对应 User 类型。
  3. 使 user 状态具有 User 类型的相关提示。

没有具体默认值

实际开发时，有些时候 useState 的初始值可能为 null 或者 undefined，按照 泛型 的写法是不能通过类型校验的，此时可以通过完整的 类型联合 null 或者 undefined 类型即可。

import { useState } from "react"

function App () {
  type User = {
    name: string
    age: number
  }

  const [user, setUser] = useState<User | null>(null)

  setUser({
    name: '张三',
    age: 18
  })

  return (
    <div>APP</div>
  )
}

export default App

说明：

  1. 限制 useState 函数参数的初始值可以是 User | null。
  2. 限制 setUser 函数的参数类型可以是 User | null。

props 和 TypeScript

基本使用

为组件 prop 添加类型，本质是给 函数的参数做类型注释，可以使用 type 对象类型 或者 interface 接口 来做注解。

type Props = {
  type: string
}

const Button = (props: Props) => {
  return (
    <button className={ props.type === 'primary' ? 'primary' : '' }>
      click me
    </button>
  )
}

function App () {
  return (
    <div>
      <Button type="primary"></Button>
    </div>
  )
}

export default App

为 children 添加类型

children 是一个比较特殊的 prop，支持多种不同类型数据的传入，需要通过一个 内置的 ReactNode 类型 来做注解。

import { ReactNode } from "react"

type Props = {
  type: string
  children: ReactNode // 给子组件的插槽限制类型
}

const Button = (props: Props) => {
  return (
    <button className={ props.type === 'primary' ? 'primary' : '' }>
      { props.children }
    </button>
  )
}

function App () {
  return (
    <div>
      <Button type="primary">
        {/* 向子组件的插槽中插入标签 */}
        <span>点击按钮</span>
      </Button>
    </div>
  )
}

export default App

说明：添加 children 注解之后，children 可以是多种类型，包括：React.ReactElement、string、number、React.ReactFragment、React.ReactPortal、boolean、null、undefined。

为事件 prop 添加类型

组件经常执行类型为函数的 prop 实现 子传父，这类 prop 重点在于 函数参数类型的注解。

type Props = {
  onGetMsg?: (msg: string) => void // 限制事件 prop 的类型
}

const Button = (props: Props) => {

  function clickHandle() {
    props.onGetMsg && props.onGetMsg('子组件的消息')
  }

  return (
    <button onClick={ () => clickHandle() }>点击</button>
  )
}

function App () {
  function onGetMsg(msg: string) {
    console.log(msg);
  }
  
  return (
    <div>
      <Button onGetMsg={ onGetMsg }></Button>
    </div>
  )
}

export default App

说明：

  1. 在组件内部调用时需要遵守类型的约束，参数传递 需要满足要求。
  2. 绑定 prop 时如果绑定内联函数直接可以推断出参数类型，否则需要单独注解匹配的参数类型。

useRef 和 TypeScript

在 TypeScript 的环境下，useRef 函数返回一个 只读 或者 可变 的 引用，只读 的场景常见于 获取真实 dom，可变 的场景，常见于 缓存一些数据，不跟随组件渲染，下面分俩种情况说明。

获取 DOM

获取 DOM 时，通过 泛型参数 指定具体的 DOM元素类型 即可。

import { useEffect, useRef } from "react"

function App () {
  // 尽可能提供一个具体的 dom type, 可以帮助我们在用 dom 属性时有更明确的提示
  // inputRef 的类型为 HTMLInputElement
  const inputRef = useRef<HTMLInputElement>(null)
  
  useEffect(() => {
    inputRef.current && inputRef.current.focus()
  }, [])

  return (
    <div>
      <input type="text" ref={ inputRef } />
    </div>
  )
}

export default App如果你可以确保`divRef.current` 不是null，也可以在传入初始值的位置

说明：如果想要获取 div 的类型则是 HTMLDivElement，如果想要获取 span 的类型，则是 HTMLSpanElement，以此类推。

稳定引用存储器

useRef 是一个 React Hook，它能帮助你操作 DOM，也可以引用一个不需要渲染的值。

把 useRef 当做为 可变存储容器（不需要渲染的值） 使用的时候，可以通过 泛型参数 指定容器存入的 数据类型，在还未存入实际内容时通常把 null 作为 初始值，所以依旧可以通过联合类型做指定，比如定时器管理的场景，如下所示：

import { useEffect, useRef } from "react"

function App () {
  const timerRef = useRef<number | undefined | NodeJS.Timer>(undefined)
  
  useEffect(() => {
    timerRef.current = setInterval(() => {
      console.log('1');
    }, 1000)

    return () => clearInterval(timerRef.current)
  }, [])

  return (
    <div>App</div>
  )
}

export default App

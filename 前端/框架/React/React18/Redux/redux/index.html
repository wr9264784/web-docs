Redux 的使用

Redux 介绍

Redux 是 React 最常用的集中状态管理工具，类似于 Vue 中的 Pinia（Vuex），可以独立于框架运行。 作用：通过集中管理的方式管理应用的状态。

为什么要使用 Redux？

  1. 独立于组件，无视组件之间的层级关系，简化通信问题。
  2. 单项数据流清晰，易于定位 bug。
  3. 调试工具配套良好，方便调试。

Redux 快速体验

1. 实现计数器

需求：不和任何框架绑定，不使用任何构建工具，使用 纯Redux 实现计数器。

使用步骤：

  1. 定义一个 reducer 函数 （根据当前想要做的修改返回一个新的状态）。
  2. 使用 createStore 方法传入 reducer 函数 生成一个 store 实例对象。
  3. 使用 store 实例 的 subscribe 方法 订阅数据的变化（数据一旦变化，可以得到通知）。
  4. 使用 store 实例 的 dispatch 方法 提交 action 对象 触发数据变化（告诉 reducer 你想怎么改数据）。
  5. 使用**store 实例 **的 getState 方法 获取最新的状态数据更新到视图中。

代码实现：

<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Redux</title>
</head>

<body>
  <button id="decrement">-</button>
  <span id="count">0</span>
  <button id="increment">+</button>

  <!-- 下载地址：https://unpkg.com/browse/redux@4.2.0 -->
  <script src="./redux.js"></script>
  <script>
    /**
     * 1. 定义 reducer 函数
     * 作用：根据不同的 action 对象，返回不同的 "新" 的 state
     * 
     * 第一个参数是状态，这个状态就是初始的全局状态。
     * 
     * 第二个参数是 action 对象：这个对象身上有一个属性 type，
     * 这个 type 用来标记当前想要做什么样的修改。
     */
    function reducer(state = { count: 0 }, action) {
      /**
       * 为什么这里根据 action 的 type 属性，要返回一个新的对象呢？
       * 而不是在 state 的 count 里直接 +1 或 -1 呢？因为这里要求：
       * 叫作 "数据不可变"，我们必须基于原始状态生成一个 "新" 的状态。
       */

      // type 为 INCREMENT，则新增，type 为何值可以自定义
      if (action.type === 'INCREMENT') {
        // 拿到全局状态中的 count 进行 +1
        return { count: state.count + 1 }
      }

      // type 为 DECREMENT，则减少
      if (action.type === 'DECREMENT') {
        // 拿到全局状态中的 count 进行 -1
        return { count: state.count - 1 }
      }

      return state
    }

    // 2. 使用 Redux 的 createStore 函数，并且传入 reducer 生成 store 实例。
    const store = Redux.createStore(reducer)

    /**
     * 3. 通过 store 实例的 subscribe 订阅数据的变化
     * 回调函数会在每次 state 发生变化的时候自动执行
     */
    store.subscribe(() => {
      console.log('state 变化了', store.getState());
      document.querySelector('#count').innerHTML = store.getState().count
    })

    /**
     * 4. 通过 store 实例的 dispatch 函数提交 action 更改状态
     * 在 redux 中，修改状态的唯一方式就是调用 dispatch 函数，
     * 并提交一个 action 对象。
     */
    const inBtn = document.getElementById('increment')
    inBtn.addEventListener('click', () => {
      // 新增
      store.dispatch({
        type: 'INCREMENT'
      })
    })

    const deBtn = document.getElementById('decrement')
    deBtn.addEventListener('click', () => {
      // 较少
      store.dispatch({
        type: 'DECREMENT'
      })
    })
  </script>
</body>

</html>

为了职责清晰，Redux 代码被分为 三个核心 的概念，我们学 redux，其实就是学这三个核心概念之间的配合，三个概念分别是：

  1. state: 一个对象 存放着我们管理的数据。
  2. action: 一个对象 用来描述你想怎么改数据。
  3. reducer: 一个函数 根据 action 的描述更新 state。

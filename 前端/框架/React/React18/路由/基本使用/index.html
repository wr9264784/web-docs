路由快速上手

1. 什么是前端路由

一个路径 path 对应一个 组件 component 当我们在浏览器中访问一个 path 的时候，path 对应的 组件 会 在页面中进行渲染，如下图所示：

上图中，routes 表示一个路由列表，里边有两个路由，分别是 /about 和 /article，这两个路由对应的 组件 分别是 About 和 Article。

2. 创建路由开发环境

# 使用 CRA 创建项目
npm create-react-app react-router-pro

# 安装最新的 ReactRouter 包
npm i react-router-dom

# 启动项目
npm run start

3. 快速开始

需求：创建一个可以切换 登录 和 文章 页面的路由系统：

在 /src/index.tsx 中：

import React from 'react'
import ReactDOM from 'react-dom/client'

const root = ReactDOM.createRoot(
  document.getElementById('root') as HTMLElement
);

// 路由列表
const router = createBrowserRouter([
  {
    path:'/login',
    element: <div>登录</div> // 这里直接渲染模板标签
  },
  {
    path:'/article',
    element: <div>文章</div>
  }
])

root.render(
  // React.StrictMode 是 react 中的严格模式节点
  // StrictMode 是一个用来检查项目中潜在问题的工具
  <React.StrictMode>
	{/* 路由绑定 */}
    <RouterProvider router={router}/>
  </React.StrictMode>
);

然后，在页面中输入 http://localhost:3000/login 或 http://localhost:3000/article 就可以进行两个页面的切换了。

这里有几个问题：

  1. 在 路由表列 中，路由路径 对应渲染的 不是一个组件，而是一个模板标签元素，这是不合理的。
  2. 我们直接把 路由 写在 /src/index.tsx 里边，也是不合理的。
  3. 我们应该有一个 router 的目录来专门管理路由。

抽象路由模块

  1. 新建 /src/views 目录，在该目录下分别新建 /src/views/Login/index.tsx 和 /src/views/Article/index.tsx 组件：

Login 组件：

const Login = () => {
  return (
    <div>登录</div>
  )
}

export default Login

Article 组件：

const Article = () => {
  return (
    <div>文章</div>
  )
}

export default Article
  1. 新建 /src/router/index.tsx，用于管理路由以及路由对应的要渲染的组件：
import { createBrowserRouter } from 'react-router-dom';

import Login from '../views/Login'
import Article from '../views/Article'

const router = createBrowserRouter([
  {
    path:'/login',
    element: <Login />
  },
  {
    path:'/article',
    element: <Article />
  }
])

export default router
  1. 在 /src/index.ts 中，引入该路由文件：
import React from 'react';
import ReactDOM from 'react-dom/client';
import { RouterProvider } from 'react-router-dom';
import router from './router';

const root = ReactDOM.createRoot(
  document.getElementById('root') as HTMLElement
);

root.render(
  <RouterProvider router={router} />
);
  1. 在浏览器地址栏上输入 http://localhost:3000/login 或 http://localhost:3000/article 就可以切换对应路由的页面。

路由导航

1. 什么是路由导航

路由系统中的多个路由之间需要进行路由跳转，并且在跳转的同时有可能需要传递参数进行通信。

2. 声明式导航

声明式导航是指通过在模版中通过 **<Link/> 组件 **描述出要跳转到哪里去，比如后台管理系统的左侧菜单通常使用这种方式进行：

语法说明：通过给组件的 to属性 指定要跳转到 路由path，组件会被渲染为浏览器支持的 a 链接，如果需要传参直接通过字符串拼接的方式拼接参数即可。

3. 编程式导航

编程式导航是指通过 useNavigate 钩子得到导航方法，然后通过调用方法以命令式的形式进行路由跳转，比如想在登录请求完毕之后跳转就可以选择这种方式，更加灵活

语法说明：通过调用 navigate 方法传入地址 path 实现跳转。

导航传参

路由导航的 传参 和 获取参数 如下图所示：

第二种传参方式，需要在定义路由时，声明对应 占位符 进行匹配：

const router = createBrowserRouter([
  {
    // 动态匹配后面的 id、type 占位符参数
    path:'/article/:id/:type',
    element: <Article />
  }
])

跳转路由传参：

import { useNavigate } from "react-router-dom"

const Login = () => {
  const navigate = useNavigate()
  return (
    <div>
      <span>首页</span>
      <button onClick={ () => navigate('/article/1/abc') }>点击跳转文章</button>
    </div>
  )
}

export default Login

路由参数获取：

import { useParams } from "react-router-dom"

const Article = () => {
  const params = useParams()
  const id = params.id // 1
  const type = params.type // abc
  
  return (
    <div>文章页面 - { id } - { type }</div>
  )
}

export default Article

路由懒加载

什么是 路由懒加载？

路由懒加载 是 指路由的 JS资源 只有在被访问时才会动态获取，目的是为了优化项目首次打开的时间。

从上图可以看到，当前正在预览的页面是首页，在没有没使用 路由懒加载前，所有菜单路由的 JS 资源都会加载，而使用了 路由懒加载后，只会加载当前被访问的路由的 JS资源。

如何配置路由懒加载？

  1. 把路由修改为由 React 提供的 lazy 函数 进行动态导入。
  2. 使用 React 内置的 Suspense组件 包裹路由中 element 选项对应的组件。
import { createBrowserRouter } from 'react-router-dom';
// import Home from '@/pages/Home'
import { Suspense, lazy } from 'react';
import Loading from '@/components/Loading'; // 路由页面加载组件

// react 的 lazy api 能够让你在组件第一次被渲染之前延迟加载组件的代码（路由懒加载）
const Home = lazy(() => import('@/pages/Home'))

const router = createBrowserRouter([
  // 使用 路由懒加载前 的编写方式
  // {
  //   path: '/',
  //   element: <Home />
  // },
  // 
  {
    path: '/',
    // Suspense 是 React 的内部组件，可以提供异步渲染的能力。
    // Suspense 可以接收一个 fallback 属性，因为现在的路由是一个异步的路由，
    // 所以，fallback 是提供一个占位，相当于是当前组件渲染之前，要显示什么内容。
    // fallback 可以接收一个组件。
    element: <Suspense fallback={<Loading />}><Home /></Suspense>
  }
])

export default router

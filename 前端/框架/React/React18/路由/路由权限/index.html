路由权限

路由权限在各种后台管理系统是是很常见的，路由权限的控制在 Vue 的项目中可以通过路由导航守卫来进行控制，那在 React 中应该怎么呢？React 可没有 Vue 那样的导航守卫，在 React 的函数式组件中，我们可以通过封装一个自定义的 高阶组件（HOC Component）来控制路由的跳转。

封装高阶组件

首先，新建 /src/components/AuthRoute/index.tsx，封装一个 高阶组件，用来控制路由组件的跳转，通过该组件可以在跳转路由页面前进行路由权限的判断：

// 封装高阶组件 -- 权限路由组件
import { Navigate } from "react-router-dom";
import { useSelector } from "react-redux";
import type { RootState } from '@/store/type'

/**
 * 权限路由组件
 * @param props 接收的子组件
 * @returns 组件
 */
function AuthRoute(props: any) {
  // 获取子组件和该路由对应的权限码
  const { children, auth } = props;

  // 获取全局存储的用户信息
  const userState = useSelector((state: RootState) => state.user);

  // 获取该用户的权限路由
  const { authRoutes } = userState

  // 如果该用户有对应路由页面的权限，直接正常跳转对应的路由页面
  // 或者如果该路由页面没有设置权限，也正常跳转
  if ((auth && authRoutes.includes(auth)) || !auth) return children
  
  // 否则跳转 404 页面
  // replace 表示替换之前的路由记录
  return <Navigate to={'/404'} replace />
}

export default AuthRoute

声明路由列表

在 src/router/route.tsx 中，声明路由列表并使用该 路由权限高阶组件：

import { Suspense, lazy } from 'react';
import Loading from '@/components/Loading';
import AuthRoute from '@/components/AuthRoute';
import { Navigate } from 'react-router-dom'

// 布局容器组件，所有路由页面均在布局组件内显示
const Layout = lazy(() => import('@/Layout'))
const Home = lazy(() => import('@/pages/Home'))
const NotFound = lazy(() => import('@/pages/404'))
const Path1 = lazy(() => import('@/pages/path1'))
const Path2 = lazy(() => import('@/pages/path2'))
const Path3 = lazy(() => import('@/pages/path3'))
const Path4 = lazy(() => import('@/pages/path4'))
const Path5 = lazy(() => import('@/pages/path5'))
const Path6 = lazy(() => import('@/pages/path6'))

export const routes = [
  // 这个路由是用来作路由重定向的
  {
    path: '/',
    meta: { hidden: true },
    element: (
      <Suspense fallback={<Loading />}>
        {/* 路由重定向 */}
        <Navigate to='/home'></Navigate>
      </Suspense>
    ),
  },

  {
    path: '/home',
    element: <Suspense fallback={<Loading />}><Layout /></Suspense>,
    children: [
      {
        path: '',
        meta: { title: '首页' },
        index: true,
        element: (
          <AuthRoute>
            <Suspense fallback={<Loading />}><Home /></Suspense>
          </AuthRoute>
        ),
      }
    ]
  },

  {
    path: '/path1',
    element: <Suspense fallback={<Loading />}><Layout /></Suspense>,
    children: [
      {
        path: '',
        meta: { title: '路由页面1', auth: 'path1' },
        index: true,
        element: (
          // 重点在于要给 AuthRoute 路由组件传递当前路由的权限码
          <AuthRoute auth="path1">
            <Suspense fallback={<Loading />}><Path1 /></Suspense>
          </AuthRoute>
        ),
      }
    ]
  },

  {
    path: '/path2',
    element: <Suspense fallback={<Loading />}><Layout /></Suspense>,
    children: [
      {
        path: '',
        meta: { title: '路由页面2', auth: 'path2' },
        index: true,
        element: (
          <AuthRoute auth="path2">
            <Suspense fallback={<Loading />}><Path2 /></Suspense>
          </AuthRoute>
        ),
      }
    ]
  },

  {
    path: '/multiple-path',
    meta: { title: '多路由页面' },
    element: <Suspense fallback={<Loading />}><Layout /></Suspense>,
    children: [
      {
        path: 'path3',
        meta: { title: '路由页面3', auth: 'path3' },
        element: (
          <AuthRoute>
            <Suspense fallback={<Loading />}><Path3 /></Suspense>
          </AuthRoute>
        ),
      },
      {
        path: 'path4',
        meta: { title: '路由页面4', auth: 'path4' },
        element: (
          <AuthRoute auth="path4">
            <Suspense fallback={<Loading />}><Path4 /></Suspense>
          </AuthRoute>
        ),
      },
      {
        path: 'path5',
        meta: { title: '路由页面5', auth: 'path5' },
        element: (
          <AuthRoute auth="path5">
            <Suspense fallback={<Loading />}><Path5 /></Suspense>
          </AuthRoute>
        ),
      },
      {
        path: 'path6',
        meta: { title: '路由页面6', auth: 'path6' },
        element: (
          <AuthRoute auth="path6">

          </AuthRoute>
        )

  },

  {
    path: '*', // 404 页面

    element: <Suspense fallback={<Loading />}><NotFound /></Suspense>
  }
]

我们在配置路由列表并使用路由权限组件时，重点在于：

当我们给路由权限组件传入一个 auth 属性和插入一个路由组件时，会在 AuthRoute 组件的 props 属性中获取到，分别可以通过 props.auth 和 props.children 获取对应的属性和插入的组件。

auth 属性就表示当前配置的该路由页面的权限码，只要用户有该权限码就可以跳转该页面，否则就跳转 404 页面。

使用路由列表

在 src/router/index.ts 中，使用之前声明好的路由列表：

import { createBrowserRouter } from 'react-router-dom';
import { routes } from './route';

const router = createBrowserRouter(routes)

export default router

注册路由

在 src/index.ts 中注册路由：

import React from 'react';
import ReactDOM from 'react-dom/client';
import { Provider } from 'react-redux' // redux 状态管理组件
import { RouterProvider } from 'react-router-dom'; // 路由组件
import router from './router'; // 路由

const root = ReactDOM.createRoot(
  document.getElementById('root') as HTMLElement
);

root.render(
  <Provider store={ store }>
    <RouterProvider router={router}/>
  </Provider>
);

测试页面权限

假设现在有两个用户，分别是 admin 和 visitor，admin 拥有的权限码是 ['path1', 'path2', 'path3', 'path4', 'path5', 'path6']，visitor 拥有的权限码是 ['path1', 'path4', 'path6']。

也就是说 admin 用户可以查看所有页面，visitor 用户只能查看 path1、path4、path6 页面。

可以看到 admin 用户登录后，所有页面都可以正常访问，而切换了 visitor 用户后，只能观看没有配置权限的首页、path1、path4 以及 path6 页面。

注意：这里左侧的菜单栏不管哪个用户登录，都是显示了所有菜单选项的。主要是为了方便展示效果，暂时没有处理根据对应用户路由的不同来进行过滤显示，我们暂且忽略这个小问题，只看路由页面权限控制效果。

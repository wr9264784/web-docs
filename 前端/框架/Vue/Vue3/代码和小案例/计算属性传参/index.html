计算属性传参

前言

我们都知道在 vue 中，可以通过计算属性 computed 来进行计算，并且 computed 是依赖于缓存的。只要依赖的数据不发生变化，每次调用 computed 都是拿的上一次的计算的缓存结果。

好了，现在有个疑问，计算属性能传参吗？

有人可能会说当然可以，只需要在 computed 内部返回一个函数，并且通过这个内部返回的函数来进行参数的接收，如下所示：

<script setup lang="ts">
import { computed } from 'vue'
    
const res = computed(() => {
  return (value) => {
    // ...
  }
})
</script>

其实这样虽然能达到在 computed 中传参的目的，但是这会破坏了计算属性的 依赖缓存 这一机制。也就是说如果按照上述的代码进行传参，那么这个 computed 其实和普通函数没有区别，你每调用一次 computed，它都会重新计算一次。

封装可传参依赖缓存的计算属性方法

我们先看一下以下这个 Table 表格：

上述表格中是一个 Table 组件，表格中有一个计算总价的功能，也就是输入或点击商品数量，会自动计算商品总价。这个计算总价是需要 商品单价 * 商品数量 进行计算的。

普通函数存在的问题

我们可以定义一个简单的 getSum 函数进行计算，并且函数执行时会输出打印内容，然后在表格中的每一行总价中都调用这个方法 getSum 方法来计算总价：

<template>
  <div class="computed-params">
    <el-table :data="tableData" border style="width: 100%">
      <el-table-column
        v-for="item in columns"
        :key="item.key"
        :prop="item.key"
        :label="item.label"
        :align="item.align"
      >
        <!-- 插槽 -->
        <template v-if="item.hasOwnProperty('slot')" #default="{ row }">
          <!-- 数量插槽内容 -->
          <template v-if="item.slot === 'count'">
            <el-input-number v-model="row.count" :min="0" :max="10" />
          </template>

          <!-- 总价插槽内容 -->
          <template v-else-if="item.slot === 'sum'">
            {{ getSum(row) }}
          </template>
        </template>
      </el-table-column>
    </el-table>
  </div>
</template>

<script setup lang="ts">
import { reactive } from 'vue'

defineOptions({
  name: 'ComputedParams'
})

Type Row = {
  name: string,
  price: number,
  count: number,
  sum: number
}    
    
const columns = [
  { label: '商品名称', key: 'name', align: 'center' },
  { label: '商品单价（￥）', key: 'price', align: 'center' },
  { label: '商品数量', key: 'count', align: 'center', slot: 'count' },
  { label: '商品总价（￥）', key: 'sum', align: 'center', slot: 'sum' },
]

const tableData = reactive([
  { name: '华为 Mate60 Pro+', price: 9999, count: 1, sum: 9999 },
  { name: '华为 Mate60 Pro', price: 7999, count: 1, sum: 7999 },
  { name: '荣耀 Magic 60 Pro', price: 6699, count: 1, sum: 6699 },
  { name: '荣耀 Magic 60', price: 4999, count: 1, sum: 4999 }
])

/**
 * 当我们改变某一个商品的数量的时候，需要重新计算那个商品的总价，但是，
 * 这里有一个效率问题，就是改变了某一个商品的数量，会把所有商品的总价都
 * 重新计算了一遍，这不是我们想要的。我们想要的应该是改变哪个商品就重新
 * 计算哪个商品的总价。
 * 
 * 在控制台中，我们可以发现不管是初始化还是改变了某一件商品的数量，都会
 * 打印很多次 'computed'，这是非常耗性能的，如果这是一个复杂的表格，页面
 * 会直接卡死。
 */
function getSum(row: Row) {
  if (row.name) {
    console.log('computed');
    return row.count * row.price
  }
}
</script>

计算属性存在的问题

可能有些人会想到用 计算属性 来替换普通的 getSum 方法，但是前面说过了，计算属性不能传参，否则会导致缓存机制失效。

我们可以使用使用 计算属性 来试验一下：

<template>
  <div class="computed-params">
    <el-table :data="tableData" border style="width: 100%">
      <el-table-column
        v-for="item in columns"
        :key="item.key"
        :prop="item.key"
        :label="item.label"
        :align="item.align"
      >
        <!-- 插槽 -->
        <template v-if="item.hasOwnProperty('slot')" #default="{ row }">
          <!-- 数量插槽内容 -->
          <template v-if="item.slot === 'count'">
            <el-input-number v-model="row.count" :min="0" :max="10" />
          </template>

          <!-- 总价插槽内容 -->
          <template v-else-if="item.slot === 'sum'">
			<!-- 使用计算属性 -->
            <!-- {{ getSumCompouted(row) }} -->
          </template>
        </template>
      </el-table-column>
    </el-table>
  </div>
</template>

defineOptions({
  name: 'ComputedParams'
})

Type Row = {
  name: string,
  price: number,
  count: number,
  sum: number
}    
    
const columns = [
  { label: '商品名称', key: 'name', align: 'center' },

  { label: '商品数量', key: 'count', align: 'center', slot: 'count' },

const tableData = reactive([
  { name: '华为 Mate60 Pro+', price: 9999, count: 1, sum: 9999 },
  { name: '华为 Mate60 Pro', price: 7999, count: 1, sum: 7999 },
  { name: '荣耀 Magic 60 Pro', price: 6699, count: 1, sum: 6699 },
  { name: '荣耀 Magic 60', price: 4999, count: 1, sum: 4999 }

/**
 * 计算属性
 * 
 * 我们可能会想到用计算属性来计算总价，因为计算属性是依赖缓存的，当数据没有
 * 发生变化时，对应的商品是不是重新计算的。
 * 
 * 但是，计算属性应该怎么传参呢？有人可能会想到在计算属性中 return 一个函数，
 * 并通过这个 return 的函数来接收参数。这样是可以，但是这会破坏了计算属性的
 * "依赖缓存" 这一机制。
 * 
 * 导致计算属性变成了普通的函数了。
 */
const getSumCompouted = computed(() => {
   return (row: Row) => {
     console.log('computed');
     return row.count * row.price
   }
})
</script>

封装具有缓存的方法

我们希望 getSum 方法可以像计算属性一样具有缓存的特性，又能进行参数的传递，那么需要怎么做呢？

我们可以封装一个自定义 hooks 函数，这里我定义这个函数的名称为 useComputed，这个函数是一个通用型的函数，它可以接收一个函数作为参数。

然后对这个函数进行处理，最后返回一个新的函数，这个新的函数作用与原来的函数的作用相同，只是新增了缓存的效果。

// useComputed 接受一个函数，并且返回一个新的函数
// computedPrice 的作用和 getSum 的作用相同，只是 computedPrice 具有缓存效果
const computedPrice = useComputed(getSum)

好了，接下来我们就针对这个 useComputed 函数进行封装，我们把这个函数封装成自定义 hooks，到时候可以直接在 vue 组件中引用使用：

import { computed } from "vue"

/**
 * 一个带有缓存的函数
 *  
 * 首先，这个缓存一定是一个 key value 的结构，那么这个 key 是什么呢？
 * 这个 key 就是参数列表，value 就是一个计算属性。
 * 
 * @param fn 要执行的函数
 * @returns 返回一个具有缓存的函数
 */
export const useComputed = (fn: any) => {
  // 定义一个缓存
  const caches = new Map()

  /**
   * 辅助函数，判断某个参数列表是否有缓存
   */
  function getCache(args: any) {
    /**
     * 注意：这里不能直接通过 caches.get(args) 来获取对应的缓存，因为每次调用
     * 这个函数的时候，参数有可能是一个引用型的参数，参数的内容是一样的，但是
     * 地址不一样，比方说，第一次调用的时候传入的是 [1, 2, 3]，第二次调用的时候
     * 传入的也是 [1, 2, 3]，但是这两次的参数的地址不同，是不同的参数。所以第二次
     * 会拿不到这个缓存的结果。
     */
    // return caches.get(args)

    // 所以，我们需要改造一下，把这个 key 参数列表，改造成一个 JSON 格式的字符串。
    return caches.get(JSON.stringify(args))
  }


  // 返回一个具有缓存的函数
  return function (...args: any) {
    // 通过参数列表，拿到对应的缓存数据
    const cache = getCache(args)

    // 判断该参数是否有对应的缓存
    if (cache) {
      // 如果有缓存，直接返回缓存的结果
      return cache
    }

    // 如果没有缓存，就运行原函数，拿到运行的结果把这个结果保存到缓存中，
    // 然后再把这个结果返回
    // const result =  fn(...args)

    // 因为拿到的每一个结果都是需要缓存的，所以，上面需要改动，改成通过
    // computed 来计算结果，这样子才能保证它是响应式且具有缓存的。
    const result = computed(() => fn(...args))

    // 保存结果到缓存中
    caches.set(JSON.stringify(args), result)
    
    return result
  }
}

接下来我们在 vue 组件页面中使用该 hooks：

<template>
  <div class="computed-params">
    <el-table :data="tableData" border style="width: 100%">
      <el-table-column
        v-for="item in columns"
        :key="item.key"
        :prop="item.key"
        :label="item.label"
        :align="item.align"
      >
        <!-- 插槽 -->
        <template v-if="item.hasOwnProperty('slot')" #default="{ row }">
          <!-- 数量插槽内容 -->
          <template v-if="item.slot === 'count'">
            <el-input-number v-model="row.count" :min="0" :max="10" />
          </template>

          <!-- 总价插槽内容 -->
          <template v-else-if="item.slot === 'sum'">
            <!-- {{ getSum(row) }} -->
            <!-- {{ getSumCompouted(row) }} -->

            <!-- 使用带有缓存的方法 -->
            {{ computedPrice(row) }}
          </template>
        </template>
      </el-table-column>
    </el-table>
  </div>
</template>

<script setup lang="ts">
import { reactive } from 'vue'

// 引入自定义 hooks
import { useComputed } from './hooks/useCompouted'

defineOptions({
  name: 'ComputedParams'
})

Type Row = {
  name: string,
  price: number,
  count: number,
  sum: number
}  
    
const columns = [
  { label: '商品名称', key: 'name', align: 'center' },
  { label: '商品单价（￥）', key: 'price', align: 'center' },
  { label: '商品数量', key: 'count', align: 'center', slot: 'count' },
  { label: '商品总价（￥）', key: 'sum', align: 'center', slot: 'sum' },
]

const tableData = reactive([
  { name: '华为 Mate60 Pro+', price: 9999, count: 1, sum: 9999 },
  { name: '华为 Mate60 Pro', price: 7999, count: 1, sum: 7999 },
  { name: '荣耀 Magic 60 Pro', price: 6699, count: 1, sum: 6699 },
  { name: '荣耀 Magic 60', price: 4999, count: 1, sum: 4999 }
])

// 计算总价
function getSum(row: Row) {
  if (row.name) {
    console.log('computed');
    return row.count * row.price
  }
}

/**
 * useComputed() 函数接收一个参数，这个参数是一个函数。
 * 并且返回一个和接收的函数作用相同的新函数，该新函数新增了缓存效果。
 * 
 * 从控制台打印结果来看，初始化页面时，打印了 4 次，因为有 4 行数据，当我们
 * 改变某一个商品的数量时候，控制台只会打印一次，其他商品不会打印，因为它们的
 * 数量没有发生变化，也是就说具有缓存。
 */
const computedPrice = useComputed(getSum)
</script>

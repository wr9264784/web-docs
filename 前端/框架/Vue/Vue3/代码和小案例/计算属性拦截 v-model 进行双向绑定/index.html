计算属性拦截 v-model 进行双向绑定

前言

我们在封装表单组件的时候，我们肯定会用到 v-model，如下图所示：

这里有一个 父组件 和 一个 子组件，父组件 里边有一些数据，我希望把这些数据通过 v-model 的形式传递给 子组件，这是没有问题的。

因为我们都知道 v-model 实际上就是一个 语法糖，它本质上是传递了一个 modelValue 的属性值给到了 子组件，当 子组件 想要更改这个数据的时候，子组件 不能直接更改，因为直接更改就打破了 单项数据流 了。

所以，当 子组件 想要更改这个数据的时候，会抛出一个事件通知 父组件，这个 modeValue 的数据要变化了，让 父组件 去更改这个数据。如下图所示：

正是因为 v-model 这个语法糖把上诉的步骤封装起来了，所以我们才可以在 父组件 中 直接通过 v-model 直接给 子组件 传值，如下代码所示：

<!-- 父组件 -->
<template>
  <div class="v-model_computed">
    <!-- 子组件，自定义封装的表单组件 -->
    <my-input v-model="form" />
</template>

<script setup lang="ts">
import { ref } from 'vue';
import MyInput from './MyInput.vue'

defineOptions({
  name: 'VmodelComputed'
})

interface IterfaceForm {
  keyword: '',
  selectValue: ''
}    
    
const form = ref<IterfaceForm>({} as IterfaceForm)
</script>

v-model 在子组件中存在的问题

在上述的代码中，目前是没有问题的。问题在哪呢，在于 子组件，因为 子组件 是一个封装了表单的组件，也就意味着 子组件 里边会有一些 表单项，像使用 Element Plus 组件库的 el-input、el-select ... 等等。

表单项 它也有 v-model，那么这个时候就比较麻烦了，在 子组件 中，我们就可能会直接把 父组件 传递过来的这个 modelValue 属性值绑定到 表单项 的 v-model 里边。

这样一绑定就出问题了，会出为什么问题呢？看以下的图：

上图中，这样一绑定就相当于，父组件 把数据通过 modelValue 属性传递给了 子组件，子组件 要修改 modeValue 就必须要通过 emit 抛出 update:modelValue 事件给 父组件，让 父组件 进行修改。

例如以下代码示例：

<!-- 这是一个简单的示例子组件 -->
<template>
  <div class="children">
    <div>父组件传递过来的数据：{{ props.modelValue }}</div>
    <button @click="clickHandle">点击通知父组件修改数据</button>
  </div>
</template>

<script setup lang="ts">
defineOptions({
  name: 'ChildrenComp'
})

const props = defineProps<{
  modelValue: number
}>()
const emit = defineEmits(['update:modelValue'])

function clickHandle() {
  emit('update:modelValue', 123)
}
</script>

但是，这里 子组件 通过 v-model 把接收到的 modelValue 直接绑定到了 表单项 中。那么，当这个文本框发生变化的时候，这个文本框通知的 子组件 修改数据，子组件 通过这个 v-model 就直接把 modelValue 的数据修改了，从而没有把修改 modelValue 的事件抛出给 父组件。

也就是说，这个过程造成了一个现象：子组件 修改了属于自己的数据，打破了 单向数据流。如下代码所示：

<!-- 这是一个简单的示例子组件 -->
<template>
  <div class="children">
    <!-- 当这个输入框的内容发生变化时，会抛出事件通知当前子组件直接更改 modelValue 的值， -->
    <!-- 但是 modelValue 并不是子组件本身的数据，而是父组件传递过来的 props 数据，打破了单向数据流。 -->
    <el-input v-model="props.modelValue"></el-input>
  </div>
</template>

<script setup lang="ts">
defineOptions({
  name: 'ChildrenComp'
})

const props = defineProps<{
  modelValue: string
}>()
</script>

有些人可能觉得打破了 单向数据流 好像也没有什么问题，代码也能正常运行。其实当时确实不会怎么样，但是你也就埋下了一个隐患，你的代码也就离 "💩山" 更近一步了。

如果你对这一块不加注意的话，长久下来，你某一个组件的数据可能在其它组件里边各处去私自更改，一旦出了问题你就很不好调试。

本来数据发生了问题，我们只需要去查找修改这个数据的所属组件就好了，但是因为你打破了 单向数据流，这个数据可能还在别的组件中被修改了，那这件事就麻烦了，变的非常不好调试。

解决方案 - 拆分 v-model

那么，知道了问题的所在，解决办法也很简单，就是在 子组件 里边，我们 不能直接使用 v-model 去绑定不属于自己的数据。

那么具体怎么做呢？我们要在 子组件 的 表单项 中不使用 v-model 这个语法糖来进行数据的传递，而是给 表单项 的 绑定 modelValue 属性值，以及给 表单项 绑定一个监听事件 @update:modelValue。

如下代码所示，以下是 子组件 的相关代码，里边有一个 el-input 表单项，该表单项里边还通过插槽嵌套了一个 el-select 表单项：

<!-- MyInput.vue -->
<template>
  <el-input
    :modelValue="modelValue.keyword"
    placeholder="请输入"
    class="input-with-select"
    @update:modelValue="onChange($event, 'keyword')"
  >
    <template #prepend>
      <el-select
        :modelValue="modelValue.selectValue"
        placeholder="请选择"
        style="width: 115px"
        @update:modelValue="onChange($event, 'selectValue')"
      >
        <el-option
          v-for="option in selectOptions"
          :key="option.value"
          :label="option.label"
          :value="option.value"
        />
      </el-select>
    </template>

    <template #append>
      <el-button :icon="Search" />
    </template>
  </el-input>
</template>

<script setup lang="ts">
import { Search } from '@element-plus/icons-vue'

// 对应的 ts 数据类型，可以单独封装出去
interface IterfaceForm {
  keyword: '',
  selectValue: ''
}

interface SelectOptionItem {
  label: string
  value: string
}

const props = defineProps<{
  modelValue: IterfaceForm,
  selectOptions: SelectOptionItem[]
}>()

// 声明事件
const emit = defineEmits(['update:modelValue'])

// 输入框内容发生变化时触发
function onChange(e: string, key: string) {
  console.log(e);

  // 抛出更改 modelValue 的事件，要更改的是整个 modelValue 对象，
  // 虽然每次表单项修改，只改变了 modelValue 的其中某个字段，但是
  // 也要告诉父组件，整个对象发生了变化。
  emit('update:modelValue', {
    ...props.modelValue, // 展开原来的 modelValue，相当于复制了一份
    [key]: e // 修改了某个属性
  })
}
</script>

然后在父组件中调用：

<!-- 父组件 -->
<template>
  <div class="v-model_computed">
    <!-- 自定义封装的表单组件 -->
    <my-input v-model="form" :select-options="options" />
      
    <br>
    <br>
    <!-- 用来测试 v-model 是否正常 -->
    <div>v-model 的值：{{ form }}</div>
    <br>
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import MyInput from './MyInput.vue'

defineOptions({
  name: 'VmodelComputed'
})

// 对应的 ts 数据类型，可以单独封装出去
interface IterfaceForm {
  keyword: '',
  selectValue: ''
}

interface SelectOptionItem {
  label: string
  value: string
}    
    
const form = ref<IterfaceForm>({} as IterfaceForm)

const options = ref<SelectOptionItem[]>([
  { label: 'JavaScript', value: '1' },
  { label: 'Vue3', value: '2' },
  { label: 'Vue2', value: '3' },
])
</script>

解决方案 - 通过 computed 拦截 v-model

上述的代码呢还有一点不好的地方，因为现在这里的 子组件 中的表单项比较简单，也比较少。但是，我们实际项目开发中封装的表单组件肯定不值这么一点，至少会有十几个表单项。

那么针对十几个的表单项，我们都需要对其进行 :modelValue 的绑定、进行 @update:modelValue 的监听。那么就显得很麻烦，那么我们是不是可以优化、简化一下这个操作呢？

我们如果仍然是希望在 表单项 里边可以直接通过使用 v-model 来绑定数据，但是又不破坏 单向数据流 可不可以实现呢？

这个时候我们就可以借助 computed 计算属性 来帮助我们进行优化，我们可以看以下图所示：

我们可以通过上图中的模式进行优化，父组件 通过 modelValue 传递了一个属性给 子组件，当 子组件 想要修改 modelValue 的时候需要抛出一个 update:modelValue 事件。

然后 子组件 内部有 表单项，这里假设是 el-input 输入框，这个输入框不能直接通过 v-model 绑定 modelValue。我们给它的中间套一层 computed 计算属性，也就是说通过 v-model 给 表单项 绑定一个 计算属性。

计算属性 的 完整写法 里边有 get 和 set 用于数据的获取和修改。当我们读取 modelValue 中某个属性的时候，通过 计算属性 的 get 方法返回 modelValue 中对应的要读取的属性。

当输入框发生变化的时候，它会重新给这个 计算属性 赋值，那么就会触发了 计算属性 中的 set 方法，我们只需要在 set 里边触发 update:modelValue 事件就行了。

所以，计算属性 在这个场景中就是一个 中间层，相当于是对这个属性的读取和赋值做了一个拦截，自定定义读取和赋值的行为。

修改之前的 子组件：

<!-- MyInput.vue -->
<template>
  <!-- v-model 绑定的是计算属性-->
  <el-input
    v-model="keyword"
    placeholder="请输入"
    class="input-with-select"
  >
    <template #prepend>
      <!-- v-model 绑定的是计算属性-->
      <el-select
        v-model="selectValue"
        placeholder="请选择"
        style="width: 115px"
      >
        <el-option
          v-for="option in selectOptions"
          :key="option.value"
          :label="option.label"
          :value="option.value"
        />
      </el-select>
    </template>

    <template #append>
      <el-button :icon="Search" />
    </template>
  </el-input>
</template>

<script setup lang="ts">
import { Search } from '@element-plus/icons-vue'
import { computed } from 'vue';

interface IterfaceForm {
  keyword: '',
  selectValue: ''
}

interface SelectOptionItem {
  label: string
  value: string
}

const props = defineProps<{
  modelValue: IterfaceForm,
  selectOptions: SelectOptionItem[]
}>()

// 声明事件
const emit = defineEmits(['update:modelValue'])

// 计算属性
const keyword = computed({
  get() {
    return props.modelValue.keyword
  },
  set(val) {
    emit('update:modelValue', {
      ...props.modelValue, // 展开原来的 modelValue，相当于复制了一份
      keyword: val // 修改了 keyword 属性
    })
  }
})

// 计算属性
const selectValue = computed({
  get() {
    return props.modelValue.selectValue
  },
  set(val) {
    emit('update:modelValue', {
      ...props.modelValue, // 展开原来的 modelValue，相当于复制了一份
      selectValue: val // 修改了 selectValue 属性
    })
  }
})
</script>

父组件还是按照之前的方式进行使用，不用做任何修改，我们看下页面效果：

可以看到，页面能正常使用，父组件中通过 v-model 也能正常获取到数据。

computed 拦截 v-model 优化

但是，上述的代码还是有弊端，因为 modelValue 中有多个属性，就要定义多个 计算属性，对于 子组件 来说不友好。

我们可以再进一步优化，我不希望针对每一个字段都去写一次 computed，而是希望针对整个 modelValue 对象去写这个一个 computed。

而对于表单项进行 v-model 绑定时，就可以通过 计算属性.属性 的方式进行绑定，而更新数据的时候，在计算属性的 set 中更新整个对象，如下所示：

<!-- MyInput.vue -->
<template>
  <!-- v-model 绑定的是计算属性-->
  <el-input
    v-model="computedModelValue.keyword"
    placeholder="请输入"
    class="input-with-select"
  >
    <template #prepend>
      <!-- v-model 绑定的是计算属性-->
      <el-select
        v-model="computedModelValue.selectValue"
        placeholder="请选择"
        style="width: 115px"
      >
        <el-option
          v-for="option in selectOptions"
          :key="option.value"
          :label="option.label"
          :value="option.value"
        />
      </el-select>
    </template>

    <template #append>
      <el-button :icon="Search" />
    </template>
  </el-input>
</template>

<script setup lang="ts">
import { Search } from '@element-plus/icons-vue'
import { computed } from 'vue';

interface IterfaceForm {
  keyword: '',
  selectValue: ''
}

interface SelectOptionItem {
  label: string
  value: string
}

const props = defineProps<{
  modelValue: IterfaceForm,
  selectOptions: SelectOptionItem[]
}>()

// 声明事件
const emit = defineEmits(['update:modelValue'])

// 计算属性
const computedModelValue = computed({
  get() {
    return props.modelValue
  },
  set(val) {
    console.log('修改了~~');
    emit('update:modelValue', val)
  }
})

</script>

那么，上述的代码的运行结果会不会有问题呢？我们看一下结果：

通过上图我们可以看到，当我们输入了内容，输入框发生了变化，父组件 的 v-model 也能接收到数据，但是控制台却没有打印 计算属性 中 set 的 console.log() 的内容，这是为什么呢？

因为 get 方法中返回读取的是 props.modelValue，返回的是整个 modelValue 属性对象。

所以，上述代码中 计算属性 的 set 只有这种情况下才会运行：

computedModelValue = xxx // 对整个计算属性赋值

而我们是通过 v-model="computedModelValue.xxx" 进行 v-model 的绑定的，也就是说当我们的输入框发生变化时，变化的是 computedModelValue 的某个属性值，相当于：

computedModelValue.xxx = yyy // 针对的是计算属性的某个属性值进行修改

所以当 computedModelValue 的某个属性值发生变化时，set 并不会触发。

那此时应该要怎么办呢？

我们是不是可以在 计算属性 中的 get 做一些处理呢？我们可以在 get 中返回一个 代理对象。这个 代理对象 代理的就是 modelValue。

代理对象 里边有各种 陷阱函数，当你读取这个 代理对象 里面的某个属性时，就会触发 代理对象 的 get，这个 get 就可以把对应的属性返回。

当你给这个对象的属性赋值的时候，就会触发 代理对象 的 set，在 set 里边抛出 update:modelValue 事件。如下代码所示：

<!-- MyInput.vue -->
<template>
  <!-- v-model 绑定的是计算属性-->
  <el-input
    v-model="computedModelValue.keyword"
    placeholder="请输入"
    class="input-with-select"
  >
    <template #prepend>
      <!-- v-model 绑定的是计算属性-->
      <el-select
        v-model="computedModelValue.selectValue"
        placeholder="请选择"
        style="width: 115px"
      >
        <el-option
          v-for="option in selectOptions"
          :key="option.value"
          :label="option.label"
          :value="option.value"
        />
      </el-select>
    </template>

    <template #append>
      <el-button :icon="Search" />
    </template>
  </el-input>
</template>

<script setup lang="ts">
import { Search } from '@element-plus/icons-vue'
import { computed } from 'vue';

interface IterfaceForm {
  keyword: '',
  selectValue: ''
}

interface SelectOptionItem {
  label: string
  value: string
}

const props = defineProps<{
  modelValue: IterfaceForm,
  selectOptions: SelectOptionItem[]
}>()

// 声明事件
const emit = defineEmits(['update:modelValue'])

// 计算属性
const computedModelValue = computed({
  get() {
    // 这里不应该返回某个具体字段，因为我们这里双向绑定的对象中的某个字段，
    // 当对象的某个字段的值发生变化时，setter 是不会触发的，
    // 只有整个对象重新赋值为一个新的对象时才会触发 setter，
    // 例如 props.modelValue = xxx，会触发，props.modelValue.keyword = xxx 不会触发。
    // return props.modelValue

    // 所以，我们这里应该返回一个代理对象
    // ！！！！！！注意：！！！！！
    // 如果这里使用的是代理，那么父组件传递进来的 props.modelValue 必须是用 ref 定义的。
    // 因为 emit 返回的是一个新的对象，如果用 reactive 会失去响应式。
    return new Proxy(props.modelValue, {
      /**
       * 将来读取或修改这个代理对象里的某个字段赋值的时候，
       * 都会执行对应的陷阱函数 getter 和 setter。
       * @param target 改动的哪个对象
       * @param key 改动了对象里的哪个字段
       * @param value 改动后的值
       */
      get(target, key) {
        return Reflect.get(target, key)
      },
		
      // 当我们需要改动某个字段的时候，不要真正地去修改对象的属性，而是触发 update:movelValue 事件，
      // 认为你想要改动这个对象里的某个属性，就意味着你是改动了整个对象。要生成一个新的对象。
      set(target, key, value) {
        console.log('update~~~');
        emit('update:modelValue', {
          ...target, // 展开以前的值
          [key]: value // 修改对象的某个字段
        })
        return true
      }
    })
  },
    
  // 其实这里的 set 并不会触发了，这里仍然写在这里是因为 computed 中，
  // 不能只单独写 get，所以，这里还需要把 set 也写上。
  set(val) {
    console.log('修改了~~');
    emit('update:modelValue', val)
  }
})

</script>

页面效果如下所示：

可以看到，当我们修改输入框的内容时，代理对象 中的 set 就会触发，并且在 父组件 中可以正常获取 v-model 绑定的值。

抽离封装成 Hooks

可能到这里有些人会认为，我就做了一个 v-model 的功能，你却要写这么一大堆的代码，是不是有些繁琐了。

其实，我们仔细观察可以发现，这个 计算属性 拦截 v-model 的功能是可以封装成 hooks 的。这样，以后在其它地方要使用类似的功能时就可以复用了。

这里我们新建一个名为 useVModel.ts 的 hooks 文件，在该文件中进行上述代码逻辑的封装处理：

// useVModel.ts

import { computed } from 'vue'

/**
 * 封装自定义的 v-model 和 computed 结合的双向绑定函数
 * @param props 父组件传递的 props 属性
 * @param propName 需要拦截的 props 属性里的那个属性字段
 * @param emit emit 函数
 */
export function useVModel(props: any, propName: string, emit: any) {
  /**
   * 为了不打破单向数据流，在使用双向数据绑定 v-model 时，不能直接使用父组件传递过来的 props，
   * 而是使用 v-model 绑定一个计算属性的值，使用计算属性拦截 modelValue，使用 computed 完整写法。
   * 通过 get 返回需要读取的值、通过 set 设置修改的值。
   */
  const computedModelValue = computed({
    get() { // getter
      // 这里不应该返回某个具体字段，因为我们这里双向绑定的对象中的某个字段，
      // 当对象的某个字段的值发生变化时，setter 是不会触发的，
      // 只有整个对象重新赋值为一个新的对象时才会触发 setter，
      // 例如 props.modelValue = xxx，会触发，props.modelValue.keyword = xxx 不会触发。
      // return props.modelValue

      // 所以，我们这里应该返回一个代理对象
      // ！！！！！！注意：！！！！！
      // 如果这里使用的是代理，那么父组件传递进来的 props.modelValue 必须是用 ref 定义的。
      // 因为 emit 返回的是一个新的对象，如果用 reactive 会失去响应式。
      return new Proxy(props[propName], {
        /**
         * 将来读取或修改这个代理对象里的某个字段赋值的时候，
         * 都会执行对应的陷阱函数 getter 和 setter。
         * @param target 改动的哪个对象
         * @param key 改动了对象里的哪个字段
         * @param value 改动后的值
         */
        get(target, key) {
          // return target[key]
          return Reflect.get(target, key)
        },

        set(target, key, value) {
          // 当我们需要改动某个字段的时候，不要真正地去修改对象的属性，而是触发 update:movelValue 事件，
          // 认为你想要改动这个对象里的某个属性，就意味着你是改动了整个对象。要生成一个新的对象。
          emit(`update:${propName}`, {
            ...target, // 展开以前的值
            [key]: value // 修改那个对象的某个字段
          })
          
          emit('change', { key, value })
          return true // 需要返回 true
        }
      })
    },

    // 其实这里的 set 并不会触发了，这里仍然写在这里是因为 computed 中，
    // 不能只单独写 get，所以，这里还需要把 set 也写上。
    set(newValue) { // setter
      emit(`update:${propName}`, newValue)
    }
  })

  return computedModelValue
}

Hooks 最终优化

这个 hooks 其实还有一些点可以优化，比如这里有一些效率问题，就是每一次都会创建一个新的代理对象，好像是没什么必要。

那么这个也很简单，我们只需要做个缓存即可，针对每一个对象，我们都进行缓存，当调用的时候判断一下缓存里边是否有该对象的 代理对象，如果有，就使用缓存中的 代理，如果没有，就新创建一个 代理对象：

// useVModel.ts

import { computed } from 'vue'

// 缓存
const cacheMap = new WeakMap()

/**
 * 封装自定义的 v-model 和 computed 结合的双向绑定函数
 * @param props 父组件传递的 props 属性
 * @param propName 需要拦截的 props 属性里的那个属性字段
 * @param emit emit 函数
 */
export function useVModel(props: any, propName: string, emit: any) {
  /**
   * 为了不打破单向数据流，在使用双向数据绑定 v-model 时，不能直接使用父组件传递过来的 props，
   * 而是使用 v-model 绑定一个计算属性的值，使用计算属性拦截 modelValue，使用 computed 完整写法。
   * 通过 get 返回需要读取的值、通过 set 设置修改的值。
   */
  const computedModelValue = computed({
    get() { // getter

      // 如果缓存中有对应的代理对象，直接返回缓存中的代理对象，
      // 就不需要创建一个新的代理对象。
      if (cacheMap.has(props[propName])) {
        return cacheMap.get(props[propName])
      }

      // 这里不应该返回某个具体字段，因为我们这里双向绑定的对象中的某个字段，
      // 当对象的某个字段的值发生变化时，setter 是不会触发的，
      // 只有整个对象重新赋值为一个新的对象时才会触发 setter，
      // 例如 props.modelValue = xxx，会触发，props.modelValue.keyword = xxx 不会触发。
      // return props.modelValue

      // 所以，我们这里应该返回一个代理对象
      // ！！！！！！注意：！！！！！
      // 如果这里使用的是代理，那么父组件传递进来的 props.modelValue 必须是用 ref 定义的。
      // 因为 emit 返回的是一个新的对象，如果用 reactive 会失去响应式。
      const proxy = new Proxy(props[propName], {
        /**
         * 将来读取或修改这个代理对象里的某个字段赋值的时候，
         * 都会执行对应的陷阱函数 getter 和 setter。
         * @param target 改动的哪个对象
         * @param key 改动了对象里的哪个字段
         * @param value 改动后的值
         */
        get(target, key) {
          // return target[key]
          return Reflect.get(target, key)
        },

        set(target, key, value) {
          // 当我们需要改动某个字段的时候，不要真正地去修改对象的属性，而是触发 update:movelValue 事件，
          // 认为你想要改动这个对象里的某个属性，就意味着你是改动了整个对象。要生成一个新的对象。
          emit(`update:${propName}`, {
            ...target, // 展开以前的值
            [key]: value // 修改那个对象的某个字段
          })
          
          emit('change', { key, value })
          return true // 需要返回 true
        }
      })

      // 添加到缓存中
      cacheMap.set(props[propName], proxy)
      return proxy
    },

    // 其实这里的 set 并不会触发了，这里仍然写在这里是因为 computed 中，
    // 不能只单独写 get，所以，这里还需要把 set 也写上。
    set(newValue) { // setter
      emit(`update:${propName}`, newValue)
    }
  })

  return computedModelValue
}

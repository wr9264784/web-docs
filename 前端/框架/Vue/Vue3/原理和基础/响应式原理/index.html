简单理解 defineProperty 和 Proxy

前言

我们经常会见到这样的说法：vue3 的 proxy 的效率远远高于 vue2 的 defineProperty。

但是为什么呢？我们要搞清楚这个为什么，与其花时间去背一些八股文，还不如花同样的时间，从源码的角度把它们彻底的认识清楚。

Vue 响应式要做什么

首先我们要思考一个问题，vue 的响应式到底要做什么？

无非就是做两件事：

  1. 当我们读取这个对象的属性的时候，我们要知道它被读取了，我们可能需要在读取的时候做一些事情。
  2. 当我们给这个对象的属性重新赋值的时候，我们也要知道它被赋值了，我们也可能需要在赋值的时候做一些事情。

但是，仅仅是通过对这个 对象 进行 取值 或 赋值 操作，我们无法插手想要做的事情。

如下代码示例，我们无法插手对象的获取和赋值操作：

const obj = {
  a: 1,
  b: 2,
  c: {
    a: 1,
    b: 2
  }
}

// 读取 obj 对象中的 a 属性时，直接就读取了，
// 我们无法知道它被读取了，无法在读取时做一些事件
console.log(obj.a)

// 赋值 obj 对象的 a 属性时，直接赋值了，
// 我们同样无法知道它被赋值了，无法在赋值的时候做一些事情
obj.a = '1'

defineProperty

我们想要插手对象的读取或赋值，就必须要想个办法，把这个属性的读取和赋值变成一个函数：

  1. 我希望将来在读取这个属性的时候，运行这么一个函数。
  2. 给这个属性赋值的时候，同样运行这个函数，并且传递一个参数，参数为需要修改的新值。
  3. 只要一变成函数就简单了，我们就可以随时插手这个对象的读取和赋值的操作了。

那么怎么样变成一个函数呢？在 ES6 之前没有别的办法，只有 Object.defineProperty。

通过以下代码实例，简单观察 Object.defineProperty：

let v = obj.a // 拿到这个对象中 a 的原始值

/**
 * 通过 Obejct.defineProperty 把这个对象的某一个属性，给它变成这么一个
 * get 和 set 函数。当读取 'a' 属性的时候，运行这个 get 函数，当赋值 'a'
 * 属性的时候，运行这个 set 函数。
 */
Object.defineProperty(obj, 'a', {
  /**
   * 读取这个对象的某个属性时，就返回这个属性的原始值
   * @returns 某个属性的原始值
   */
  get() {
    // 并且在 get 函数内部，我们就可以插手一些事情了，
    // 也就可以知道这个属性被读取了。
    console.log('属性 a 被读取了');
    return v
  },

  set(val) {
    console.log('属性 a 被修改了');
    obj['a'] = val // 这里的赋值后续会说
  }
})

obj.a; // 打印 => 属性 a 被读取了
obj.a = '1' // 打印 => 属性 a 被修改了

观察 - vue2 里的 observe

Object.defineProperty 是针对某个对象的属性，对这个 obj 的 属性a 是如此，属性b 是如此，属性c 也是如此，对 属性c 里的 a 和 b 也是同样如此。

由于 vue2 是针对属性的监听，所以它就必须要去深度遍历这个 obj 对象里边的每一个属性。它必要要有这个动作，因为它是针对属性的监听。

所以呢，在 vue2 里边它写了这么一个函数叫 observe，你要使用这个对象之前，就必须要通过这个函数对这个对象进行监听，这个过程就叫做 观察。

在 vue2 里边，观察 的方式就是深度遍历每一个属性，把每一个属性的 读取和赋值变成函数。

/**
 * 观察函数
 * 对某个对象进行监听，对它的每一个属性进行深度遍历，监听它里面的每一个属性。
 * @param { Obejct } obj 需要监听的对象
 */
function observe(obj) {
  for (const k in obj) {
    let v = obj[k]

    // 如果某个属性值仍然是一个对象，继续递归调用
    if (isObj(v)) observe(v)

    // 针对 obj 的每一个属性，做同样的事情，监听这个属性
    Object.defineProperty(obj, k, {
      /**
       * 读取这个对象的某个属性时，就返回这个属性的原始值
       * @returns 某个属性的原始值
       */
      get() {
        // 并且在 get 函数内部，我们就可以插手一些事情了，
        // 也就可以知道这个属性被读取了。
        console.log(`属性 ${k} 被读取了`);
        return v
      },
    
      set(val) {
        console.log(`属性 ${k} 被修改了`);
        obj[k] = val
      }
    })
  }
}

/**
 * 辅助函数，用来判断某个值是否为对象
 * @param { any } value 要判断的值
 */
function isObj(value) {
  return typeof value === 'object' && v !== null
}

observe(obj)
obj.a;
obj.a = '1'

在这种做法下，它就有一个天生的缺陷：

由于它是针对每一个属性的监听，所以必须要进行深度的遍历，这会有效率的损失。由于在观察这个步骤里边，它完成了深度遍历，就是说在这个时间节点里，这个对象的有的属性都被监听到了，都被改成 get 和 set 函数了。但是这一步一旦完成了之后，你再去新增属性，它就不知道你有新增的操作了。因为对这个新增的属性而已，它是没有进行监听的。监听的步骤在新增之前就已经结束了。删除也是同理。

这就是为什么 vue2 它无法监听属性的新增和删除。

vue2 中不能被监听到的操作

对象和数组中不能被监听到的操作：

  • 对象：
    • 直接通过下标新增：obj[key] = value
    • 通过 delete 关键字删除属性：delete obj.key
  • 数组：
    • 直接通过下标赋值：arr[i] = value
    • 直接修改数组长度：arr.length = newLength

分析 vue2 无法监听数组和对象的这些变化的原因，首先，vue2 是通过 Object.defineProperty(obj, key, value) 这种方式监听数据的。

  • 对于对象
    • Object.defineProperty() 针对的是对象的某个属性，而且这个操作在 vue 的初始化阶段就完成了，所以新增的属性无法监听，通过 set 方法 新增对象就相当于初始化阶段的数据响应式处理。而删除无法监听是因为 Object.defineProperty() 没有提供监听删除的方法。
  • 对于数组
    • Object.defineProperty() 是可以对数组实现监听操作的，但是 vue 并没有实现这个功能，因为数组长度不定而且数据可能会很多，如果对每一个数据都实现监听，性能代价太大。但是注意：数组中的元素是引用类型时是会被监听的。

proxy

根据前面的 defineProperty，我们知道了它存在的问题：在 vue2 中无法监听到属性的 新增 和 删除。

那么 vue3 是怎么解决的呢？其实核心道理是一样的，无论是 vue3 还是 vue2 都必须要把 **读取 **和 赋值 变成函数，只不过变成函数的方式不一样。

在 vue3 里边它就不再去针对这些属性进行监听了，而是直接监听整个对象。监听整个对象就不需要遍历了，不再需要管里边的属性是什么。

将来你要往这个对象里边添加属性或删除属性都无所谓了，因为监听的是整个对象，只要这个对象发生了变化，不管是新增、修改、删除都能接收到通知。

vue3 中就是通过 proxy 来完成 观察 的。

在 vue2 中，defineProperty 是直接去遍历 原始对象 里边的每一个属性，改动 原始对象，监听每一个属性。

在 **vue3 **中，Proxy 不再遍历监听每一个属性了，不再修改 原始对象，而是产生一个新的代理对象，所有操作都是针对于这个 代理对象 进行的。只要 代理对象 发生了变化，Proxy 里边的 get 和 set 就会收到通知。

const obj1 = {
  a: 1,
  b: 2,
  c: {
    a: 1,
    b: 2
  }
}

/**
 * 通过 new Proxy 创建一个实例对象，这个对象称之为 代理对象。
 * 将来不管是 读取属性 还是 修改属性，都是通过这个 代理对象 去完成的。
 */
const proxy = new Proxy(obj1, {
  /**
   * 读取对象的属性时触发
   * @param { Object } target 原始对象
   * @param { String } k 读取的哪个属性
   */
  get(target, k) {
    console.log(`属性 ${k} 被读取`);
    let v = target[k]
    return v
  },

  /**
   * 修改对象的属性时触发
   * @param { Object } target 原始对象
   * @param { String } k 修改的哪个属性
   * @param { any } val 修改后的值
   */
  set(target, k, val) {
    let v = target[k]

    // 判断一下修改的值和原始的值是否不相同，不同就进行修改
    if (v !== val) {
      target[k] = val
      console.log(`属性 ${k} 被修改`);
    }
  }
})

proxy.a
proxy.a = 'proxy'

观察 - vue3 里的 observe

注意：

vue3 的 proxy 中，虽然监听的是整个对象，但是仍然会需要递归判断属性是否为对象，如果是，就把这个属性变成代理。

这里可能会有人说，那这里不是有递归吗？不是同样需要遍历属性吗？

虽然这里有递归，但这个递归不是一开始进行的，一开始是观察函数接收一个对象，创建一个代理就完事了。只有当你读到对象的某个属性的时候，Proxy 发现这个属性是一个对象，这时候才会把这个属性变成代理。没有读取属性的时候，这里的递归是不会进行判断执行的。

/**
 * 观察
 */
function observe_vue3(obj) {
  const proxy = new Proxy(obj1, {
    get(target, k) {
      let v = target[k]

      // 如果某个属性值是对象，就把这个属性值变成代理。
      // 没有读取属性的时候，这里的递归是不会进行判断执行的。
      if (_isObject(v)) {
        v = observe_vue3(v)
      }

      console.log(`属性 ${k} 被读取`);
      return v
    },
  
    set(target, k, val) {
      let v = target[k]
  
      if (v !== val) {
        target[k] = val
        console.log(`属性 ${k} 被修改`);
      }
    }
  })

  return proxy
}

const proxyObject = observe_vue3(obj1)

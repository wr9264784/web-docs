    WebSocket 的心跳机制

    心跳机制主要是用来检测服务端和客户端是否建立了 WebSocket 的连接，是否保持着这个连接的关系。

    为什么叫心跳呢？顾名思义就是每隔一段时间去看看服务端和客户端的连接关系。如果说客户端和服务端没有这连接的关系，那么就要尝试重连。

    前端实现

    我们通过类来继承并进一步封装 WebSocket：

    // socket.js

const WS_MODE = {
  MESSAGE: 'MESSAGE',
  HEART_BEAT: 'HEART_BEAT'
}

// 我们可以写一个类继承 WebSocket 对象
class WS extends WebSocket {
  constructor(url) {
    super(url);

    // 定义两个连接状态，一个是连接状态，一个是未连接的状态
    this.connectedStatus = false // false 未连接，true 已连接

    // 当我们重连的时候也是需要用到 url 的，所以我们定义一个变量来存储 url
    this.wsUrl = url

    this.heartBeatTimer = null // 心跳检测的定时器
    this.reconnectTimer = null // 重新连接的定时器

    this.init()
  }

  // 初始化方法
  init() {
    this.bindEvent()
  }

  // 绑定的一些方法
  bindEvent() {
    // 这里的 this 就是 websocket 的实例对象
    this.onopen = this.openHandler
    this.onmessage = this.messageHandler
    this.onclose = this.closeHandler
    this.onerror = this.errorHandler
  }

  openHandler() {
    console.log('-- 客户端已连接 --');
    this.connectedStatus = true // 连接成功后，将连接状态改为 true
    this.startHeartBeat() // 连接成功后，开启心跳检测
  }

  messageHandler(e) {
    console.log('-- 客户端接收到数据 --', e.data);
    const { mode, msg } = JSON.parse(e.data)

    switch (mode) {
      case WS_MODE.MESSAGE:
        console.log('-- MESSAGE --', msg)
        this.send(JSON.stringify(JSON.parse(e.data)))
        break;
      case WS_MODE.HEART_BEAT:
        console.log('-- HEART_BEAT --', msg)
        this.connectedStatus = true // 收到心跳消息后，将连接状态改为 true
        break;
      default:
        break;
    }
  }

  closeHandler(e) {
    console.log('-- 连接已关闭 --', e);
    this.connectedStatus = false // 断开连接后，将连接状态改为 false

    // 在 startHeartBeat 函数中，会进行 setInterval 定时器，所以需要清除
    if (this.heartBeatTimer) {
      clearInterval(this.heartBeatTimer)
      this.heartBeatTimer = null
    }

    if (this.reconnectTimer) {
      clearTimeout(this.reconnectTimer)
      this.reconnectTimer = null
    }

    this.reconnect() // 当连接断开的时候，重新连接
  }

  errorHandler(e) {
    console.log('-- 连接出错 --', e);
    this.connectedStatus = false // 连接出错后，将连接状态改为 false
    this.reconnect() // 当连接出错的时候，重新连接
  }

  // 开启心跳检测
  startHeartBeat() {
    // 隔一段时间发送一次心跳消息
    this.heartBeatTimer = setInterval(() => {

      // 如果是连接中的状态才发送消息
      if (this.connectedStatus) {
        // 发送心跳消息
        const msg = JSON.stringify({
          mode: WS_MODE.HEART_BEAT,
          msg: '心跳检测'
        })
        this.send(msg)
      }
      else {
        clearInterval(this.heartBeatTimer)
        this.heartBeatTimer = null
      }

      // 模拟关闭
      this.waitForResponse()
    }, 5000)
  }

  // 重新连接
  reconnect() {
    // 一段时间后，重新连接
    return new Promise((resolve) => {
      this.reconnectTimer = setTimeout(() => {
        resolve(new WS(this.wsUrl))
      }, 3000)
    })
  }

  // 模拟关闭
  waitForResponse() {
    this.connectedStatus = false

    setTimeout(() => {
      this.close() // 关闭连接
    }, 2000)
  }
}

export default WS

    WebSocket 的基本使用

    WebSocket 是一种网络通信协议，它提供了在单个 TCP 连接上进行 全双工通讯 的通道。这意味着 服务器和客户端之间可以相互发送信息，而不需要客户端像传统 HTTP 请求那样不断地向服务器发送请求来等待响应。WebSocket 非常适合需要实时数据通信的应用，如在线游戏、聊天应用、实时通知系统等。

    useReducer

    作用：useReducer 和 useState 的作用类似，用来管理 相对复杂 的状态数据。

    基础使用

    1. 定义一个 reducer 函数（根据不同的 action 返回不同的新状态）。
    2. 在组件中调用 useReducer，并传入 reducer 函数和 状态的初始值。
    3. 事件发生时，通过 dispatch 函数分派一个 action 对象（通知 reducer 要返回哪个新状态并渲染UI）

    React.memo

    作用：允许组件在 props 没有改变的情况下跳过重新渲染，也是常用来做 性能优化 的 API。

    React 组件默认的渲染机制

    默认机制：顶层组件发生重新渲染，这个组件树的子级组件都会被重新渲染。

    思考：如果 Son 组件本身并不需要重新渲染更新，是否存在浪费？

    // memo
// 作用：允许组件在props没有改变的情况下跳过重新渲染

import { useState } from 'react'

function Son() {
  console.log('子组件被重新渲染了')
  return <div>this is son</div>
}

function App() {
  const [, forceUpdate] = useState()
  console.log('父组件重新渲染了')
  return (
    <>
      <Son />
      <button onClick={() => forceUpdate(Math.random())}>update</button>
    </>
  )
}

export default App

    useCallback

    作用：在组件多次重新渲染的时候缓存函数，这也是一个常用来做 性能优化 的 Hook。

    看个场景

    当父组件给子组件传递一个 引用类型 prop 的时候，即使我们使用了 React.memo 函数依旧无法阻止 子组件 的渲染，其实传递 prop 的时候，往往传递一个回调函数更为常见，比如实现子传父，此时如果想要避免子组件渲染，可以使用 useCallback 缓存 回调函数。

    Redux 的使用

    Redux 介绍

    Redux 是 React 最常用的集中状态管理工具，类似于 Vue 中的 Pinia（Vuex），可以独立于框架运行。 作用：通过集中管理的方式管理应用的状态。

    为什么要使用 Redux？

    1. 独立于组件，无视组件之间的层级关系，简化通信问题。
    2. 单项数据流清晰，易于定位 bug。
    3. 调试工具配套良好，方便调试。

    useState 自动推导

    简单场景

    通常 React 会根据传入的 useState 的默认值来自动推导 TS 类型，不需要显式标注类型。

    使用场景：一般适用于 明确的初始值。

    import { useState } from "react"

function App () {
  // val 会被自动推断为布尔类型
  // setVal 方法调用时只能传入布尔类型
  const [val, setVal] = useState(false)

  setVal(true) // 调用时就只能传入 boolean 类型
  // setVal(100) // 报错

  // list 会被自动推导为 number[] 类型
  const [list, setList] = useState([1, 2, 3])

  setList([4, 5, 6]) // 调用时，只能传入 number[] 类型
  // setList(['1', '2', '3']) // 报错

  return (
    <div>APP</div>
  )
}

export default App
